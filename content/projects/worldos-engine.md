# WorldOS 世界生成引擎

WorldOS 的核心不是某个页面，而是生成管线。

一个节点进入系统后，可以投影到：

- 首页最近点亮。
- Atlas 世界地图。
- Timeline 时间流。
- Archive 现实检索。
- Paths 精选路径。
- Node 详情页。

这就是 Node → Projection → Page 的思路。内容只维护一份，世界可以有多种呈现。

## 为什么是投影而不是复制

如果每个页面各自维护一份内容，更新时必然遗漏某个副本。WorldOS 的做法是维护唯一事实源——节点的数据文件和正文文件——然后由投影管线生成各个页面需要的数据结构。首页看到的是节点列表的子集，地图看到的是节点和区域的拓扑，时间线看到的是节点的时间排序。同一份数据，不同的切片。

## 投影管线的边界

投影只读不写。它从数据文件读取节点、区域、关系和事件，生成公开 JSON 索引和页面所需的数据结构，但不会修改原始数据。这保证了内容维护和页面生成是两个独立的环节，不会互相污染。
