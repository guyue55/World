# 正式主线导入边界

公开产品页只允许依赖：

- World Kernel 数据与路由策略；
- product / world / node / common 组件；
- 公开内容读取工具；
- 低光 AI 与路径导览；
- 明确经过审计的 UI 工具。

不得直接导入：

- R8 动态宇宙 runtime；
- V/R 阶段实验功能；
- 私密档案、AI 工作台、创世台服务层；
- 只为审计、治理、生产实验存在的内部页面组件。

这条法则保证"后端控制权限，前端控制体现"，也保证公开体验不会重新变成阶段页集合。

## 如何检查导入边界

`check:worldos-mainline-governance` 脚本会扫描所有公开页面和主线组件的 import 语句，确认它们没有引用 `_legacy`、`r8-*`、`v*` 等阶段目录。如果有人在新开发中不小心引入了旧组件，门禁会立即拦截并给出修复建议。这是防止 legacy 回流的自动防线，比人工 code review 更可靠，因为它每次 `check:daily` 都会运行。
