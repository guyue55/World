# WorldOS 项目运转原则

项目继续推进时必须遵守：

- 小步提交，中文说明。
- 先做边界，再做表现。
- 先复用内核，再新增概念。
- 公开路径必须可解释、可返回、可搜索。
- 私密内容默认不进入公开构建。
- 任何生产状态都必须有真实证据。

工程里的世界法则，决定这个世界能不能长期被维护。

## 为什么是这些原则

每条原则都来自真实的踩坑经验。小步提交是因为大范围变更无法审查；先做边界是因为后补权限会漏；先复用内核是因为重复造轮子会让脚本爆炸；公开路径三要素是因为没有返回路径的页面就是死胡同；私密不进公开构建是因为前端隐藏不可靠；生产状态需要真实证据是因为 build wrapper 不能代替线上验证。

## 原则与门禁的关系

原则是价值观，门禁是执行手段。`check:daily` 验证主线代码可理解，`check:boundary` 验证权限和脚本边界，`check:rc:fast` 验证发布候选。门禁的存在不是为了阻碍开发，而是让原则在日常工作中自动生效，不需要每次都靠记忆和自觉。
