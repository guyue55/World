# 每周世界心跳

周心跳是世界的整周体检。每周日下午运行一次 npm run release:local-rc，让 22 次 HTTP 检查和 20 次浏览器检查一起跑完，然后在 status 页对照一遍指标：节点、关系、路径、事件、构建大小。

## 检查项

周心跳并不是"跑完就好"的仪式，它要求三个动作：读数、比较、决定。读数指的是把关键指标写到当周的 memory 节点；比较指的是与上周对照，判断增速与结构变化；决定指的是给下一周画一条最低执行线，比如"新增 5 节点或修复 3 个 lint"。

## 为什么要有

没有周心跳，世界会在不知不觉里进入负债。技术债、内容债、心情债都会累积。有了周心跳，创作者不需要靠"节奏感"来维护世界，只需要相信仪表盘。

## 与 Phase 结束节奏的关系

每个 Phase 结束都会跑 release:local-rc，那是 Phase 门禁；而周心跳是 Phase 之内的可视化节拍。两者共同保证世界的呼吸不会中断。
