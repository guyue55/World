# 服务端权限边界说明

前端显隐只能改善体验，不能承担安全职责。

因此：

- 私密、家庭、保险箱和创世台内容不得进入公开索引。
- 旧阶段入口必须由 middleware 或服务端页面重定向/阻断。
- `robots.txt` 与 `sitemap.ts` 必须和 route policy 一致。
- 公开 AI 只能读取公开上下文。
- 任何危险动作都必须有服务端边界和审计记录。

这条规则保护世界的公共前厅，也保护还没有公开的内室。

## 为什么前端显隐不够

前端隐藏可以被绕过：查看源码、修改 CSS、拦截网络请求。如果私密内容存在于前端 bundle 中，它就已经泄漏了。古月浮屿的做法是：私密内容不进入公开构建产物，私密路由由 middleware 在服务端直接重定向，API 路由各自校验权限。前端组件只根据已过滤的数据决定"展示什么"，不承担"能不能访问"的判断。这是安全边界，不是体验优化。
