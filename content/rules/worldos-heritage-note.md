# 世界传承约定

传承约定的目的不是准备结束，而是让世界始终有下一段生命的可能。有了约定，世界的持续性不再仅取决于创作者的健康与热情。

## 三条约定

第一条：owner-only 数据永不进入 AI 训练与公开索引，但可通过导出管线打包给指定继承人。第二条：world-index.json 与 birth certificate 是最小传承包，可脱离创作者本人被理解。第三条：schema 演化日志与 code map 一起构成"世界说明书"，让接手人理解结构。

## 接手人范围

默认为家庭成员；也可指定为特定信任的朋友或组织。传承协议不涉及所有权流转，只涉及"能否继续读、继续写"。

## 与 schema 演化日志的关系

演化日志是传承包里最容易被低估的一份文件。有了它，接手人才能理解为什么每个字段都存在。

## 触发条件

传承包默认每季度自动生成一次，存放在 owner-only 的备份目录。接手人清单每年年末复审一次。

## 与家庭记录索引的关系

传承包的默认接收人是家庭；家庭记录索引原则保证\"被继承\"时不会失控。
