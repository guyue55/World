# 首次 Codex 提交故事

第一条由 Codex 独立完成、格式合规的中文 commit 是一条不起眼的 chore 提交。但那次提交把"AI 协作"从口头约定变成了工程事实。

## 那条 commit

提交内容是修正一个 legacy 目录的错别字。commit 信息用 chore(legacy): 修正拼写错误 的格式。整个 diff 只有一行，却是协作的分水岭：从此每一次 Codex 的动作都留下了可回溯的痕迹。

## 为什么记录

因为它证明规范可以被 AI 稳定执行。规范一旦被证明可执行，就会自然扩散到所有 commit。日后翻 git log 时，中文 commit 成为世界呼吸的记录。

## 与协作工作流的关系

这条 commit 是 codex-agent-workflow 规则的起点。规则不是先写出来再执行的，而是在这条 commit 之后逐步凝固的。

## 后来的每一条提交

从那条 chore 之后，git log 里的中文渐渐多了起来。翻回去看，每一条中文 commit 都能唤起当时的情境。这也是中文 commit 存在的另一个理由——它不只是格式规范，也是记忆的锚点。
