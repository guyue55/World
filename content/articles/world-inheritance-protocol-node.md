# 世界传承协议

传承协议节点是把传承约定 (rule) 变成可被 AI、外部系统读取的文档形态。

## 三个字段

inheritanceContact（接手人邮箱/信物）、inheritanceTriggerConditions（何时启动）、inheritanceBundlePath（打包物存放位置）。

## 与 rule 节点的关系

worldos-heritage-note 是 rule 类型的宣告，本节点是 document 类型的规格。规格更细节，宣告更简短。两者互为对照。

## 与导出工具的关系

导出工具在打包时把本节点也写进 zip 首页，让接手人一打开就能读到"这个世界是谁的、要交给谁、怎么继续"。

## 一份说明

本节点与 worldos-heritage-note (rule) 互为对照。rule 说"应该如何"，document 说"具体怎样"。规格更细节，宣告更简短。

## 与备份策略的关系

传承包 = 世界快照 + 数据备份 + 说明书三者的合集。任何一份缺失，传承就不完整。
