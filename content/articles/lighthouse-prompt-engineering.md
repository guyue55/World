# 灯塔提示词工程

灯塔的 prompt 工程有两条铁律：不喂私密数据、总是给出可验证来源。

## 上下文裁剪

每次调用 AI 之前，先从 world-index.json 里挑相关节点：优先 lifeStage=bloom 的节点、优先与查询有共同 tag 的节点、优先属于当前访客所在路径的节点。裁剪之后的上下文不超过 8 个节点。

## Prompt 骨架

系统 prompt 声明"你是灯塔，只根据给定节点回答"。用户 prompt 附上节点摘要与查询。回答必须以"节点 slug"标注来源。

## 与 ai-prompt-hygiene 的关系

本文档是 hygiene 规则的具体执行。规则说"不喂什么"，本文档说"喂进去的怎么写"。

## 一次实验

把上下文从 20 个节点裁到 8 个，AI 回答质量反而提升。原因是相关性变高、噪声变少。少即是多，在 prompt 里尤其真。
