# 数据备份策略

数据是世界的记忆。失去数据，世界就消失了。

## 3-2-1 原则

- **3** 份数据副本
- **2** 种不同存储介质
- **1** 份离线/异地备份

## 备份层次

1. **本地即时备份**：rsync 到外置硬盘
2. **本地定期备份**：cron + tar 压缩包
3. **云端异地备份**：rclone 到云存储

## 验证策略

备份不验证等于没备份。定期从备份恢复数据，验证完整性。

## 加密

异地备份必须加密。使用 gpg 或 rclone 内置加密功能。

```bash
rclone copy /local/data remote:backup --crypt-encrypt
```

## 古月浮屿的实践

`npm run backup:world` 一键导出并备份世界数据，保留 30 天历史。


## 恢复演练

光有备份不够，必须定期做恢复演练。每季度从备份中随机抽取几个文件恢复，验证内容完整、格式正确。把恢复步骤写成文档，确保在紧急情况下不需要回忆就能操作。记录每次演练的结果，包括恢复耗时和发现的问题。
