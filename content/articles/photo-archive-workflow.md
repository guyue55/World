# 照片归档工作流

照片是最容易堆积的数据。没有归档工作流，一年后就会累积几万张，逐渐失去被翻看的可能。WorldOS 的做法是每季度做一次分层。

## 四层结构

精选层：进入公开 memory 节点，仅少数几张（3-5 张/季度）。日常层：owner-only 相册，按月份组织，仅索引卡片进入世界。原始层：外部硬盘冷备，不进入 world-index。废片层：季度整理时直接删除。

## 每季度动作

季度末的一个周日下午，把当季所有照片过一遍。精选出候选，写入 memory 节点的护照。日常层用命名规则批量整理。剩下的按体积压缩后写入冷备。

## 与家庭记录索引的关系

照片工作流是家庭记录索引原则的落地：精选进入公开层的照片必须是可展示的，其他都在护照层留下痕迹但不暴露内容。

## 一年后的复盘节奏

每年年末做一次跨季度回顾：把四个季度的精选照片放在一起，选出年度封面。年度封面进入年度总结节点，成为那一年记忆的入口。这一动作只花一小时，但让相册每年都有可回到的顶点。
