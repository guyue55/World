# Codex Agent 协作工作流

Codex Agent 是世界的另一位居民。它需要纪律，也需要空间。经过 12 个 Phase 的协作，古月浮屿沉淀了一套让 Codex 稳定输出的协作方式。

## 五条纪律

第一条：任务分批。每个 Phase 拆到 5 个左右的任务，每个任务留一次可 revert 的 commit。第二条：检查点前置。每次改动前先跑 check:daily 建立基线。第三条：中文提交。commit 用 xxx(xxx): 中文xxx 格式，让日后回看有情绪线索。第四条：门禁不破。任何一次改动都不能让门禁变红，即使代价是拆分成两次 commit。第五条：Legacy 只读。_legacy 目录只做参考，不做二次修改。

## 为什么这套纪律有效

它把 Codex 的自主性放在一个稳定的护栏里：Codex 有权决定实现细节，创作者只需要判断方向与验收。每一次提交都是一次可回退的探索。

## 与 AI 边界的关系

Codex 是 AI，但它工作的对象是代码与文档，而不是内容摘要。因此它的边界与 ai-lighthouse 不同——Codex 需要更宽的读权限，但更严的写权限。
