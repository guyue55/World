# WorldOS M29 高保真体验打磨标准

> [!IMPORTANT]
> M29 的目标是把“能用”打磨到“想逛”。这不是加更多特效，而是让视觉、音频、交互、空状态和错误状态统一。

## 1. 目标

- 每个场景有稳定节奏。
- 动效不过度。
- 空状态有世界语义。
- 错误状态不破坏沉浸。
- 声景与视觉同频。
- 移动端和 reduced-motion 同样专业。

对应支柱：S1 独立空间、S2 真实穿梭、S5 统一世界观、S6 长期回访。

## 2. 打磨维度

| 维度 | 标准 |
| --- | --- |
| 视觉 | 场景可区分，层级清楚，不像模板 |
| 动效 | 服务叙事，不遮挡，不晕动 |
| 音频 | 默认静音，开启后短、轻、可控 |
| 空状态 | 告诉用户下一步，而不是空白 |
| 错误状态 | 解释原因和可恢复路径 |
| 移动端 | 首屏可用，按钮不挤，内容不遮挡 |
| 可访问性 | focus、keyboard、contrast、reduced-motion |

## 3. 实施步骤

1. 全场景截图墙。
2. 标注重复模板感。
3. 标注遮挡、溢出、移动端问题。
4. 按 P0/P1/P2 修复。
5. 复拍截图和关键路径录屏。

## 4. 验收

- 人工体验量表高分。
- 遮掉文案仍能区分场景。
- 关键路径录屏可解释。
- 无 P0/P1 视觉缺陷。
