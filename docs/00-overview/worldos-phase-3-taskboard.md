# 阶段 3：内容生命系统任务板

**目标：** 从展示内容，升级为内容可生长、可归档、可被重新发现。

## 批次 3.1：内容模型收束

**目标：**
- [ ] 明确内容类型：文章、项目、笔记、生活记录、作品、日志、路径节点。
- [ ] 每个内容必须具备：标题、摘要、公开性、区域、关系、生命周期、推荐路径。
- [ ] 建立内容质量门槛：无摘要、无关系、无区域、无正文的内容不得进入精选。

**改动范围（执行留痕）：**
- [ ] 数据层：在 schema 层（如 Zod 校验）收紧字段验证规则
- [ ] 页面层：处理残缺内容的降级展示
- [ ] 组件层：内容信息卡的元数据呈现优化
- [ ] 权限边界：无
- [ ] 检查脚本：强化数据合法性验证 `validate:world`

**不做：**
- 暂不开发图形化在线编辑器。

**验收命令：**
```bash
npm run check:content
npm run validate:world
```

## 批次 3.2：真实内容补齐

**目标：**
- [ ] 选择优先补齐的 20 到 30 个真实节点（替换填充物假数据）。
- [ ] 确保每个一级区域至少有代表节点。
- [ ] 确保每条核心路径至少有足够的内容节点形成闭环。
- [ ] 确保时间线事件与节点、区域之间互相关联。

**改动范围（执行留痕）：**
- [ ] 数据层：新增和替换 `data` / `content` 目录下的 JSON / Markdown 数据
- [ ] 页面层：无
- [ ] 组件层：无
- [ ] 权限边界：新增真实内容的权限属性验证
- [ ] 检查脚本：执行内容密度门禁

**不做：**
- 不随意上传庞大的原始视音频资产，仅使用经优化的图片和文本。

**验收命令：**
```bash
npm run check:worldos-content-density
npm run check:experience:public
```

## 批次 3.3：检索与再发现

**目标：**
- [ ] 档案馆支持稳定搜索、筛选、主题入口。
- [ ] 节点页支持“相关节点”、“路径返回”、“下一步推荐”。
- [ ] 首页和地图呈现“今天可以从哪里开始”的心智。

**改动范围（执行留痕）：**
- [ ] 数据层：加强基于反向链接（Backlinks）与标签的聚合逻辑
- [ ] 页面层：完善 Archive 搜索交互
- [ ] 组件层：设计并接入“相关推荐”模块
- [ ] 权限边界：无
- [ ] 检查脚本：跑通归档与首页浏览测试

**不做：**
- 不做基于大模型的动态 Embedding 全文检索（保持静态构建优先）。

**验收命令：**
```bash
npm run check:content-archive
npm run check:node-reading
npm run check:homepage
```
