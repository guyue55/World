# WorldOS 生命世界体验验收契约

> [!IMPORTANT]
> 本文档只回答“用户实际看见、听见、操作和感受到什么”。代码存在、动画注册、截图文件、报告 `passed` 和旧完成结论都不能替代本契约。

机器可执行的固定集合、采样数、状态阶梯和否决项同时冻结在 `data/domains/experience/living-world-acceptance.json`。两者冲突时必须暂停并由用户在 Goal 外修订控制包，不得由执行者选择较低标准。

## 1. 当前基线

截至 2026-07-11，真实基线是：

- 七场景拥有不同的月下电影感静态位图，文字隐藏后大体可以凭图片辨认空间。
- 交互主要是位图上的热点、标签、drawer、短入场和局部 GSAP 反馈。
- Timeline 画面里有河，但没有 60 至 120 秒证据证明河持续流动。
- Atlas 有星图构图，但未证明内容状态、关系与真实时间持续驱动星体生命。
- Lighthouse 有循环扫光，是现有最接近持续环境的样板，但未纳入统一生命周期。
- 最新终审录屏长度约 1.72 至 10.16 秒，只能证明短流程，不证明生命运行。
- `WorldRuntimeProvider` 首次挂载后不继续更新时间和季节。
- public 目录没有真实音频文件，程序化音色没有完成真实长期试听。

基线截图与录屏位于：

```text
docs/90-archive/reports/worldos-reality-first/run-2026-07-11_07-28-49-583Z/
```

任何新完成结论必须明显超越这组基线，而不是换一张背景或增加热点。

## 2. 用户结果

候选完成时，访问者无需阅读工程说明即可在前 30 秒形成四个判断：

1. 这是一个属于某个人、由真实内容构成的世界。
2. 我知道自己在哪里、能去哪里、每个方向为什么值得探索。
3. 这个世界在我停留时仍持续运行，变化有时间、内容、关系或旅程依据。
4. 我可以继续探索，也可以安静阅读、关闭声音和减少运动。

候选只能证明体验具备形成长期价值的条件，不能证明真实用户已经愿意数月回访。

## 3. 双向验收方法

每项体验同时从两个方向验证：

```text
设计方向：期望感受 -> 可观察动态 -> 最小机制
审查方向：真实页面 -> 真实动态 -> 用户感受
```

若机制存在但观察不到，体验失败。若视觉热闹但变化无事实来源，真实性失败。

## 4. 场景必须通过的九种视图

每个核心场景都必须有最新生产态证据：

| 视图 | 目的 | 失败示例 |
| --- | --- | --- |
| desktop | 主构图与鼠标 / 键盘 | 场景主体退化为卡片区 |
| mobile | 独立 framing 与触控 | desktop 缩小、固定控件遮挡 |
| text-hidden | 检查空间人格 | 七页只剩相同背景布局 |
| background-hidden | 检查位图依赖 | 关掉 WebP 后只剩按钮和空层 |
| quiet-static | 检查静谧表达 | 停止运动后状态和方向消失 |
| reduced-motion | 检查系统偏好 | 主要路径不可操作 |
| reduced-sensory | 检查无声与低刺激 | 信息只靠声音表达 |
| JavaScript-off | 检查静态世界 | 白屏、空 Canvas、无正文或出口 |
| resource-failure | 检查位图 / 音频 / API 失败 | 开发术语、破图、无返回路径 |

`background-hidden` 必须隐藏 `picture`、CSS `background-image`、纯装饰 bitmap / SVG / Canvas 与其他非语义美术，且不得换另一张静态图补位。它验证场景的语义对象、几何关系、主要操作和主生命行为没有被烘焙进资产；语义 SVG / Canvas 可以保留，但必须有 DOM 等价入口。

## 5. 统一世界语法

### 5.1 世界观

统一母题保持为“月下浮屿中的个人观测与记忆系统”。它应具有：

- 自然与人工并存：岛屿、河流、雾、树林、房间、道路、档案、灯塔。
- 克制电影感：真实空间层次、局部暖光、广阔冷色环境，不是紫色霓虹科技模板。
- 中文可读性：标题和操作清楚，氛围词不替代命令。
- 真实内容痕迹：世界里的光点、门、卷宗、停靠点和道路都对应事实。

### 5.2 场景五层

每个动态场景至少由以下层组成，不能把五层烘焙成一张图：

1. **空间底层**：可静态加载的地形、建筑、房间或材质。
2. **语义对象层**：区域、节点、事件、卷宗、站点、关系门或灯塔方向。
3. **持续环境层**：河面、星光、雾、窗光、尘、道路流向等可暂停生命。
4. **交互与迁移层**：聚焦、拖动替代、路径、离开对象和抵达对象。
5. **内容与操作层**：正文、摘要、搜索、问路、清除和明确出口。

静态世界保留 1、2、5；动态世界增强 3、4；静谧世界保留 3 的状态但停止不必要位移。

### 5.3 活动度

| 活动度 | 场景 / 状态 | 允许表现 |
| --- | --- | --- |
| 高 | Gateway、Atlas 聚焦、有限迁移 | 多层低速运动，可暂停 |
| 中 | Timeline、Paths、Lighthouse | 一个主运动和少量环境响应 |
| 低 | Archive、Node 阅读 | 稀疏光线、材质或窗外远景 |
| 静 | reduced-motion、页面隐藏、错误、打印 | 无位移运动，保留状态与语义 |

持续生命不等于每页都飘粒子。

## 6. Gateway

**体验身份：** 世界入口和观测门厅，不是营销首页。

**必须看见：** 可进入的世界远景、当前天光、至少三个有空间方向的去处、远处灯塔和一个明确主动作。

**持续生命：**

- 入场结束后，星点、雾、浮屿或灯塔至少有两类低活动度变化。
- 天光随 `dayProgress` 缓慢插值，季节改变雾、植被 / 漂浮物或声音参数，不只改 hue。
- 新内容或未完成旅程只在对应方向出现一次有因果的提示。

**交互：** 进入、观察方向、选择空间、继续上次旅程、清除访问记忆、问灯塔。

**首访：** 3 至 6 秒内完成可跳过抵达；声音不自动开启；动作后立即进入探索。

**回访：** 同日再访优先继续最近地点 / 路径；长期离开显示真实变化摘要，不重播完整仪式。

**否决：** 大标题占主视口、四个 CTA、统计卡、静态远景加三个普通按钮、每次访问完全相同。

## 7. Atlas

**体验身份：** 内容地理和关系航行空间。

**主体：** 可平移 / 聚焦的区域星域、节点星体、关系航线和可见远近层次。

**持续生命：**

- 星体亮度由内容生命周期、新鲜度和访问状态投影。
- 关系能量只沿当前语义子图缓慢传递，不让全部线闪烁。
- 时间与季节改变区域材质、天光和活性，不改变事实布局。
- 离开焦点区域后降低活动度；离场后停止所有 adapter。

**交互：** 平移、缩放、区域聚焦、节点预览、关系理由、搜索、进入 Node / Timeline。

**等价输入：** 拖动之外提供单击、键盘、方向控件和可访问列表。

**规模：** 首屏只显示语义子图，不能一次绘制全部标签与关系。

**否决：** 背景图承担全部星域；节点只是绝对定位按钮；所有星点随机呼吸；关闭背景后没有地图结构。

## 8. Timeline

**体验身份：** 可停靠、可回看、确实沿时间方向流动的河。

**主体：** 河道、河段、时间锚、事件停靠点、岸边内容地点。

**持续生命：**

- 入场结束后河面纹理持续沿固定时间方向流动。
- 事件只有聚焦或真实状态变化时产生局部涟漪。
- 内容密度可改变河段宽度 / 岸边活动，但不能改事件日期。
- 四季改变水色、雾、漂浮物和岸边状态；昼夜改变反射与灯光。

**交互：** 顺流浏览、跳转时间锚、停靠事件、进入 Node、返回原河段。

**等价输入：** 滚动、锚点、键盘和事件列表能定位同一事件。

**风险原型：** SVG 合成层与 Canvas 2D 以 mobile p95 frame、可访问替代和实现复杂度比较；原型失败先简化，不直接引入 PixiJS。

**否决：** 河流只是静态背景；事件卡片放在河上；动画只在进入或点击时发生；录屏看不出流向。

## 9. Archive

**体验身份：** 可检索、可抽取、可返回原架位的记忆与资料空间。

**主体：** 馆厅、分区、书架 / 抽屉、卷宗和索引台。

**持续生命：**

- 默认活动度低：稀疏窗光、尘、纸张或柜体材质缓慢变化。
- 搜索与筛选使馆内分区、光线和卷宗位置产生可解释重排。
- 新增 / 更新内容进入“新入馆”投影，访问后按规则回到常规分区。

**交互：** 搜索、筛选、分区、抽卷、预览、进入 Node、返回原查询和架位。

**否决：** 搜索栏下方仍是卡片瀑布；书架只是背景线；持续动画妨碍输入、扫描和长列表。

## 10. Paths

**体验身份：** 可以开始、继续、停留和抵达的旅程。

**主体：** 道路、站点、路标、脚印、岔路、当前位置和终点。

**持续生命：**

- 未完成道路有低速前向流动；完成段落沉静并留下本地印记。
- 下一站只由真实进度点亮，不随机推荐。
- 季节改变道路环境和声景，不改路径顺序。

**交互：** 选择路线、开始、继续、回到已走站、进入 Node、完成、重置和返回 Atlas。

**完成感：** 抵达后世界出现与该路径相关的真实变化或下一条路线，不只显示 `7/7`。

**否决：** 路线图只是背景；核心操作是步骤列表；进度只能从数字文字得知。

## 11. Node

**体验身份：** 可停留阅读、拥有区域物件和关系出口的内容地点。

**主体：** 与内容类型相符的地点、舒适正文、轻量护照、关系门、来源与出口。

**持续生命：**

- 正文附近保持静止；只允许窗光、远景或稀疏一次性细节。
- 生命周期通过房间材质、光线、护照和关系门共同表达。
- 昼夜只改变外部窗光 / 环境，不把纸白正文强行变暗或反色。

**交互：** 阅读、查看来源、穿过有理由的关系门、回来源、回 Path、看 Timeline / Archive、问灯塔。

**阅读：** 正文宽度约 60 至 72ch；200% zoom 和 mobile 不被固定控件遮挡；代码、图片、长文和空内容稳定。

**否决：** 正文一直运动；护照大侧栏压缩正文；地点只是文章页上方一张图；读完没有自然出口。

## 12. Lighthouse

**体验身份：** 知道当前来路、照亮下一站、承认边界的陪伴型观测站。

**主体：** 塔、海面、扫光、观测方向、当前来源和问路入口。

**持续生命：**

- 光束稳定扫过可达方向；Provider thinking 只改变局部灯光节奏。
- day / night 与四季改变海雾、天空、灯塔强度和声景。
- Gateway 与 Lighthouse 共享一个可辨认但不过度重复的音乐动机变奏。

**能力：** 回答“我在哪里、为什么相关、下一步看什么、如何回去”，返回公开来源、下一站和不确定性。

**low-light：** 明确说明未连接实时模型，使用确定性检索与关系解释；不能冒充 AI 生成。

**Provider：** 当前固定为服务端私有 LAN Ollama `qwen2.5:7b`。必须先预热，再通过正常问路、来源约束、未知拒答、私密拒答、超时、schema 错误和限流评测；URL/Key 不进入客户端。冷启动实测可接近 30 秒，不能用普通热请求超时误杀，正常交互仍必须在 12 秒预算内完成或诚实回退。

**否决：** 全屏聊天框；灯塔只是插画；建议与当前位置无关；无来源长文；低光结果冒充模型。

## 13. 世界时间与季节

共享快照至少包含：

```ts
type WorldTimeSnapshot = {
  nowEpochMs: number
  timeZone: string
  dayProgress: number
  dayPeriod: 'dawn' | 'day' | 'dusk' | 'night'
  season: 'spring' | 'summer' | 'autumn' | 'winter'
  seasonProgress: number
  worldDateKey: string
}
```

验收要求：

- 逻辑快照至少每分钟和页面恢复时更新；视觉可在快照之间插值。
- hidden 时停止环境绘制，visible 时从真实时间重新派生，不补跑帧。
- 七场景消费同一时刻，但以各自自然规律表达。
- Gateway 与 Node 覆盖 16 个昼夜 / 四季组合；其他场景至少人工审查四个代表组合。
- 每组时间 / 季节结论必须有成对证据：固定场景、内容、viewport、worldDateKey 与质量档，只改变时间或季节；变化必须属于场景自身，不能来自随机噪声或全局滤镜。
- `/status` 可以注入测试时间；公开路由不得出现测试控件。
- 只使用全站滤镜、状态标签或换背景图表达四季即失败。

第一版不要求天气、节气、天文服务或复杂农历。

## 14. 内容生命

节点生命周期只能来自事实规则或作者确认：

| 状态 | 事实含义 | 可见投影 |
| --- | --- | --- |
| seed | 新建、内容 / 关系较少 | 微弱新光、等待连接 |
| growing | 正文、关系或路径持续补充 | 门和航线更清晰 |
| active | 近期更新并被事件 / 路径使用 | 稳定明亮、多个入口 |
| settled | 长期稳定且关系完整 | 沉静材质与档案感 |
| archived | 作者明确归档 | Archive 主投影，仍可追溯 |

一个真实公开节点的新增 / 更新演练必须无需改场景组件，并自动影响适用的：

- Atlas 星体与关系。
- Timeline 事件或“无时间事件”说明。
- Archive 分区、搜索和新入馆状态。
- Paths 站点或“非旅程节点”说明。
- Node 房间与关系门。
- Lighthouse 公开上下文和推荐。
- 导出 manifest、checksum 和恢复。

至少抽样 24 个来源可追溯节点，覆盖 6 个区域及项目、记忆、思考、创作、规则等内容气质。不得为凑数伪造作者经历。

## 15. 迁移语法

每次主要导航包含：

1. **离开**：来源对象响应并保留几何、意图和返回信息。
2. **途中**：介质表达方向，如星域、河面、卷宗、道路、关系门或光束。
3. **抵达**：目标对象先被识别，目标环境随后接管。
4. **沉淀**：焦点、滚动、路径、查询和返回上下文稳定。

固定路线族：

| 路线 | 迁移对象 | 必须延续 |
| --- | --- | --- |
| Gateway -> Atlas | 远处浮屿 / 星域 | 目标区域 |
| Atlas -> Node | 星体 -> 地点物件 | 节点、来源区域 |
| Timeline -> Node | 涟漪 -> 上岸点 | 事件、河段 |
| Archive -> Node | 卷宗 -> 房间 | 查询、筛选、架位 |
| Paths -> Node | 站点 -> 地点门 | 路径、进度 |
| Node -> Node | 关系门 | 关系理由、返回门 |
| 任意 -> Lighthouse | 光束 | 来源、当前位置、问题 |
| 任意 -> Gateway | 拉远回到世界 | 最近地点、继续入口 |

要求：

- 不使用统一全屏 fade 换颜色冒充所有路线。
- 快速连点、后退、刷新、深链、404、资源失败均不留下遮罩和不可点击层。
- reduced-motion 在 0 至 160ms 内以状态和焦点表达抵达，不播放大位移。
- 30 次主要迁移后 active adapter、ticker、listener、animation 和 audio source 回到稳定基线。
- 必须从真实 route / link 生成全部跨场景边清单，逐项标记迁移协议或合法 bypass；未覆盖边数量必须为 0，不能只抽测上表八个示例。

## 16. 声音与音乐

### 16.1 功能

- 首次加载和每次刷新默认静音，用户启用前音频下载为 0 B。
- 只有用户手势后创建 / resume `AudioContext`。
- 同时最多一个 ambience 和一个短 cue；迁移 crossfade 后释放旧 source。
- hidden、mute、quiet、错误和卸载立即 suspend / dispose。
- 声音关闭后所有状态仍有视觉和文本表达。

### 16.2 技术验证与人类听感

- 世界先用一个 60 至 120 秒可循环样板验证，不直接制作七套音频。
- 自动技术验证必须覆盖：用户手势前 0 B、单一 `AudioContext`、同时最多一个 ambience 与一个 cue、crossfade 释放旧 source、hidden / mute / quiet / dispose、true peak、循环接缝、十分钟离线渲染、波形与频谱异常。
- Codex 不能把频率表、peakGain、离线渲染或自己的文字判断冒充真实听感。
- 若有真实人类签收，必须用耳机和扬声器各试听至少 10 分钟，记录设备、开始 / 结束时间、场景、疲劳、刺耳、增益跳变、阅读干扰和结论。
- 技术验证和 Ollama Provider 验收通过可以结束本 Goal，但状态始终只能是 `LOCAL_LIVING_WORLD_CANDIDATE_AI_PROVIDER_HUMAN_AUDIO_PENDING`；自报人类记录不触发自动晋级。
- 七场景最终应可辨但共享材质与动机；Node / Archive 最稀疏。

### 16.3 资产

每个文件或 patch 记录来源 / 生成方式、作者、许可、URL、下载日期、hash、大小 / 时长、峰值、用途、加载策略、技术验证与人类签收状态。来源不明或许可不清即 blocked。

## 17. 回访与世界记忆

访问记忆只保存公开、最小、可清除信息：最近 route、公开对象 ID、路径进度、已读提示、声音 / 动效偏好和时间。

必须验证：

- 首访、同会话、同日、近期和长期离开投影不同。
- 返回摘要只报告真实内容更新和未完成旅程，不制造假事件。
- 一键清除后恢复首访语义，不白屏、不留敏感副本。
- storage disabled、解析失败、quota error 时世界继续可用。
- 不持久化问题全文、AI 回答、正文、私密 slug、owner 身份和 Provider 信息。

## 19. 性能与长时运行

| 项 | 候选预算 |
| --- | --- |
| shared First Load JS | <= 130 KB |
| 单场景普通新增 route JS | gzip <= 80 KB |
| lazy visualization | gzip <= 150 KB，需有可见收益 |
| desktop / mobile 首屏 bitmap | <= 700 KB / 350 KB |
| 用户开启前音频下载 | 0 B |
| ambient | 目标约 30fps；p95 frame <= 34ms |
| 交互 / 迁移 LoAF | 不得 > 100ms；> 50ms 必须审查 |
| INP / LCP / CLS | <= 200ms / 2.5s / 0.1 |
| hidden | 无 ambient tick，AudioContext suspended |
| 10 分钟样板 | 无持续 heap、listener、adapter、source 增长 |

最低测量：开发 Mac Chrome、mobile touch emulation、CPU 4x slowdown、localhost 和 LAN IP。

固定采样：每个最终性能 run 至少 3 次 cold start、1 次 warmup 后 5 次 measured warm；ambient 至少采 300 帧，并分别记录 desktop / mobile、normal / CPU 4x / LAN 条件，不能只保留最好一次。

## 20. 可访问与失败边界

- Canvas / SVG 有 DOM 等价入口和可访问名称。
- 拖动有点击、键盘和控件替代；颜色和声音不是唯一状态。
- 持续自动运动有暂停 / 静谧控制，尊重系统 reduced-motion。
- 迁移后焦点落到目标主体或合理标题；后退恢复来源焦点。
- 文本放大 200% 无遮挡，mobile 触控目标和固定安全区稳定。
- bitmap、audio、storage、Provider、public API 失败不白屏、不泄漏、不显示工程术语。
- Node 正文、关系、路径顺序、事件、搜索入口和低光导览在 JS-off 仍可达。

## 21. 证据与独立审查

每个最终 run 必须记录：commit、源码 / 数据 / 资产时间、build、server、浏览器版本、URL、viewport、motion、sensory、time、season、AI mode、步骤、期望、截图 / 录屏 / trace / 音频 hash 和观察缺陷。每张九模式场景截图都必须有独立 sidecar，绑定 scene、view、sourceCommit、buildId、origin、viewport、mode、server PID / start time、capture time、command、nonce 与图片 hash，不能只靠文件 mtime。录屏必须连续、未剪辑、非重复拼接，并记录 wall-clock、单调时钟、`ffprobe` 时长、capture command、server-bound sidecar 与 checksum；F1-F14 的 trace summary 和录屏也必须绑定同一 build/server。

最低证据：

- 七场景每个一段 60 至 120 秒静置录屏。
- Gateway-Atlas-Node-Gateway 十分钟 soak。
- F1-F14 连续流程录屏或 trace。
- desktop、mobile、text-hidden、background-hidden、quiet-static contact sheet。
- 四季 / 昼夜成对因果证据。
- 音频技术验证包；有真实人类参与时附耳机与扬声器签收记录，没有时保留状态后缀。
- 权限、AI、资产、导出恢复、性能、资源计数和 freshness 报告。

审查顺序：

1. 实现者先看页面、长录屏和音频技术记录；真实人类签收存在时一并查看，登记缺陷并修复。
2. 两个不同 fresh reviewer context 以随机场景顺序，先看隐藏标题、导航、标签与报告状态的截图、background-hidden 集合、连续录屏和音频技术 / 人类签收记录，再看报告和源码。
3. 工程与权限审查。
4. 最终修复后再做一次完整视觉 / 交互 / soak 审查。

同一实现者的主观结论不能成为唯一完成证据。

最终 evidence manifest 只是索引，不拥有 `pass` 权力。终局校验器必须读取并回算带 SHA-256 的原始 flow trace、时间成对截图、迁移边清单、音频分析、灯塔 case、权限扫描、public export checksum / 临时恢复、性能样本、复杂度指标、资产许可与盲审包；裸 `verdict: pass`、`openP0: 0` 或 `vetoesTriggered: []` 均无效。审查正文必须来自两个独立只读 task 的独立文件，主执行器只引用路径与 hash；本地文件只能证明程序性独立，不能声称拥有平台级密码学身份认证。

为防止“带 hash 的自报 JSON”，冻结终局校验器还必须亲自完成独立抽查：以本机 Playwright / Chromium 在 attested origin 上执行 CPU 4x 帧采样和 live migration；现场重放十个 Lighthouse case；启动第二个真实 `next start` 注入固定 private canary；crawl hydration 后导航；读取 PID、cwd、端口、LAN 网卡、完整 `.next` Merkle、live chunk、文件 mtime 和临时 restore。流水线全量证据与冻结抽查任一失败均不能完成。

## 22. 固定否决项

任一项存在，整体未完成：

- 任一场景仍主要是静态大图 + 热点 + 短入场。
- background-hidden 后场景主体消失或只剩普通按钮列表。
- 进入动画结束后没有持续生命，或 60 秒录屏看不出自然推进。
- 河流不流、星图不活、灯塔不扫，或所有场景使用同一种粒子背景。
- 日夜 / 四季只是全站滤镜、状态标签或 16 张整图替换。
- 主要导航绕过迁移协议，返回和深链丢失上下文。
- 内容更新需要修改场景组件或复制事实。
- Node / Archive 的运动、声音或灯光干扰阅读。
- 声音自动播放、无法立即停止、技术验证失败、人类签收伪造或许可不清。
- AI 无来源、伪造 Provider、越权、写事实或 Key 进入客户端。
- 私密事实进入公开载荷、记忆、Canvas、证据或本 Goal 的 public export。
- JS-off、reduced-motion、reduced-sensory、mobile 或资源失败丢失主路径。
- 30 次迁移或十分钟运行后资源持续增长。
- 新依赖、文档、脚本、registry 或万能抽象明显膨胀且无删除路径。
- 仅靠脚本、分数、DOM token、截图存在或报告自报完成。

## 23. 允许的完成结论

自动工程、浏览器与独立审查全部通过，但尚无真实人类音频签收时，只能说明：

```text
LOCAL_LIVING_WORLD_CANDIDATE_AI_PROVIDER_HUMAN_AUDIO_PENDING
```

真实人类完成耳机与扬声器签收后，用户可在 Goal 外复核并批准新的控制版本，才可能晋级为：

```text
LOCAL_LIVING_WORLD_CANDIDATE_AI_PROVIDER
```

若 Ollama 在执行期间无法通过固定评测，只能据实记录下列 fallback-pending 状态并继续修复或按外部阻断规则暂停，不能把它当作本 Goal 的替代完成态：

```text
LOCAL_LIVING_WORLD_CANDIDATE_AI_FALLBACK_HUMAN_AUDIO_PENDING
```

真实用户长期回访、完整私密世界、家庭 / 继承流程和多年保存不在本次候选声明内。
