# 阶段 7：运营与长期演化任务板

**目标：** 世界不是一次性网站，而是可持续更新的个人系统。

## 批次 7.1：内容运营节奏

**目标：**
- [x] 制定内容日历：每周补哪些节点、路径、档案。
- [x] 建立主题展览：把离散内容组织成专题。
- [x] 关注观测指标：访问量、阅读时长、路径完成率、搜索无结果情况。

**改动范围（执行留痕）：**
- [x] 数据层：新增运营计划与监控数据占位符
- [x] 页面层：专题展览模板页面
- [x] 组件层：埋点与数据收集桩代码
- [x] 权限边界：无新增私密能力
- [x] 检查脚本：运营更新频次检查（如果有自动化脚手架）

**不做：**
- 暂不接入重型的大数据商业埋点 SDK。

**验收命令：**
```bash
npm run check:content-growth
npm run check:theme-exhibitions
```

## 批次 7.2：年度归档与导出

**目标：**
- [x] 实现年度世界册：把一年的内容导出成可保存版本。
- [x] 建立资产库：将图片、封面、插图、多媒体素材规范归档。
- [x] 确立长期治理机制：包括废弃陈旧内容、合并冗余节点、更新路径逻辑。

**改动范围（执行留痕）：**
- [x] 数据层：资产目录梳理与迁移
- [x] 页面层：导出中心视图
- [x] 组件层：无新增组件，沿用导出中心现有视图能力
- [x] 权限边界：控制导出功能的访客与所有者权限
- [x] 检查脚本：资产有效性检查

**不做：**
- 不做基于数据库的在线全量 PDF 即时生成（先依赖静态方案）。

**验收命令：**
```bash
npm run check:export-center
npm run check:operations-export-planning
```
