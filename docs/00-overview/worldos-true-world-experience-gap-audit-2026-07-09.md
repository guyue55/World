# WorldOS 真格世界体验差距审计

> [!IMPORTANT]
> 本文档用于纠正一个关键误判：M0-M7 已经让 WorldOS 具备本地运行闭环和可信验收，但还没有达到“真格世界 / 宇宙体验”。后续 M8-M12 必须以用户直观看到的世界感为第一目标。

## 1. 结论

当前状态不是“目标完成”，而是：

**可信底座基本完成，世界体验落地不足。**

已经成立的能力：

- 本地 / LAN 可运行。
- 公开内容事实源可被场景和灯塔读取。
- Scene QA、LAN smoke、RC summary、audit 有证据。
- 转场、场景、运行时、灯塔、内容生命已有统一底座。

尚未成立的体验：

- 第一眼仍容易像产品页 / 博客页。
- 场景人格偏文本说明，空间感不足。
- 动态效果偏轻，缺少“进入、穿越、抵达、回看”的强感知。
- Atlas / Timeline / Archive / Paths 仍有卡片、列表、面板感。
- Node 仍偏文章详情，不像抵达地点。
- 灯塔仍偏静态推荐区，不像可感知的观测站。

## 2. 根因

| 根因 | 表现 | 后果 |
| --- | --- | --- |
| 工程验收强，体验验收弱 | DOM、报告、截图通过 | 用户仍觉得像骨架 |
| 场景语义强，舞台表达弱 | 有 sceneId、portal、identity | 空间隐喻不明显 |
| 动态过度克制 | 避免遮挡和性能风险 | 世界活性不足 |
| 内容事实源可用但呈现浅 | 200 节点可索引 | 阅读仍像列表和文章 |
| 转场状态可观测但叙事弱 | 有 cue 和状态 | 不像真正迁移 |

## 3. 差距清单

| 模块 | 当前状态 | 目标状态 | 缺口 |
| --- | --- | --- | --- |
| Home | 世界入口语义成立 | 像进入一个可探索世界 | 需要主舞台、入口仪式、空间纵深 |
| Atlas | 有区域、节点、关系 | 像地图 / 星图 | 需要可探索地图视图、区域聚焦、关系路径 |
| Timeline | 有事件和时间流组件 | 像时间河 | 需要河道、年份层、事件漂移、回看感 |
| Archive | 有检索和标签 | 像档案馆 / 记忆库 | 需要分区、抽屉、卷宗、密度与沉淀感 |
| Paths | 有路径和进度 | 像旅程系统 | 需要行进、下一站、抵达、完成状态 |
| Node | 有关系、下一步 | 像抵达地点 | 需要地点开场、阅读房间、关系门、出口 |
| Transition | 有状态和 cue | 像场景迁移 | 需要来源残影、目标预告、抵达确认 |
| Lighthouse | 有低光只读 | 像观测站 / 导览者 | 需要问路仪式、扫描、推荐依据和边界可视化 |

## 4. 不再接受的完成口径

以下证据只能证明“底座可用”，不能单独证明“真格世界体验达标”：

- 只有 `release:local-rc` 通过。
- 只有 Scene QA DOM 信号通过。
- 只有截图存在。
- 只有“场景人格”文案。
- 只有轻量 fade / scale 入场。

## 5. 新完成口径

M8-M12 完成必须同时满足：

1. 自动门禁通过。
2. 浏览器截图无白屏、无遮挡、无溢出。
3. 人工体验量表通过。
4. 每个阶段都有“是否还像骨架”的明确结论。
5. 场景表现层确实减少卡片 / 列表 / Hero 页面感。
6. reduced-motion 和 mobile 仍然可读。
7. 不引入重型依赖，不默认音频，不做外部上线。

## 6. 后续阶段

- M8：世界入口与主舞台重塑。
- M9：Atlas / Timeline / Archive / Paths 四大场景舞台化。
- M10：Node 地点化与内容沉浸阅读。
- M11：场景迁移叙事化。
- M12：灯塔导览体验可感知化。
