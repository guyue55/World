# WorldOS 音频 / 音乐治理规格

## 1. 目标

声音是可选氛围增强，不是默认负担。音频必须可静音、可降级、可授权、可按场景加载。

## 2. 播放原则

- 默认静音，不自动播放。
- 用户 opt-in 后才加载音频。
- 提供全局静音与音量控制。
- 记住用户选择。
- reduced-sensory 关闭声音。

## 3. 资产要求

- 所有音频必须有来源和授权记录。
- 优先短音效和轻环境音。
- 背景音乐不得影响阅读。
- 音频文件按场景懒加载。

## 4. 技术路线

1. 原生 HTMLAudio / Web Audio API 试点。
2. 管理复杂循环或多格式时评估 Howler.js。
3. 生成式声景明确必要时再评估 Tone.js。

## 5. 验收

- 首次访问无声音。
- 用户能一键关闭。
- mobile 不因音频增加首屏负担。
- 关闭音频不影响核心体验。
