# WorldOS M8 世界入口与主舞台重塑执行计划

## 1. 目标

首页必须从“产品首页 / 博客首页”转为“进入世界的主舞台”。访问者第一眼要知道：这里是古月浮屿，一座可以探索的个人数字世界。

## 2. 核心问题

- 首屏是否还像 Hero + CTA + 卡片。
- 世界入口是否有空间纵深、入口仪式和可探索方向。
- 动态是否服务“进入世界”，而不是背景装饰。

## 3. 执行项

- [ ] 审计当前首页 desktop / mobile / reduced-motion 截图。
- [ ] 重构首页首屏为 `WorldGatewayStage` 或等价主舞台模块。
- [ ] 将入口动作分成：进入地图、沿路径、查档案、问灯塔。
- [ ] 加强空间层：前景入口、中景路线、远景世界状态。
- [ ] 首访仪式只负责唤醒世界，不遮挡正文，不超过 2.5 秒。
- [ ] 保留静态 fallback：无 JS 时仍能看到入口和四个方向。
- [ ] 保持 `SceneWorldPortal` / runtime / content facts 复用，不重复造数据。

## 4. 验收标准

- [ ] Home 人工体验量表“世界入口”得 2 分。
- [ ] 首屏不能像普通博客 Hero。
- [ ] 主 CTA、移动导航、核心状态卡仍可见。
- [ ] reduced-motion 不空白、不遮挡。
- [ ] `npm run check:homepage` 通过。
- [ ] `npm run check:scene-qa` 通过。
- [ ] `npm run release:local-rc` 通过。

## 5. 边界

- 不做外部上线。
- 不引入 3D / WebGL。
- 不默认音频。
- 不新增第二套首页数据源。
