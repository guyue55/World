# 阶段 5：AI 灯塔与辅助系统任务板

**目标：** AI 不是噱头，而是帮助用户理解、检索、整理世界。

## 批次 5.1：公开灯塔

**目标：**
- [ ] `/ask` 页面明确定位：仅用于公开问答、路径推荐、内容导航。
- [ ] AI 必须且只能基于公开上下文和协议回答。
- [ ] 建立风险提示：遇到不确定、不可回答、需人工确认的问题要有预案。
- [ ] 保持“降低门槛、中文优先”的心智。

**改动范围（执行留痕）：**
- [ ] 数据层：梳理投喂给 AI 的公开数据协议结构
- [ ] 页面层：开发或完善 `/ask` 交互流
- [ ] 组件层：对话流与系统反馈卡片
- [ ] 权限边界：限制后端查询大模型时允许下发的上下文文件范围
- [ ] 检查脚本：AI 安全边界检查

**不做：**
- 不做可全权代表本人的克隆型对话机器人。

**验收命令：**
```bash
npm run check:ai-boundary
npm run check:lighthouse
```

## 批次 5.2：Owner 模式规划

**目标：**
- [ ] 建立内部建议审核队列：AI 可建议，但不能自动改动世界源文件。
- [ ] 为后续扩展 owner 模式预留基础（如整理草稿、生成路径建议、内容归档建议）。

**改动范围（执行留痕）：**
- [ ] 数据层：设计审核队列状态（Pending / Accepted / Rejected）
- [ ] 页面层：后台审批面板原型结构
- [ ] 组件层：无
- [ ] 权限边界：限定此功能仅 Owner 可见
- [ ] 检查脚本：后台功能专属检测

**不做：**
- 不让 AI 拥有写文件或提交 Git 的直接权限。

**验收命令：**
```bash
npm run check:ai-suggestion-audit
npm run check:ai-world-companion
```
