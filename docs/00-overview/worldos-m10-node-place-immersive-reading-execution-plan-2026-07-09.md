# WorldOS M10 Node 地点化与内容沉浸阅读执行计划

## 1. 目标

Node 不再只是文章详情，而是世界中的一个地点。读者进入节点时，要感到自己抵达了某个房间、星体或档案位置。

## 2. 核心体验

- 进入前知道来源。
- 进入时知道地点、区域、生命阶段。
- 阅读时正文舒适，不被动效干扰。
- 读完后能从关系门、路径门、时间门继续走。

## 3. 执行项

- [ ] 审计当前 Node 页面是否仍像文章详情。
- [ ] 强化 `NodeOpeningRitual` 为地点开场，而非装饰标题。
- [ ] 建立节点“地点护照”：区域、生命周期、路径、关系、时间锚点。
- [ ] 将关系 rail 改成“出口门”语义：同区域、强关系、路径下一步、时间回看。
- [ ] 正文保留阅读舒适度，不把内容藏在动画后。
- [ ] 将内容生命 facts 接入节点页面证据。
- [ ] 为无关系、无路径、无摘要的节点设置不能进入精选的门禁。

## 4. 验收标准

- [ ] Node 人工体验量表“内容生命”和“可探索性”不低于 2。
- [ ] `npm run check:node-reading` 通过。
- [ ] `npm run check:content-life` 通过。
- [ ] `npm run check:scene-qa` 覆盖 node route。
- [ ] `npm run release:local-rc` 通过。

## 5. 边界

- 不牺牲正文可读性。
- 不把私密关系或 vault 信息带入公开节点。
- 不把 AI 生成内容伪装成人工事实。
