# WorldOS M25 作者世界编辑台产品规格

> [!IMPORTANT]
> M25 的目标是让作者能持续养世界，而不是每次更新都靠改代码。

## 1. 目标

作者应能用中文低门槛维护：

- 节点。
- 区域。
- 关系。
- 路径。
- 时间事件。
- 资产元数据。

对应支柱：S3 内容生命体、S7 作者共生。

## 2. 编辑台范围

| 模块 | 能力 | 首版形式 |
| --- | --- | --- |
| 节点编辑 | 标题、摘要、正文、区域、生命周期 | 本地表单 / 文件写入待评估 |
| 关系编辑 | 关联节点、关系原因 | 可视表单 |
| 路径编辑 | 路线、步骤、下一站 | 路径清单 |
| 时间编辑 | 事件、日期、关联节点 | 时间事件表 |
| 资产编辑 | 来源、授权、用途、体积 | manifest 表单 |
| 预览校验 | 影响范围、权限、缺字段 | dry-run 报告 |

## 3. 边界

- 不先做云端 CMS。
- 不绕过内容契约。
- 不让前端直接写私密数据。
- 任何生成内容必须能审计和回滚。

## 4. 实施步骤

1. 整理内容事实源 schema。
2. 建立 dry-run 校验命令。
3. 做只读影响范围预览。
4. 再决定是否做本地写入 UI。
5. 增加回滚和备份策略。

## 5. 验收

- 作者新增一个公开节点不需要理解代码。
- 新节点能进入 Atlas、Archive、Paths。
- 校验能阻止无摘要、无区域、无权限字段的内容进入精选。
