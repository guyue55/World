# 阶段 9：本地成熟度打磨任务板

**目标：** 暂不考虑外部 Preview / Production，先在 localhost / LAN IP 下把 WorldOS 打磨成可真实使用、可阅读、可探索、可维护的本地产品。

## 批次 9.1：真实访客路径闭环

**目标：**
- [x] 首页能清楚提供地图、路径、档案馆、AI 灯塔等低门槛入口。
- [x] 新手路径能从路径页进入，再进入正文节点。
- [x] 节点阅读后能回到地图、档案馆或路径，不形成死路。
- [x] 档案馆、地图、时间线、灯塔都使用公开数据与中文导览。

**改动范围（执行留痕）：**
- [x] 数据层：新增本地产品成熟度契约。
- [x] 页面层：复核公开页面的反迷路导览和返回出口。
- [x] 组件层：复核路径、节点、灯塔、动态首页组件的成熟度证据。
- [x] 权限边界：继续坚持后端权限控制，前端只体现公开结果。
- [x] 检查脚本：新增 `check:local-product-maturity`。

**不做：**
- 不引入外部 Preview / Production 验收。
- 不把 LAN RC 当作公网发布证据。

**验收命令：**
```bash
npm run check:local-product-maturity
npm run check:mainline
npm run release:local-rc
```
