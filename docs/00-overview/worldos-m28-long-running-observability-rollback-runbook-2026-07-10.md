# WorldOS M28 长期运行观测与回滚手册

> [!IMPORTANT]
> M28 的目标是让本地 / LAN 世界长期运行可观测、可修复、可回滚。

## 1. 目标

- 本地启动状态可见。
- 最近检查结果可见。
- 失败可定位。
- 内容和资产可备份。
- 错误变更可回滚。

对应支柱：S6 长期回访、S7 作者共生、S8 真实可信。

## 2. 观测对象

| 对象 | 证据 |
| --- | --- |
| build | `build:production-ci` 输出 |
| mainline | `check:mainline` 输出 |
| RC | `release:local-rc` 报告 |
| browser | 截图、录屏、console、network |
| content | 内容密度、关系、路径 |
| permission | 私密泄漏扫描 |
| assets | 授权、体积、缺失文件 |

## 3. 回滚策略

- Git commit 是最小回滚单位。
- 内容变更应有备份或生成前快照。
- 资产变更应保留 manifest。
- AI 配置变更必须可禁用。

## 4. 实施步骤

1. 汇总本地运行报告。
2. 建立失败分类。
3. 编写回滚命令清单。
4. 做一次回滚演练。
5. 把演练结果写入报告。

## 5. 验收

- 能定位最近一次失败。
- 能从错误提交回到上一个可用版本。
- 回滚不破坏内容事实源。
