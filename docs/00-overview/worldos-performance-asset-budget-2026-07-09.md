# WorldOS 性能与资产预算

> [!IMPORTANT]
> 世界感不能以臃肿为代价。本预算控制 JS、CSS、图片、音频、Canvas / WebGL、AI 请求和证据产物。

## 1. 目标

确保动态世界按需加载，不拖垮静态阅读；确保场景、氛围、音频和 AI 都有明确预算与降级。

## 2. 预算表

| 类型 | 第一阶段预算 | 约束 |
| --- | --- | --- |
| 首屏 JS | 不因单个场景引入全站重库 | 场景模块按需加载 |
| CSS | 避免重复主题和一次性样式 | 使用共享 tokens 和组件 |
| 图片 | 优先本地优化资产 | 不使用未授权素材 |
| 音频 | 默认不加载，用户 opt-in 后按场景加载 | 不自动播放 |
| Canvas / WebGL | 场景证明必要后引入 | mobile 有静态替代 |
| AI 请求 | 服务端裁剪上下文 | 缓存、限流、超时 |
| 截图证据 | 只保留关键稳定证据 | 避免大量时间戳噪声 |

## 3. 引入依赖规则

新增运行时依赖必须满足：

1. 对应具体场景收益。
2. 有静态 / reduced fallback。
3. 有本地 / LAN 验收方式。
4. 不与现有工具重复主控。
5. 不违反技术栈调研文档的否决项。

## 4. 按需加载

- 场景动效只在场景进入时加载。
- 音频只在用户开启声音后加载。
- AI Provider 只在服务端请求时运行。
- 3D / WebGL 不进入基础首屏。

## 5. 验收

- `npm run build` 不因资产或依赖失败。
- `npm run check:performance-budget` 或后续预算检查通过。
- mobile 首屏无明显阻塞。
- 静态阅读不依赖动态资产。
