# WorldOS M26 世界记忆与回访体验规范

> [!IMPORTANT]
> M26 的目标是让用户愿意回来，并能接着上次探索。

## 1. 目标

世界应记住低风险探索上下文：

- 最近访问节点。
- 未完成路径。
- 最近场景。
- 用户偏好：动效、声景、主题。
- 清除历史入口。

对应支柱：S6 长期回访、S8 真实可信。

## 2. 存储边界

| 数据 | 是否记录 | 存储建议 |
| --- | --- | --- |
| 最近公开节点 | 可以 | localStorage，规模大后 IndexedDB |
| 路径进度 | 可以 | localStorage |
| 声景偏好 | 可以 | localStorage |
| 私密内容正文 | 不记录 | 禁止 |
| AI 对话全文 | 默认不记录 | 后续需明确 opt-in |
| 身份敏感信息 | 不记录 | 禁止 |

## 3. 隐私要求

- 用户能清除回访记忆。
- 不记录私密档案内容。
- 不把本地记忆发送给 AI，除非用户明确触发并经过服务端裁剪。

## 4. 实施步骤

1. 定义 memory schema。
2. 加读取 / 写入 / 清除 API。
3. 首页显示继续探索。
4. Paths 显示未完成路线。
5. Node 显示上次相关出口。
6. 增加清除入口和验证脚本。

## 5. 验收

- 第二次访问与第一次不同。
- 能继续上次路径。
- 能一键清除历史。
- 清除后不再显示继续探索。
