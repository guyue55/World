# WorldOS M20 世界空间连续性规范

> [!IMPORTANT]
> M20 的目标是让切换像真实穿梭，而不是普通页面跳转。

## 1. 目标

用户在 Home、Atlas、Timeline、Archive、Paths、Node、Lighthouse 之间移动时，应知道：

- 从哪里来。
- 为什么去那里。
- 即将到哪里。
- 是否已经抵达。
- 如何返回或继续。

对应支柱：S2 真实穿梭、S6 长期回访。

## 2. 迁移状态

| 状态 | 说明 | 可见证据 |
| --- | --- | --- |
| source | 来源场景残影 | 来源名称、来源视觉余晖 |
| leaving | 离开动作 | 当前场景收束 |
| preview | 目标预告 | 目标场景标识、入口对象 |
| arriving | 抵达动作 | 新场景展开 |
| settled | 沉淀 | 下一步、返回、继续探索 |

## 3. 技术策略

- 主路径继续使用 GSAP 编舞。
- View Transition API 可作为渐进增强候选，不作为唯一方案。
- 状态复杂后再评估 XState。
- `prefers-reduced-motion` 下必须使用静态状态文本替代。

## 4. 实施步骤

1. 定义迁移状态模型。
2. 记录上一个场景和目标场景。
3. 为核心路径制作迁移 grammar。
4. 增加录屏检查脚本或手工录屏清单。
5. 检查快速连续点击、回退、刷新、移动端。

## 5. 验收

- 关键路径录屏能看出迁移连续性。
- reduced-motion 下仍有来源 / 目标 / 抵达说明。
- 不遮挡导航和主要内容。
- 无状态错乱。
