# WorldOS 转场编舞规范

> [!IMPORTANT]
> 转场不是装饰动画，而是场景迁移语法。任何页面切换都必须表达“从哪里来、去哪里、为什么去、如何抵达”。

## 1. 目标

将现有轻量页面入场升级为可感知的场景迁移，同时保持速度、可中断、可降级和本地 / LAN 可验收。

## 2. 转场状态

| 状态 | 含义 | 视觉 / 交互要求 |
| --- | --- | --- |
| `source` | 当前场景仍可识别 | 保留来源标题、区域或关系线索 |
| `leaving` | 用户触发迁移 | 轻微收束，不遮挡主内容 |
| `preview` | 目标场景预告 | 展示目标空间的颜色、纹理或关键词 |
| `arriving` | 目标场景入场 | 目标场景主体先可读，再补动态 |
| `settled` | 抵达完成 | 下一步和返回路径可见 |
| `reduced` | 降级迁移 | 无复杂动画，仅保留状态文本或轻量淡入 |

## 3. 路线编舞

| 路线 | 语义 | 编舞重点 | 禁止 |
| --- | --- | --- | --- |
| Home -> Atlas | 从入口进入地图 | 世界状态展开到地图节点 | 突然跳分类页 |
| Atlas -> Node | 从地图抵达地点 | 来源区域与关系线保留片刻 | 普通文章淡入 |
| Node -> Paths | 从地点进入旅程 | 当前节点转为路径中的一步 | 只显示链接列表 |
| Paths -> Timeline | 从旅程回看时间 | 进度转化为时间位置 | 生硬切换时间线 |
| Timeline -> Archive | 从时间进入档案馆 | 时间事件沉入档案分区 | 只跳归档页 |
| 任意 -> Home | 回到世界中心 | 当前场景缩回入口状态 | 浏览器后退感 |

## 4. 时间预算

| 项 | 预算 |
| --- | --- |
| 常规转场 | 300-600ms |
| 首访仪式 | 默认不超过 2500ms |
| reduced-motion | 0-180ms |
| 可交互恢复 | 不晚于内容可见后 100ms |

## 5. 实现约束

- GSAP 负责复杂时间线编排。
- Motion 只处理 React presence 和局部组件进出。
- 不同时让多套动画系统控制同一节点。
- 不把导航阻塞在动画完成之后。
- 转场注册表必须包含语义名、来源、目标、降级行为。

## 6. 验收

- desktop / mobile 能看到来源与目标语义。
- reduced-motion 不出现残影、遮挡或空白。
- 浏览器后退仍可理解当前位置。
- 所有转场可被 Playwright smoke 或人工清单验收。
