# WorldOS Lighthouse 观测站规格

## 1. 目标

Lighthouse 必须像只读观测站，帮助用户问路、解释和推荐，不像普通聊天工具。

## 2. 必备结构

- 灯塔状态：disabled、ready、thinking、fallback。
- 问路入口：基于当前位置和公开事实源。
- 推荐卡：路径、节点、理由。
- 边界说明：AI 不能读私密原文、不能修改世界。
- 静态回退：Provider 不可用时仍可推荐。

## 3. 动态要求

- 信标和扫描感服务状态，不制造噪声。
- 输出可流式，但内容必须先可控。
- 错误状态可理解。

## 4. 验收

- 前端不持有 Provider Key。
- 推荐有来源和理由。
- AI 不暗示能读取私密内容。
