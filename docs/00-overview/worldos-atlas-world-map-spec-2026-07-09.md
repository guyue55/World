# WorldOS Atlas 世界地图规格

## 1. 目标

Atlas 必须成为世界主地图，而不是分类页。它展示区域、节点、关系、距离和推荐方向。

## 2. 必备结构

- 区域层：世界的主要空间分区。
- 节点层：内容节点像地点。
- 关系层：解释节点为何相关。
- 推荐路径：从当前状态进入下一段旅程。
- 静态列表 fallback：无动态时仍可浏览。

## 3. 动态要求

- 先用 SVG / CSS / GSAP 表达空间关系。
- 关系布局不足时再评估 D3 Force。
- 不为了背景星点引入全站 3D。

## 4. 验收

- 用户能看出区域和节点关系。
- 点击节点像“抵达地点”，不是打开普通文章卡片。
- reduced-motion 下关系仍以文本或静态线索表达。
