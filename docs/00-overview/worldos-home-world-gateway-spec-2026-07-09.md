# WorldOS Home 世界入口规格

## 1. 目标

Home 必须从普通首页升级为世界入口。用户第一眼应感到“抵达一个世界”，并能立刻选择进入地图、路径、时间、档案或灯塔。

## 2. 必备结构

- 首访仪式：可跳过、可降级、默认不超过 2500ms。
- 世界状态：当前时间、内容活性、灯塔状态、推荐方向。
- 主入口：Atlas、Paths、Timeline、Archive、Lighthouse。
- 精选节点：必须有摘要、区域、关系和下一步。

## 3. 动态要求

- 入场表达“抵达”，不做纯 Hero 动画。
- 入口聚焦可用 GSAP 编排。
- reduced-motion 下直接显示世界入口。

## 4. 验收

- 首屏不是 Hero + 卡片列表。
- 用户不懂项目背景也知道下一步。
- mobile 首屏入口不被动效遮挡。
