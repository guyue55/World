# WorldOS AI 灯塔运行规格

> [!IMPORTANT]
> AI 灯塔是只读导览者，不是内容管理员。真实 AI 能力必须服务端运行、接受权限边界、可审计、可回退。

## 1. 目标

让 AI 灯塔能够基于公开事实源解释世界、推荐路径、回答问路问题，同时不读取私密原文、不修改世界、不绕过人工确认。

## 2. 运行原则

- Provider 只在服务端适配。
- 前端不持有 API Key，不拼接权限上下文。
- 默认只读 `public` 与明确授权的 `semiPublic` 摘要。
- 私密、vault、sealed、silent 内容默认排除。
- AI 输出必须可解释、可审计、可降级。

## 3. 能力矩阵

| 能力 | 当前阶段 | 要求 |
| --- | --- | --- |
| 问路 | P1 | 推荐公开路径和节点 |
| 解释 | P1 | 解释节点、区域、关系 |
| 摘要 | P1 | 基于公开摘要，不读取私密原文 |
| 推荐 | P1 | 输出推荐理由和下一步 |
| 维护建议 | P2 | 进入审计队列，不自动执行 |
| 内容修改 | 禁止 | 需要人工和后端权限，当前不做 |

## 4. 请求流程

1. 前端提交问题和当前位置。
2. 服务端解析角色、权限、场景和公开上下文。
3. `context-policy` 裁剪候选事实源。
4. Provider adapter 调用真实或 dry-run Provider。
5. 输出结构化结果：回答、推荐、来源、限制、回退。
6. 写入审计摘要，不写入世界源数据。

## 5. 性能与回退

- 支持静态推荐回退。
- 支持请求超时与取消。
- 常见问题优先缓存。
- 长上下文使用裁剪和摘要，不堆全量内容。
- Provider 不可用时展示“灯塔离线，但可继续浏览”。

## 6. 验收

- 前端无法获取 Provider Key。
- 私密 / vault / sealed 候选被排除并记录原因。
- AI 输出包含推荐依据。
- 禁止自动发布、删除、修改可见性。
- 本地 / LAN 可在 dry-run 与真实 Provider disabled 状态下运行。
