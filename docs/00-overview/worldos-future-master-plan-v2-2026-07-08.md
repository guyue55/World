## 一、执行摘要

古月浮屿是一个以"世界模型"为核心的个人数字世界系统。经过 Phase 1-12 的建设与 8 轮深度审查收敛，当前世界已进入"可走、可停留、可维护"的稳定态。

本规划在 v1 的基础上，根据真实进展重新校准目标。v1 中 M13（内容生态）和 M14（技术收束）的大部分目标已在本轮工作中提前完成，因此 v2 将焦点转向本地可达的体验深化和内容收尾。

### 2.2 核心洞察

1. WorldOS 独有定位未被任何项目覆盖：世界模型（Node-Area-Relation-Path-Event）+ 权限分层 + 门禁宪法 + AI 边界化 + 生命档案 + 传承协议，这六个维度组合在当前生态中没有对标。
2. oldwinter 的本地优先+隐私优先值得学习：privacy-first 标签与 WorldOS 的"私密层不进前台"原则高度一致。
3. 构建产物从 29MB 增长到 56MB 需要关注：虽然本地/局域网可承受，但说明内容增长带来了静态资源膨胀，需在 Phase 20 中评估。

## 三、SWOT 更新

### 3.1 优势
- 门禁体系成熟：四合一验证（daily + boundary-full + strict + local-rc）全绿
- 架构收敛到位：lib 89 文件、warning 全零、_legacy/ 双向零耦合
- 权限规范清晰：后端 gate 主导，前端零硬编码
- 动效体系完整：motion-grammar 六型抽象 + GSAP/FM 职责分离
- 内容形成规模：200 节点覆盖 8 个区域，全部有正文、路径和关系吸收

### 3.2 劣势
- 构建产物膨胀：56MB（v1 基线 29MB），需评估静态资源优化
- 无深色模式：当前只有浅色主题，夜间体验缺失
- 无键盘导航体系：skip-link 存在但无系统化的焦点管理
- 内容相似度需要持续守门：本轮 jaccard 最大 0.5565，仍需保持复写质量
- AI 未真接：抽象层完备但 provider 未接入

### 3.3 机会
- 深色模式零依赖实现：CSS 变量 + Tailwind dark: 变体即可
- 本地 AI 可接 Ollama：无需公网即可体验 AI 灯塔
- 内容增长已达 Phase 19 目标：后续重点转为内容质量、路径体验和可维护性
- 体验精修 ROI 高：门禁已保底，可安全打磨

### 3.4 威胁
- 构建产物持续膨胀：若不控制，200 节点时可能超 100MB
- 单人维护疲劳：门禁脚本 265 个，需警惕仪式化
- 技术栈锁定：React 19 生态尚未完全成熟

## 四、终极目标（v2 校准）

让 WorldOS 成为本地可运行、体验精致、内容丰富、可维护可传承的个人数字世界产品。

四个维度：
1. 可运行：本地 + 局域网稳定运行，无需公网依赖
2. 体验精致：深色模式、键盘导航、无障碍达标、动效流畅
3. 内容丰富：200+ 节点、300+ 关系、50+ 事件、20+ 路径
4. 可维护可传承：门禁不破、lib 收束到位、传承协议完备

## 六、交叉关注点

### 6.1 权限体系
- 所有新功能默认走后端 gate
- 前端只做"表达"，后端做"决策"
- 每增内容验证 isPublicVisible / isPublicArea / isEventPublic

### 6.2 设计系统
- 色板：ink/paper/mist/moss/lake/night/gold 七色
- 圆角：world(1.5rem) / 2rem / 2.5rem 三级
- 阴影：soft 统一
- 动效：motion-grammar 六型（arrival/emergence/connection/flow/focus/feedback）
- 图标：Lucide React

### 6.3 中文优先
- UI 文案默认中文
- 变量/函数英文，注释和文档中文
- 日期格式 YYYY-MM-DD 或 YYYY年M月D日

### 6.4 门禁不破
- 每次提交前至少 check:daily
- 每个 Phase 结束跑四合一
- warning 必须归零，不 skip

### 6.5 提交规范
- 格式：xxx(xxx): 中文xxx
- 中文注释
