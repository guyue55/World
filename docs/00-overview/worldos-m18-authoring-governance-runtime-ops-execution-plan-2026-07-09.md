# WorldOS M18 作者治理与运行运维执行计划

## 1. 目标

让 WorldOS 不只是一次性开发产物，而是可持续维护的个人数字世界。作者应能低门槛地新增内容、维护关系、管理资产、复核 AI、备份回滚，并在本地 / LAN 下稳定运行。

## 2. 执行项

| 项 | 操作 | 验收 |
| --- | --- | --- |
| 内容新增流程 | 写清新增节点、区域、关系、路径、时间锚点步骤 | 新人按文档可完成一个节点 |
| 场景扩展流程 | 写清新增场景或能力时需更新哪些 registry / spec / check | 不出现游离组件 |
| 资产审批 | 图片、音频、字体、动效必须有来源、授权、体积、用途 | 无来源不明资产 |
| AI 审计复核 | AI 推荐、摘要、维护建议进入审计，不自动写源数据 | AI 不越权 |
| 权限复核 | private / vault / sealed / public 的事实源与展示分离 | 前端只做体现 |
| 备份回滚 | 明确本地备份、导出、恢复和失败处理 | 本地改坏可恢复 |
| 日常运行 | 明确 `npm run start`、LAN 访问、RC、常见故障处理 | 作者能自助运行 |

## 3. 需要沉淀的运维入口

- 中文作者手册。
- 内容新增清单。
- 资产新增清单。
- AI 审计清单。
- 本地 / LAN 运行手册。
- 回滚与备份手册。

## 4. 检查命令

- `npm run check:docs`
- `npm run validate:world`
- `npm run check:content-life`
- `npm run check:permission-boundary`
- `npm run check:api-boundary`
- `npm run check:mainline`
- `npm run release:local-rc`

## 5. 人工验收

- 作者能用中文文档完成一个公开节点新增。
- 作者能判断一个素材是否能放进项目。
- 作者能知道 AI 建议为什么不能自动写入。
- 作者能在局域网访问失败时定位常见问题。

## 6. 边界

- 不建设复杂后台来替代当前本地工作流。
- 不把上线、外部域名、HTTPS、外部监控纳入当前阶段。
- 不让作者流程绕开权限和内容事实源。
