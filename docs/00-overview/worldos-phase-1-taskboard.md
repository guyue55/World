# 阶段 1：公开世界主线闭环任务板

**目标：** 普通访问者第一次进入，就能看懂“这是一个世界”，并能顺畅走完。

## 批次 1.1：首页入口收束

**目标：**
- [ ] 首页只保留最关键入口：进入世界、走 8 分钟路径、看地图、进档案馆、问灯塔。
- [ ] 首页文案优先使用中文、低门槛，避免内部工程词汇。
- [ ] 首屏必须明确“这是什么”“怎么开始”“边界是什么”。

**改动范围（执行留痕）：**
- [ ] 数据层：精简首页引用的公开 surface 数据配置
- [ ] 页面层：重构 `app/page.tsx` 等入口视图结构
- [ ] 组件层：优化 `ProductHome` 或对应展示组件文案
- [ ] 权限边界：确保首页仅透出 `public` 数据
- [ ] 检查脚本：更新首页结构验证脚本

**不做：**
- 不引入复杂的滚动动效，仅收束结构和文案。

**验收命令：**
```bash
npm run check:homepage
npm run check:experience:public
```

## 批次 1.2：地图、路径、节点闭环

**目标：**
- [ ] `/atlas` 表达世界地图和区域关系。
- [ ] `/paths` 表达适合新手的阅读路线。
- [ ] `/paths/[id]` 表达路径旅程、当前进度和下一步。
- [ ] `/node/[slug]` 表达节点阅读、相关关系、返回路径。

**改动范围（执行留痕）：**
- [ ] 数据层：确保 node 和 path 的互相链接数据完整
- [ ] 页面层：处理这些主路由的互相跳转闭环
- [ ] 组件层：强化导览卡片与“下一步”按钮
- [ ] 权限边界：确认这些路由只暴露合法的内容节点
- [ ] 检查脚本：增加路径闭环静态检查

**不做：**
- 不做地图的高级 3D 渲染，保持静态 2D DOM 优先。

**验收命令：**
```bash
npm run check:atlas
npm run check:path-guidance
npm run check:node-reading
```

## 批次 1.3：时间线与档案馆闭环

**目标：**
- [ ] `/timeline` 表达时间河流，不只是事件列表。
- [ ] `/archive` 表达档案馆和检索入口。
- [ ] 搜索、筛选、空状态都使用中文、并且对用户可解释。
- [ ] 档案馆只显示公开内容，不能通过前端筛选绕过边界。

**改动范围（执行留痕）：**
- [ ] 数据层：审查事件轴（Event）数据的读取
- [ ] 页面层：优化 `/timeline` 与 `/archive` 视图
- [ ] 组件层：定制中文无结果空状态组件
- [ ] 权限边界：严查搜索接口与过滤功能，确保不能越权查私密
- [ ] 检查脚本：执行内容与边界双检

**不做：**
- 不引入外部庞大的全文检索引擎。

**验收命令：**
```bash
npm run check:timeline
npm run check:content-archive
npm run check:boundary
```
