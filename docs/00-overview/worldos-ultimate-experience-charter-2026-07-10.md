# WorldOS 终局体验宪章

> [!IMPORTANT]
> 本宪章把“真格世界 / 宇宙”的高目标翻译成不可遗漏的体验支柱。后续任何阶段、计划、实现和验收，都必须接受这 8 个支柱的约束。

## 1. 结论

M8-M18 能把 WorldOS 推到本地成熟世界 MVP+；M19-M30 才开始冲击更高目标。更高目标不是“页面更漂亮”，而是让用户感到：

> 我进入的是一个有空间、有路、有生命、有陪伴、有记忆、值得回来的个人世界。

## 2. 八个终局体验支柱

| 支柱 | 目标 | 反面表现 | 主要阶段 |
| --- | --- | --- | --- |
| S1 独立空间 | 每个场景像一个独立空间 | 同一套页面模板换文案 | M19 / M29 |
| S2 真实穿梭 | 切换像在世界中移动 | fade 一下就换页 | M20 / M29 |
| S3 内容生命体 | 内容像地点和生命体 | 文章卡片、详情页、标签列表 | M21 / M25 |
| S4 陪伴型灯塔 | AI 灯塔像真正陪伴者 | 搜索框、问答面板、工具按钮 | M22 |
| S5 统一世界观 | 音乐、氛围、视觉、交互同属一个世界 | 动效、音频、UI 各玩各的 | M23 / M29 |
| S6 长期回访 | 用户愿意长期逛、回访、继续探索 | 看完一页就离开 | M26 / M28 |
| S7 作者共生 | 作者能持续养世界 | 每次更新都靠改代码 | M25 / M28 |
| S8 真实可信 | 世界可验证、可回退、可降级 | 自证式完成、旧截图误判 | M17 / M28 / M30 |

## 3. 支柱定义

### S1：每个场景像独立空间

每个核心场景必须有自己的空间语法：

| 场景 | 空间语法 |
| --- | --- |
| Home | 门厅、抵达、选择方向 |
| Atlas | 穹顶、星域、节点、关系线 |
| Timeline | 河道、水位、涟漪、回看 |
| Archive | 大厅、书架、卷宗、索引 |
| Paths | 道路、脚印、下一站、完成印记 |
| Node | 房间、护照、正文、关系门、出口 |
| Lighthouse | 灯塔、光束、问路、边界 |

验收问题：

- 遮掉文字后，还能看出这是哪个场景吗？
- 用户能说出“我现在在哪”吗？
- 这个场景有自己的交互，而不是复用同一个卡片布局吗？

### S2：切换像真实穿梭

场景切换必须包含：

- 来源残影：用户知道从哪里来。
- 离开动作：当前场景不是突然消失。
- 目标预告：用户知道即将去哪里。
- 抵达状态：进入后有明确“到达”反馈。
- 沉淀记忆：回退或继续探索时保留上下文。

验收问题：

- 录屏里能看出迁移叙事吗？
- reduced-motion 下是否仍有文本 / 状态替代？
- 快速切换时是否不会错乱或遮挡？

### S3：内容像地点和生命体

内容不是文章集合，而是世界中的地点和生命体。每个重要节点应具备：

- 地点护照：标题、区域、生命周期、状态。
- 关系门：为什么连接到这些节点。
- 时间痕迹：何时出现、何时变化、如何回看。
- 路径位置：在哪些旅程里被访问。
- 出口：下一站、返回地图、进入档案馆、问灯塔。

验收问题：

- 一个节点是否能被 Atlas、Timeline、Archive、Paths、Lighthouse 同时吸收？
- 用户读完后是否知道下一步去哪？
- 节点是否有“生命状态”，而不是静态文章？

### S4：AI 灯塔像真正陪伴者

灯塔不是全能 AI，也不是搜索框。它是只读陪伴者：

- 解释当前位置。
- 推荐下一站。
- 回答“我该怎么看这个世界”。
- 承认不知道。
- 显示依据。
- 不修改事实源。
- 不越过权限边界。

验收问题：

- AI 是否引用公开事实源？
- AI 是否避免编造不存在的节点？
- AI 是否能在失败时给出静态导览？
- 用户是否感觉它在陪同，而不是只是在查资料？

### S5：音乐、氛围、视觉、交互形成统一世界观

统一世界观不等于到处加效果，而是：

- 色彩服务场景。
- 动效服务迁移。
- 声景服务抵达和状态。
- 微交互服务探索。
- 空状态、错误状态、加载状态都属于同一个世界。

验收问题：

- 声音关闭后体验仍完整吗？
- reduced-motion 后语义还在吗？
- UI、音频、动画是否表达同一套场景人格？

### S6：用户愿意长期逛、回访、探索

长期回访来自可持续的“继续感”：

- 最近访问。
- 未完成路径。
- 新增内容提示。
- 推荐下一站。
- 回到上次位置。
- 清除历史入口。

验收问题：

- 第二次访问是否和第一次不同？
- 用户能接着上次的路径走吗？
- 系统是否尊重隐私和清除权？

### S7：作者与世界共生

世界必须能被作者持续养护：

- 中文低门槛录入。
- 内容、关系、路径、时间、资产可预览。
- 修改前有校验。
- 修改后能看到影响范围。
- 失败可回滚。

验收问题：

- 作者新增一个节点是否需要写代码？
- 新内容是否自动进入相关场景？
- 修改错误是否可撤回？

### S8：真实可信

终局体验必须可验证：

- 生产态截图。
- 关键路径录屏。
- 自动检查。
- 人工体验量表。
- 权限和 AI 边界扫描。
- 资产授权记录。
- 失败和修复账本。

验收问题：

- 是否存在旧构建、旧截图、旧报告误判？
- 是否明确记录仍未达标项？
- 是否能复现验收结果？

## 5. 终局否决项

出现以下任一情况，不能称为 9/10 终局候选：

- 场景仍像同一套模板。
- 切换仍像普通页面跳转。
- 内容仍像文章详情和卡片列表。
- AI 灯塔不能说明依据或越过权限边界。
- 音乐、氛围、视觉和交互互不相干。
- 第二次访问没有继续探索价值。
- 作者维护仍主要依赖改代码。
- 验收只依赖脚本，没有截图、录屏和人工审查。
