# WorldOS 错误与降级体验规格

## 1. 目标

任何动态能力失败时，WorldOS 都应受控降级，而不是白屏、卡死或误导用户。

## 2. 降级场景

| 场景 | 处理 |
| --- | --- |
| 无 JS | 展示静态内容和基础导航 |
| reduced-motion | 关闭复杂转场 |
| reduced-sensory | 关闭声音和强氛围 |
| 音频不可用 | 保持静音，不报错 |
| AI 不可用 | 静态推荐回退 |
| LAN 不可用 | 提示网络原因，localhost 仍可用 |
| 权限不足 | 前端隐藏或解释，事实来自后端 / 契约 |
| 数据缺失 | 展示空状态和修复提示 |

## 3. 体验要求

- 降级状态必须可理解。
- 不暴露敏感信息。
- 不让用户困在无出口状态。
- Home 和静态阅读永远优先可用。

## 4. 验收

- 关闭 JS 或模拟低能力设备时核心内容可读。
- AI disabled 状态可正常浏览。
- 音频关闭不影响场景完成度。
