# WorldOS Paths 旅程系统规格

## 1. 目标

Paths 必须像旅程系统，而不是链接清单。它帮助新访客降低理解门槛，沿路径探索世界。

## 2. 必备结构

- 起点：说明适合谁、为什么开始。
- 节点序列：每一步有理由和摘要。
- 进度：当前、已读、下一步。
- 返回地图：可随时回 Atlas。
- 完成状态：读完后的下一段建议。

## 3. 动态要求

- 进度流表达行进感。
- 当前节点与下一步轻量聚焦。
- reduced-motion 下使用静态进度文本。

## 4. 验收

- 新用户能不看说明直接跟随路径。
- 每一步都有“为什么相关”。
- 路径能连接 Atlas、Node、Timeline。
