# WorldOS M27 多层权限与私密宇宙规范

> [!IMPORTANT]
> M27 的目标是让公开、私密、owner、家庭等边界清楚。前端只能体现权限，不能成为权限事实源。

## 1. 权限层

| 层级 | 说明 | 前端行为 |
| --- | --- | --- |
| public | 可公开展示 | 正常显示 |
| unlisted | 不在公开精选出现 | 仅直接入口或 owner 可见 |
| private | 私密档案 | 公开构建不可包含正文 |
| family | 家庭 / 特定群组 | 后端或数据契约判定 |
| owner | 作者控制台 | 只读 / 编辑按后端边界 |

对应支柱：S4 陪伴型灯塔、S8 真实可信。

## 2. 硬边界

- 私密正文不得进入公开构建。
- AI 灯塔默认只读 public。
- 前端不硬编码权限绕过。
- owner 页面必须有 API / 数据契约守卫。

## 3. 实施步骤

1. 审计所有内容 visibility。
2. 建立权限事实源。
3. 为 API route 增加守卫清单。
4. 前端只根据返回结果显隐。
5. 增加泄漏扫描。

## 4. 验收

- 公共构建不含私密正文。
- 无前端硬编码权限判断作为事实源。
- AI 不读取私密内容。
- 权限不足有降级体验。
