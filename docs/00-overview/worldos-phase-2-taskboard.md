# 阶段 2：动态宇宙体验系统任务板

**目标：** 不是每页加动画，而是形成统一的“活着的世界”。

## 批次 2.1：动态体验语法

**目标：**
- [ ] 定义 6 类动效语法：抵达、浮现、连接、流动、聚焦、反馈。
- [ ] 每类动效定义适用场景、默认时长、降级方式。
- [ ] 建立统一 hook 和组件约束（如 `useGsapEntrance` 等），避免每个组件单独硬编码动画逻辑。

**改动范围（执行留痕）：**
- [ ] 数据层：无
- [ ] 页面层：无
- [ ] 组件层：抽取并封装基础动效 Hook / Container
- [ ] 权限边界：无
- [ ] 检查脚本：更新代码质量门禁以确保统一规范

**不做：**
- 暂不实施大面积页面改造，仅在组件库层面建立规范与基座。

**验收命令：**
```bash
npm run check:dynamic-world
npm run lint
npm run typecheck
```

## 批次 2.2：主路由动态 surface 全覆盖

**目标：**
- [ ] 首页：实现世界脉冲和实时导览。
- [ ] Atlas：实现星图、连接线、区域聚焦。
- [ ] Timeline：实现时间河流、事件聚焦。
- [ ] Archive：实现档案检索导览和主题入口动效。
- [ ] Paths：实现路径旅程、进度和下一步流动。
- [ ] Node：实现节点开启仪式、关系浮现。
- [ ] Ask：实现灯塔问答台动效。
- [ ] Status：实现世界运行状态心跳。

**改动范围（执行留痕）：**
- [ ] 数据层：无
- [ ] 页面层：将动态基座组件应用到各大主路由
- [ ] 组件层：各业务组件按统一规范重构挂载点
- [ ] 权限边界：无
- [ ] 检查脚本：执行 `check:dynamic-world` 覆盖率检查

**不做：**
- 不打断页面生命周期，不在 SSR 阶段引入客户端阻塞。

**验收命令：**
```bash
npm run check:dynamic-world
npm run build
```

## 批次 2.3：可访问性与性能降级

**目标：**
- [ ] 支持 `prefers-reduced-motion` 无缝降级。
- [ ] 移动端减少大面积运动，优化资源开销。
- [ ] 保证 CSS 基础状态可读，JS/GSAP 失败时页面不空白。
- [ ] 只在必要的 client component 中使用动效，不干扰 server component。

**改动范围（执行留痕）：**
- [ ] 数据层：无
- [ ] 页面层：媒体查询与降级测试
- [ ] 组件层：优化动效 Hook 的降级处理逻辑
- [ ] 权限边界：无
- [ ] 检查脚本：执行性能与阅读舒适度脚本

**不做：**
- 不强制用户接受可能引起眩晕的视差滚动。

**验收命令：**
```bash
npm run check:reading-comfort
npm run check:performance
npm run check:preview-performance
```
