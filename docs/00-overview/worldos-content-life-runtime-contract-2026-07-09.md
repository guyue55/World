# WorldOS 内容生命运行契约

## 1. 目标

内容不再只是文章，而是世界节点。每个公开核心节点都要被 Atlas、Timeline、Archive、Paths、Node、Lighthouse 共同吸收。

## 2. 节点必备字段

| 字段 | 目的 |
| --- | --- |
| title | 节点名称 |
| summary | 中文摘要 |
| area | 所属区域 |
| relations | 关联节点与理由 |
| lifecycle | 生命周期状态 |
| paths | 所属旅程 |
| timeline | 时间位置 |
| visibility | 权限边界 |
| aiReadableSummary | AI 可读摘要 |

## 3. 准入规则

- 无摘要、无区域、无关系的内容不能进入精选。
- 私密原文不能进入公开构建。
- AI 默认只读取公开摘要。
- 节点必须至少有一个出口。

## 4. 验收

- 同一节点可在 Atlas、Timeline、Archive、Paths 中被找到。
- 相关理由不是空标签。
- 用户读完节点后有下一步。
