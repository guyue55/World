# 0. 文档说明

本文档是对本轮及前序相关会话的沟通记录整理，核心围绕“古月浮屿 / 个人数字世界 / 个人博客升级”的定位、世界观、信息架构、数据模型、AI 边界、无 AI 运行、隐私权限、仪式感、V1 落地方案与后续路线。

- 不是逐字会议纪要，而是按主题整理的“沟通记录 + 产品蓝图 + 架构备忘”。

- 为了“宁冗余不错漏”，保留了多个层级、多个候选方案和重复出现的核心原则。

- 后续可以继续拆成：世界宪章、世界协议、体验规范、创世台规范、V1 实施计划、README、开发任务清单。

# 1. 项目背景与演进脉络

最初讨论从个人博客/技术博客网站的工程化建设、README、CI/CD、命令文档、页面视觉检查、移动端检查等问题开始。用户发现传统博客式页面存在“内容缩在中间、两边空旷、顶部标签状态不一致、页面干、缺少图文和生活气息”等问题，因此逐步要求以真实视角、多维度、逐页逐组件检查和优化。

随后项目从“技术博客”升级为更大的个人数字世界：不只是文章列表和作品集，而是可以容纳技术、项目、灵感、设计、生活、家庭、孩子、爱、旅行、历史、AI、过去与未来的长期系统。

# 2. 总体定位

项目不应定义为普通博客，而应定义为“个人数字世界 / Personal Digital World / 一个正在生长的个人数字宇宙”。它对外是一个美丽清澈的个人宇宙，对内是可靠高效的创作与记忆系统，对 AI 是可读、可约束、可审核的世界协议，对未来是一份可导出、可迁移、可回望的生命档案。

- 它不是传统博客、不是单纯作品集、不是纯展示页、不是流量导向内容站。

- 它是公开宇宙、创作系统、记忆花园、私密档案、未来引擎的五合一系统。

- 它要能承载技术、产品、灵感、设计、生活、旅行、家庭、孩子、爱、旧项目、失败、未来计划、时间胶囊。

- 它要允许“完全自由”创作，但通过骨架、协议、规则、权限和草案机制避免混乱。

- AI 是增强层，不是生命层；没有 AI 时世界也必须可浏览、可搜索、可演化、可维护。

# 3. 核心命名与气质

当前最推荐的项目名为“古月浮屿”。它比“宇宙”更温柔，也更适合同时承载技术、生活、家庭、记忆和未来。英文可选方向包括 GuYue Floating World、Moonlit Atlas、GuYue Atlas。

- 中文推荐：古月浮屿。

- 英文偏个人化：GuYue Floating World。

- 英文偏产品化：Moonlit Atlas。

- 短口号：一张书桌连接星河。

- 中口号：这里安放技术、生活、灵感、爱与记忆，让它们在时间中自然生长。

- 长宣言：这不是一个传统博客，而是一个正在生长的个人数字世界。我把技术、项目、灵感、生活、旅行、家庭与未来，一点点安放在这里。有些内容已经成形，有些仍是种子，有些只在世界深处安静发光。

# 4. 世界观核心句与长期精神

- 先创造世界，再让内容降临其中。

- 让世界先存在，让万物慢慢归位。

- 一张书桌连接着一片无垠星河。

- 一粒灰尘也可能是一颗星。

- 一个想法也可能长成一片森林。

- 世界可以无限浪漫，系统必须极其清醒。

- 世界的存在是为了安放生活，不是反过来让生活服务世界。

- 公开价值不等于真实价值。

- AI 是灯塔，不是太阳。

- 探索时浪漫，阅读时清醒。

- 入口清澈，深处浩瀚。

- 世界为生活服务，不让生活服务世界。

- 真实的小世界，比空洞的大宇宙更有力量。

# 5. 世界设计公理

| **公理**                      | **说明**                                                                     |
|-------------------------------|------------------------------------------------------------------------------|
| 世界先于内容                  | 先有世界入口、地图、区域、节点模型、时间河、权限、关系、规则，再安放内容。   |
| 入口清澈，深处浩瀚            | 首页和第一层入口必须克制，复杂内容藏在可探索的深处。                         |
| 自由输入，有序安放            | 用户可以通过对话自由造物，但新造物必须经过草案、归属、权限、无 AI 模式确认。 |
| AI 是增强层，不是生命层       | 世界不能依赖 AI 才能存在；AI 只增强搜索、摘要、命名、路径、隐私检查等能力。  |
| 规则是自然法则                | 无 AI 时依靠时间、标签、生命周期、权限、关系规则实现自然演化。               |
| 公开层不是完整世界            | 公开世界只是前厅，真实世界还包括内室、保险箱、沉默区域。                     |
| 原始记录不可覆盖              | AI 摘要、公开版本、标签、关系不能覆盖 original。                             |
| 一个节点，多种投影            | 同一 Node 可以投影到地图、时间河、档案馆、路径、首页，但底层只维护一份。     |
| 世界语言必须有现实翻译        | 诗意命名负责氛围，现实解释负责理解。                                         |
| 所有深处都要有出口            | 每个页面必须能回到地图、上一区域、档案视图或灯塔。                           |
| 动效必须有语义                | 点亮、沉入、漂浮、展开、收拢、暗淡、连接都应表达真实动作。                   |
| 世界允许低光                  | 停更不是失败，世界可进入 Quiet / Frozen / Repair 等状态。                    |
| 创世台高效优先                | 前台可以浪漫，后台必须清醒。                                                 |
| 长期价值来自回望和导出        | 年度册、导出包、快照、Markdown/JSON、媒体原件都要从设计上预留。              |
| V1 只证明骨架，不追求完整宇宙 | 第一版只需证明它不是普通博客，而是可生长的世界。                             |

# 6. 世界层级与空间结构

世界可以从微观到宏观分层。分层的目的不是炫技，而是让小到一句话、一张照片、一个物件，大到长期项目、人生阶段、宇宙级主题，都能找到自己的尺度。

| **层级** | **内容与含义**                                               |
|----------|--------------------------------------------------------------|
| 微尘层   | 一句话、一个标签、一张票、一粒灰尘、星尘、墨迹、涟漪。       |
| 器物层   | 纸、笔、灯、书、信封、相机、罗盘、玻璃瓶、抽屉、石碑、种子。 |
| 场景层   | 书桌、房间、花园、湖边、阁楼、档案馆、工坊、观星台、实验室。 |
| 地貌层   | 岛屿、河流、森林、海洋、山脉、星域、云海、城市、废墟。       |
| 宇宙层   | 星辰、星系、星云、时间裂缝、未知区域、世界边界、远方坐标。   |

同一内容可在不同尺度出现。例如一次旅行：宇宙级是远方坐标中的一颗星，地貌级是旅行地图上的路线，房间级是纪念品桌上的车票，物件级是明信片，微尘级是当天写下的一句话。

# 8. 深层区域与未来枝叶

- 爱的小屋：亲密关系和共同记忆空间，公开层只展示外庭轮廓，内室默认私密/双方可见。

- 家庭灯火：家庭合影、节日、团圆饭、家庭故事、宠物、老物件。

- 孩子年轮 / 年轮树：孩子成长记录，默认 vault/private，未来孩子拥有查看、导出、隐藏、删除部分内容的权利。

- 旅行远方 / 远方坐标：城市、路线、照片、票据、天气、同行人、纪念品和下次想去。

- 时光回声 / 过去的我：基于历史内容的自我回望空间，无 AI 时按年份和主题聚合旧节点，有 AI 时生成回声式对话。

- 历史剧场：历史资料、时间线、资料卡、地图、沉浸式历史旅程，必须区分史实、解释、想象、AI 生成。

- 失败博物馆 / 遗迹区：失败项目、废弃设计、错误判断、技术踩坑、复盘。

- 声音瓶：语音、环境声、孩子声音、旅行声音、雨声、海浪。

- 世界之镜：私密反思区，观察长期主题、重复出现的想法、创作方向变化。

- 梦境图书馆 / 梦境层：未实现想法、幻想产品、未来设定、可能世界。

# 13. 生命周期模型

| **阶段**     | **说明**                                   |
|--------------|--------------------------------------------|
| seed 种子    | 只有一句话或极轻内容，尚未展开。           |
| sprout 新芽  | 已有基本说明，开始成形。                   |
| growing 生长 | 有正文、媒体、标签、区域或关联。           |
| bloom 盛放   | 内容可读、可公开、状态较完整。             |
| fruit 结果   | 成为代表作、路径核心、导出候选或项目成果。 |
| archive 归档 | 不再活跃，但有参考价值。                   |
| relic 遗迹   | 旧项目、失败、废案、历史版本。             |
| dormant 沉睡 | 暂时不处理，未来可唤醒。                   |
| silent 沉默  | 只保存，不推荐、不回流、不 AI 总结。       |

# 16. AI 设计原则

AI 不应是唯一入口，也不应成为世界的主人。AI 的定位包括灯塔、书记官、星图师、园丁、守门人、建筑师、镜子。

- AI 可做：摘要、命名、标签建议、关系建议、路径建议、公开版草案、隐私风险提示、造物草案、语义问答。

- AI 不可自动做：公开、删除、修改原始内容、改变权限、总结孩子和家庭私密内容并公开、把想象当事实。

- AI 生成内容必须有标识：AI 摘要、AI 建议、AI 推测、AI 草案、AI 想象层、AI 回声。

- 涉及“过去的我”“历史剧场”“人物对话”“家庭记忆总结”时必须标注来源和边界。

| **规则类别** | **说明**                                                                     |
|--------------|------------------------------------------------------------------------------|
| 时间规则     | 今天历史上、长期未更新提醒、纪念日点亮、年度回顾候选。                       |
| 标签规则     | 相同标签聚合，标签数量达到阈值形成星座候选。                                 |
| 生命周期规则 | seed 30 天未更新进入待浇水；项目 90 天未更新提示归档。                       |
| 权限规则     | 家庭/孩子/住宅默认 private 或 vault；公开前必须有摘要；图片公开前检查 EXIF。 |
| 关系规则     | 同一项目、同一旅行、同一天内容自动关联。                                     |
| 展示规则     | 首页只显示精选和近期点亮；主地图只显示 active 区域；私密区域只显示轮廓。     |

# 19. 体验设计与仪式感

仪式感不应只存在于进入世界时，而应覆盖关键行为：打开地图、进入区域、打开节点、阅读、记录、保存、发布、归档、删除、搜索、AI 对话、时间胶囊、回望、维护、离开。

| **场景**   | **仪式/反馈**                                                  |
|------------|----------------------------------------------------------------|
| 进入世界   | 抵达：光亮起、星尘浮动、书桌显现、灯塔微亮、宣言出现。         |
| 打开地图   | 观测：地图展开、主要区域点亮、远方区域雾中显示。               |
| 进入区域   | 抵达某地：技术星域更冷静，产品工坊像装置，记忆湖有光片和涟漪。 |
| 打开节点   | 展开：文章是纸页/卷轴，项目是工坊装置，照片是湖面光片。        |
| 阅读       | 静读：背景退远、动效降低、正文优先。                           |
| 记录新内容 | 造物：灵感成为种子，照片落入记忆湖，项目出现未点亮装置。       |
| 保存       | 安放：一颗新的种子已落入灵感云层。                             |
| 发布       | 点亮：这张卷轴被点亮，旅人现在可以读到。                       |
| 归档       | 沉淀：送入档案馆、沉入地层、化为遗迹。                         |
| 删除       | 封存优先，永久删除需清楚说明不可恢复。                         |
| AI 不可用  | 灯塔低光运行，但地图、时间河、档案馆仍可用。                   |

# 20. 反迷路机制

- 首页主入口不超过 5 个。

- 一级主区域不超过 7～9 个。

- 每页都回答：我在哪里、这里是什么、我能做什么、我能去哪、如何回到地图、如何搜索、谁能看到。

- 所有诗意名称必须配现实解释。

- 复杂图谱默认折叠，星图默认只显示强关系。

- 每个深层页面必须有返回地图、返回上一区域、切换档案视图、询问灯塔。

- 世界模式负责体验，档案模式负责效率。

- 游客地图、深潜地图、创世地图分开，不把维护信息暴露给普通访客。

# 21. 首页设计

首页是世界前厅，不是文章列表，也不是普通作品集首页。建议结构：

| **首页区块** | **内容**                                                                   |
|--------------|----------------------------------------------------------------------------|
| 第一屏       | 创世书桌 + 星河 + 核心宣言；按钮：进入世界、打开地图、沿时间河、点亮灯塔。 |
| 第二屏       | 这里是什么：不是博客，而是正在生长的个人数字世界。                         |
| 第三屏       | 五个主区域：技术星域、产品工坊、记忆湖、时间河、档案馆。                   |
| 第四屏       | 精选路径：第一次来、技术与 AI、世界如何被创造。                            |
| 第五屏       | 最近发生：世界日志、最近点亮、正在生长。                                   |
| 第六屏       | 关于古月：技术、产品、AI、生活、创作。                                     |
| 第七屏       | 世界宪章入口。                                                             |

# 22. 世界地图 / Atlas

Atlas 不只是漂亮地图，也应是世界目录。它可以提供世界视图和目录视图：世界视图给氛围，目录视图给理解。V1 初始只显示主区域，点击后再展开子区域，避免一开始展示过多节点。

- 主区域：创世原点、时间河、技术星域、产品工坊、灵感云层、记忆湖、档案馆、AI 灯塔。

- 游客版地图只显示 public / semiPublic 轮廓和精选公开区域。

- 主人版地图显示私密结构、草案、内容债务、规则、孤岛节点等维护信息。

- 记忆湖深处的爱的小屋、家庭灯火、年轮树在公开层可以只显示轮廓。

# 23. 节点详情页

节点页是世界最基本的页面。它必须统一、清晰、可读。

- 顶部：当前位置面包屑。

- 标题区：现实标题 + 世界标题。

- 元信息：类型、区域、生命阶段、权限、时间、AI 参与。

- 正文：阅读优先，动效退远，代码块清晰。

- 相关节点：强相关最多 3 个，更多关系可进深潜视图。

- 节点护照：它是什么、从哪里来、属于哪里、谁能看见、和谁相连、下一步可以做什么。

# 24. 时间河

时间河不是普通“最新文章”。它记录世界事件：新增节点、节点生长、区域点亮、规则触发、AI 草案确认、归档、复活、世界宪章修订。

- 可以显示真实日期 + 世界纪年，例如：2026-05-19｜创世初期。

- 可支持今天历史上、一年前的今天、某个旧项目复活、某颗沉睡种子回流。

- 私密内容只在主人视野回流；沉默内容不主动回流。

# 25. 档案馆 / Archive

档案馆是“现实模式”的核心，是反迷路保险。它不需要强沉浸，而要清晰高效。

- 支持搜索、类型筛选、区域筛选、标签筛选、时间筛选、状态筛选、权限筛选、排序。

- 技术内容应支持环境、命令、适用版本、更新时间、风险提示。

- 档案馆可展示全部公开内容，也可在主人视角展示私密/草案/归档内容。

# 26. 精选路径

路径比分类更重要。分类帮助整理，路径帮助理解。V1 至少准备三条路径。

| **路径**           | **内容**                                                                                       |
|--------------------|------------------------------------------------------------------------------------------------|
| 第一次来到古月浮屿 | 世界宣言、关于古月、世界地图、代表技术节点、代表项目节点、生活片刻、AI 灯塔。                  |
| 技术与 AI 探索路径 | AI Agent、CLI、MCP、ADK、开发工具、技术排错、项目实践。                                        |
| 这个世界如何被创造 | 为什么不是博客、万物皆节点、世界骨架、无 AI 自然演化、AI 是灯塔不是太阳、仪式感交互、V1 蓝图。 |

# 27. 创世台 / 后台

创世台是造物主的工作台。前台可以浪漫，后台必须清醒。创世台服务于快速记录、收集箱、草案区、内容债务、规则引擎、AI 审核、权限检查、快照回滚、导出备份。

- 每日入口：记录一粒光、整理一颗种子、继续一个项目。

- 收集箱 / 星尘港：接收一句话、图片、语音、链接、代码、报错、项目想法、AI 对话。

- 内容债务：缺标题、缺摘要、缺区域、缺权限、缺关联、缺更新时间、AI 未审核、EXIF 未清理。

- 世界健康度：云层拥挤度、星线完整度、档案清洁度、守门安全度、工坊活跃度、备份完整度。

- 维护档位：1 分钟记录一粒光；5 分钟整理一颗种子；15 分钟连接几颗星；30 分钟修复一个小区域；半天世界修复日。

# 28. 生活、家庭、孩子、爱与旅行

生活层不应作为普通相册，而应成为记忆湖、爱的小屋、家庭灯火、年轮树、远方坐标等温柔且受保护的空间。

- 生活片刻：一张照片 + 一句话 + 时间 + 心情 + 权限。

- 爱的小屋：外庭公开或半公开，内室私密/双方可见；记录纪念日、旅行、饭菜、信件、未来愿望。

- 孩子年轮：默认 vault/private，记录第一次、身高、声音、画作、趣言、写给未来的信；未来孩子应拥有查看、导出、隐藏、删除部分内容的权利。

- 家庭灯火：每个家庭成员是一盏灯，团聚时灯火靠近；记录节日、团圆饭、合影、老物件。

- 旅行远方：城市、路线、照片、票据、天气、同行人、纪念品、遗憾、下次想去。

- 住宅记忆：客厅、厨房、书房、卧室、阳台、门口的时间切片；默认私密。

# 29. 内容主权、数据主权与隐私主权

- 内容主权：内容有权先以不完整的形态存在，不被分类和模板绑架。

- 数据主权：正文使用 Markdown/MDX，元数据 JSON/YAML，关系 relations.json，区域 areas.json，路径 paths.json，权限 permissions.json，世界日志 logs.json，媒体保留原件。

- 解释主权：AI 可以整理事实和线索，但不能替用户定义记忆的意义。

- 隐私主权：公开是经过确认的动作，不是默认结果。

- 时间主权：世界允许低光、静默、冻结、修复、复活，不让更新压力控制生活。

- 审美主权：不要被赛博朋克、Web3、Notion 风、过度 3D 等短期潮流带跑；主审美稳定为纸上宇宙、书桌星河、自然轻科技。

# 30. 审美与视觉系统

- 核心画面：一张安静的书桌，连接一片无垠星河。

- 主美学：纸上宇宙 / 书桌星河 / 自然轻科技。

- 色彩：月白、雾蓝、浅金、墨绿、纸纹、柔光、星尘、玻璃、留白。

- 避免：过度霓虹、过度 3D、过度游戏化、过度 Web3、过度模板化、满屏粒子。

- 动效：轻、可跳过、不影响任务；入口稍强，日常轻量，高频操作效率优先，危险操作确认感增强。

- 移动端：掌中浮屿，底部罗盘，纵向时间河，阅读优先，动效简化。

# 31. V1 范围冻结

沟通中反复收束：V1 不做完整宇宙，只做核心城，证明它不是普通博客，而是可生长的世界。

- V1 做：世界入口、世界地图、节点系统、时间河、档案馆、精选路径、AI 灯塔低光模式、世界宪章、第一批真实内容、基础搜索、基础规则。

- V1 不做：完整 3D 宇宙、复杂多用户权限、完整家庭系统、完整孩子年轮、完整爱的小屋、全量 AI RAG、声音系统、历史剧场正式版、复杂社交系统。

- V1 成功标准：打开后不像普通博客；10 秒内知道这里是什么；地图清楚；节点能回到地图；阅读页专业；无 AI 可探索；移动端可用；公开与私密模型分离。

# 36. 风险与禁区

- 首页入口超过 10 个。

- 区域命名只有诗意，没有现实解释。

- 所有复杂功能都直接升为一级入口。

- AI 生成内容没有标注。

- 私密内容进入公开构建。

- 没有档案视图兜底。

- 阅读页动效过多。

- 新增造物没有注册。

- 灵感长期只增不整理。

- 世界地图变成一团复杂图。

- 过早做完整 3D、全量 AI、复杂权限、社交系统。

- 内容没有真实锚点，变成漂亮空壳。

1.  最初从个人博客/技术博客网站的页面体验、文档、命令、CI/CD、页面干、移动端和真实视角检查开始。

2.  提出不应只是博客，而应是闲适自然、图文结合、生活和技术并存的网站。

3.  继续上升为“个人数字世界”，提出世界/宇宙可容纳万物，小到灰尘、纸笔，大到星河、历史、未来。

4.  明确“先创造世界，再添加真正产物”的思路，提出世界需要自然演化、四季变化、万物生长。

5.  讨论世界浩瀚宏大但不繁琐、不迷路：通过分层进入、罗盘、地图、路径、档案视图、现实解释解决。

6.  讨论完全自由创作：通过对话创造新造物，例如过去的我、遨游历史；但必须有造物草案和准入规则。

7.  讨论 AI 不稳定：AI 只能是增强层；无 AI 时依靠规则、搜索、路径、地图和模板日志运行。

8.  讨论不仅入口需要仪式感，地图、区域、节点、搜索、记录、发布、归档、删除、错误、空状态都要有世界化反馈。

9.  反复要求继续补充建议，逐步形成自解释、自收束、自修复、自扩展、渐进揭示、三层体验、主权设计等理念。

10. 生成“古月浮屿 V1｜世界数据流与节点投影管线图”，表达输入层、节点化层、世界协议层、规则与演化层、投影生成层、体验层。

11. 最终开始收束到 V1：核心城、第一批区域、第一批节点、页面结构、技术底座、README、V1 冻结和第一周任务。
