## 1. 项目核心定位

### 1.1 它不是普通博客

我们反复确认：这个项目不应只是“技术博客”“个人博客”“作品集”或“文章列表网站”。

更准确的定位是：

- 个人数字世界
- 个人数字宇宙
- 自演化的个人创作与记忆系统
- 可创世、可游览、可生长的个人数字空间
- 公开宇宙 + 创作系统 + 记忆花园 + 私密档案 + 未来引擎
- 一个能够承载技术、产品、灵感、生活、爱、家庭、孩子、旅行、时间、AI、未来的长期系统

核心句式：

> 这不是一个传统博客，而是一个持续生长的个人数字世界。

> 先创造世界，再让内容降临其中。

> 让世界先存在，让万物慢慢归位。

> 一张书桌连接星河。

> 这里安放技术、生活、灵感、爱与记忆，让它们在时间中自然生长。

### 1.2 最终愿景

对外，它是一片美丽浩瀚、可探索的个人宇宙。  
对内，它是一套清醒可靠的创作、记忆、档案和维护系统。  
对亲密关系，它是被保护的记忆容器。  
对创作，它是灵感、项目和知识的生长场。  
对未来，它是一份可以被重新打开、理解、导出、迁移和传承的时间之书。

### 1.3 最终目标形态

它应当做到：

- 对外是宇宙
- 对内是工作台
- 对未来是档案
- 对 AI 是可读协议
- 对自己是一个可以长期回来、继续创造、安放生活的地方

- 回望

- 守护

## 5. 世界设计公理

这些公理应当进入《世界宪章》。

### 公理一：世界先于内容

传统博客是：

```text
先有文章，再有分类。
```

古月浮屿应当是：

```text
先有世界，再安放内容。
```

第一版最重要的是搭建世界骨架，而不是堆满内容。

### 公理二：入口清澈，深处浩瀚

世界可以无限大，但入口必须简单。

首页不应该一次性展示全部概念，而应该只给少量核心入口：

- 进入世界
- 打开地图
- 沿时间河
- 点亮灯塔
- 随机漂流

复杂性应藏在可探索的深处。

### 公理三：自由输入，有序安放

用户可以随时通过对话创造新东西，例如：

- 我想和过去的自己对话
- 我想遨游历史
- 我想创建一座梦境图书馆
- 我想让旅行照片变成一条河
- 我想给未来的孩子做一颗星球

但系统不应直接把这些想法变成杂乱页面，而应形成造物草案：

```text
自由表达
  ↓
生成造物草案
  ↓
确认现实用途
  ↓
确认世界隐喻
  ↓
确认所属区域
  ↓
确认权限
  ↓
确认无 AI 模式
  ↓
确认 AI 增强模式
  ↓
进入草案区 / 实验区 / 正式区域
```

### 公理四：AI 是增强层，不是生命层

世界不能依赖 AI 才能运行。

无 AI 时也必须具备：

- 首页
- 地图
- 节点详情
- 时间河
- 档案视图
- 本地搜索
- 精选路径
- 规则推荐
- 世界日志模板
- Markdown / JSON 数据

AI 可用时再增强：

- 语义搜索
- 摘要
- 命名
- 标签建议
- 关系建议
- 路径生成
- 隐私检查
- 过去的我回声
- 历史剧场导览

核心句：

> AI 是灯塔，不是太阳。

### 公理五：规则是世界的自然法则

无 AI 自然演化应依赖规则引擎，而不是幻想。

基础规则包括：

- fragment 超过 30 天未更新 → 进入待浇水
- 同一 tag 超过 5 个节点 → 形成星座候选
- 项目 90 天未更新 → 提示归档或复活
- 技术文章 180 天未更新 → 提示检查版本
- 照片上传 → 默认进入记忆湖 private
- 孩子 / 家庭 / 住宅内容 → 默认 private 或 vault
- 公开节点缺少摘要 → 不进入精选路径
- AI 生成内容 → 默认进入草案区

### 公理六：公开层不是完整世界

公开世界只是前厅，不是全部。

完整世界还包括：

- 私密记录
- 家庭记忆
- 爱的小屋
- 孩子年轮
- 住宅记忆
- 私密信件
- 时间胶囊
- 保险箱
- 沉默区域

关键原则：

> 公开展示温度，私密保存完整。

### 公理七：原始记录不可覆盖

AI 摘要、公开版本、标签、关系、标题优化不能覆盖原始内容。

重要节点应有分层：

- original：原始记录
- private：完整私密版
- public：公开脱敏版
- summary：摘要
- ai：AI 辅助内容
- archive：归档版本

### 公理八：一个节点，多种投影

同一个 Node 可以在多个地方出现，但底层只保留一份。

例如：

一篇“飞书 MCP 接入 ADK Agent 实践”可以投影为：

- 技术星域中的星体
- 档案馆中的卷宗
- 时间河中的事件
- 技术与 AI 路径中的一步
- 首页最近点亮中的纸页

### 公理九：世界语言必须有现实翻译

诗意命名必须配现实解释。

例如：

- 记忆湖｜生活照片、日常片刻、旅行和家庭记忆
- 时间河｜按时间组织内容、阶段和回望
- 灵感云层｜未完成想法、草稿和未来可能性
- AI 灯塔｜搜索、问答、导航和整理
- 时光回声｜基于过去内容的自我回望空间

### 公理十：所有深处都要有出口

任何页面都必须提供：

- 当前位置
- 返回上一区域
- 打开世界地图
- 打开罗盘搜索
- 切换档案视图
- 询问灯塔
- 回到创世原点

世界可以深，但不能困住人。

### 公理十一：动效必须有语义

动效不是炫技，而是表达世界动作：

- 点亮：发布、精选、完成
- 沉入：归档、记忆、封存
- 漂浮：灵感、未定型、轻内容
- 展开：阅读、打开、进入
- 收拢：整理、归位、合并
- 暗淡：过期、低光、沉睡
- 连接：建立关系

### 公理十二：世界允许低光，不强迫更新

世界应支持：

- 活跃
- 安静
- 低光
- 冻结
- 修复
- 归档
- 复活

低光时：

> 世界最近安静下来。旧星仍然可被观测，时间河仍在低声流动。

### 公理十三：创世台高效优先

前台可以浪漫，后台必须清醒。

创世台要服务：

- 快速记录
- 收集箱
- 批量整理
- 权限检查
- AI 草案审核
- 内容债务
- 规则管理
- 快照回滚
- 导出备份

危险操作必须清楚说明后果。

### 公理十四：长期价值来自回望和导出

世界应支持：

- 年度世界册
- 技术文章合集
- 项目作品集
- 旅行纪念册
- 家庭相册
- 孩子成长册
- 时间胶囊
- 公开世界快照
- 个人完整备份包

### 公理十五：V1 只证明骨架，不追求完整宇宙

V1 做：

- 首页
- 世界地图
- 节点详情
- 时间河
- 档案视图
- 精选路径
- 基础搜索
- 世界宪章
- 少量真实内容
- 基础规则
- AI 灯塔低光模式

V1 不做：

- 完整 3D 宇宙
- 复杂多用户权限
- 完整家庭系统
- 完整孩子年轮
- 完整爱的小屋
- 全量 AI RAG
- 声音系统
- 历史剧场正式版
- 复杂社交系统

## 15. 隐私与权限模型

### 15.1 权限层

1. public：所有人可见
2. semiPublic：可看到存在，不看细节
3. private：仅自己
4. family：亲友
5. partner：双方可见
6. vault：保险箱
7. sealed：封存
8. silent：沉默内容

### 15.2 默认隐私规则

- 技术内容：可公开，但需检查准确性和敏感信息
- 生活内容：默认 private 或 semiPublic
- 家庭内容：默认 private
- 孩子内容：默认 vault
- 恋爱内容：默认 partner / private
- 住宅内容：默认 private
- 工作内容：默认 private
- AI 生成内容：默认草案

### 15.3 公开前检查

公开前必须检查：

- 是否包含 private 关系
- 是否包含孩子 / 家庭 / 住宅内容
- 是否包含地址、车牌、学校、证件、API Key
- 是否包含他人聊天记录
- 是否存在 AI 生成但未审核内容
- 是否缺少公开版
- 是否应清理 EXIF

### 15.4 私密内容不应进入公开构建

非常重要：

```text
公开构建只能包含 public / semiPublic 内容。
private / family / partner / vault / sealed 不应打包进前端。
```

不要把私密 JSON 放进前端再用 UI 隐藏。

## 16. AI 设计

### 16.1 AI 的角色

AI 不只是聊天框，而是世界里的角色：

- 灯塔：给访客导航
- 书记官：整理内容
- 星图师：建立关系
- 园丁：提醒种子维护
- 建筑师：生成页面结构或专题
- 守门人：检查隐私
- 记忆整理员 / 家庭档案师
- 世界之镜：帮用户看见自己

### 16.2 AI 可以做

- 摘要
- 自动标签
- 语义搜索
- 相关内容推荐
- 文档问答
- 内容重写
- 灵感整理
- 自动生成专题
- 隐私风险检查
- 月报 / 年报 / 世界日志
- 建议合并、归档、创建星座
- 根据内容生成“世界可以长出哪些新区域”
- 脱敏公开版本生成
- 路径生成
- 主题策展
- 内容债务提示

### 16.3 AI 不能自动做

- 公开私密内容
- 删除内容
- 修改原始记录
- 替你定义一段关系
- 替孩子决定公开
- 替爱人决定共同记忆
- 把梦境当成事实
- 修改核心法则
- 自动发布长文
- 自动改变权限

核心法则：

> 灯塔可以照亮道路，但不能替造物主选择道路。

### 16.4 AI 低信任模式

AI 输出分级：

| 风险等级 | 内容 |
|---|---|
| 低风险 | 标题候选、摘要草稿、格式整理 |
| 中风险 | 标签建议、关系建议、路径建议 |
| 高风险 | 隐私判断、人物总结、公开版本生成 |
| 禁止自动执行 | 公开、删除、修改原始记录、改变权限 |

### 16.5 AI 可插拔配置

```ts
type AIConfig = {
  enabled: boolean
  provider?: 'openai' | 'gemini' | 'local' | 'custom'
  features: {
    chat: boolean
    summarize: boolean
    tagging: boolean
    relationSuggest: boolean
    privacyCheck: boolean
    pathGenerate: boolean
  }
}
```

当 `enabled: false`：

- AI 灯塔显示基础导览模式
- 搜索退回本地搜索
- 推荐退回规则推荐
- 摘要使用手写摘要
- 标签使用手动标签
- 关系使用显式关系和规则关系

## 17. 无 AI 自然演化机制

无 AI 时，世界仍然可以靠规则自然演化。

### 17.1 时间规则

- 新内容自动进入最近点亮
- 长期未更新内容进入沉淀状态
- 一年前的今天可回流
- 旧项目提示归档
- 纪念日自动点亮

### 17.2 标签规则

- 相同标签内容自动靠近
- 同一项目标签自动聚合
- 同一地点自动进入地点档案
- 同一时间段自动进入时间河

### 17.3 关系规则

- 同一项目下内容自动建立 project 关系
- 同一天的照片和文字自动建立 time 关系
- 同一旅行的票据、照片、文字自动聚合
- 同一主题超过 N 个节点形成星座候选

### 17.4 生命周期规则

- fragment 默认 seed
- 被编辑 3 次后变成 sprout
- 关联超过 5 个节点后变成 growing
- 被标记精选后变成 bloom
- 项目完成后变成 fruit
- 长期不更新后变成 archive 或 relic

### 17.5 权限规则

- 家庭、孩子、恋爱、住宅默认 private / vault
- 技术文章默认 public draft
- 工作文档默认 private
- 上传图片默认清理 EXIF
- 含敏感字段的内容阻止公开

## 18. 生命周期

### 18.1 节点生命周期

```text
seed → sprout → growing → bloom → fruit → archive → relic → dormant → silent
```

中文：

- seed：种子
- sprout：新芽
- growing：生长
- bloom：盛放
- fruit：结果
- archive：归档
- relic：遗迹
- dormant：沉睡
- silent：沉默

### 18.2 生命周期对应操作

| 状态 | 操作 |
|---|---|
| seed | 补充描述 / 继续生长 / 保持沉睡 |
| sprout | 添加标签 / 选择区域 |
| growing | 建立关联 / 生成草稿 / 加入路径 |
| bloom | 发布 / 加入精选 |
| fruit | 导出 / 成册 / 代表作 |
| archive | 标记历史 / 保留参考 |
| relic | 添加复盘 / 进入失败博物馆 |
| dormant | 等待唤醒 |
| silent | 不推荐 / 不 AI / 不回流 |

### 18.3 区域生命周期

区域也有状态：

```text
draft → experimental → active → quiet → archived → sealed
```

含义：

- draft：草案
- experimental：实验中
- active：已点亮
- quiet：低光
- archived：归档
- sealed：封存

原则：

> 仪式感是邀请，不是阻碍。

### 19.3 打开地图

像展开星图：

- 地图从中心展开
- 主要区域先出现
- 远方区域以雾状轮廓显示
- 当前位置被柔光标记
- 可探索路径逐渐亮起

### 19.4 打开内容

按类型不同：

- 文章：纸页靠近并展开
- 技术文档：卷宗打开
- 项目：工坊装置被点亮
- 照片：光片落入记忆湖
- 灵感：种子浮现
- 信件：信封打开

### 19.5 阅读模式

阅读时世界应退远：

- 动效减少
- 背景变淡
- 字体清晰
- 行高舒适
- 目录稳定
- 代码块可读
- AI 摘要可折叠
- 相关内容不干扰

原则：

> 探索时浪漫，阅读时清醒。

### 19.6 保存反馈

普通“保存成功”可改为：

- 一颗新的种子已落入灵感云层。
- 这张纸页已经收进档案馆。
- 一片光影已沉入记忆湖。
- 这台装置已在工坊中点亮。
- 这封信已放入时间抽屉。

### 19.7 错误状态

404：

```text
你走到了一片尚未命名的雾区。
这里还没有稳定的路径。
```

无权限：

```text
这扇门只为特定的人打开。
里面保存着更私密的记忆。
```

AI 不可用：

```text
灯塔暂时低光运行。
地图、时间河和档案馆仍然可用。
```

搜索无结果：

```text
罗盘没有找到对应的星。
也许它还没有被命名。
```

### 19.8 空状态

不要写“暂无内容”。

写：

- 这片云层正在等待第一颗种子。
- 这座工坊还没有装置被点亮。
- 这片湖面仍然安静。
- 这里仍在雾中，未来会长出新的路径。

## 28. 世界主权设计

包括：

- 内容主权
- 数据主权
- 解释主权
- 隐私主权
- AI 主权
- 时间主权
- 审美主权
- 维护主权

### 28.1 内容主权

内容不被形式绑架。

允许：

- 无标题内容
- 未分类内容
- 未完成内容
- 不公开内容
- 只保存不解释的内容
- 未来再整理的内容

法则：

> 内容有权先以不完整的形态存在。

### 28.2 数据主权

内容不能被平台锁死。

底层格式建议：

- Markdown / MDX
- JSON / YAML
- 图片原图
- 音频原件
- relations.json
- areas.json
- paths.json
- permissions.json
- logs.json

### 28.3 解释主权

AI 不能替你定义人生意义。

法则：

> AI 可以整理事实与线索，但不能替我定义记忆的意义。

### 28.4 隐私主权

公开不是默认结果。

### 28.5 AI 主权

AI 可插拔，不越权。

### 28.6 时间主权

你决定世界的节奏。世界不制造更新压力。

### 28.7 审美主权

不被流行风格带跑。

长期审美锚点：

- 纸
- 灯
- 书桌
- 星尘
- 湖面
- 卷轴
- 罗盘
- 地图
- 玻璃瓶
- 档案馆
- 柔光
- 留白

### 28.8 维护主权

你要能控制复杂度。

硬规则：

- 一级入口不超过 7 个
- 首页主动作不超过 5 个
- 节点页强推荐不超过 3 个
- 新区域先进草案或实验区
- 没有真实内容的区域不升格
- AI 生成的新结构必须审核
- 所有诗意名称必须有现实解释
- 任何深层页面必须能回到地图
