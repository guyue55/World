# 古月浮屿｜V1 空间坐标协议

> 阶段：V1
> 性质：世界空间骨架
> 目标：在不做 3D 的前提下，先建立未来地图、星图、2.5D、WebXR 都能继承的空间协议。

## 1. 核心原则

```text
V1 不做 3D，
但必须先有稳定空间协议。
```

## 2. 协议文件

```text
data/spatial-protocol.json
```

## 3. 坐标轴

```text
x: 横向关系 / 领域展开
y: 深度 / 亲密度 / 抽象度
z: 未来可选视觉高度
```

## 4. 深度

```text
surface
middle
deep
vault
```

## 5. 规则

```text
坐标用于投影，不用于替代信息架构。
vault/deep 不得被公开 Atlas 强制展开。
2.5D/3D 只能读取公开空间索引。
同一节点可有多个投影，但主 areaId 只能有一个。
空间坐标必须允许无图形降级。
```

## 6. 最终原则

```text
先有空间协议，
再有高级地图。
```
