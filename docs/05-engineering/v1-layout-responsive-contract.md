# 古月浮屿｜V1 布局宽度与响应式契约

> 阶段：V1
> 性质：页面布局与多端适配契约
> 目标：解决“桌面端内容缩在中间、两侧空旷”和“移动端拥挤”这类结构性问题。

## 1. 布局区域

```text
reading        长文阅读，约 72ch
content        卡片、路径、归档，约 1120px
dashboard      状态、骨架、多面板，约 1280px
immersive      地图和未来视觉层，安全边距下尽量展开
full-bleed-soft 首页和世界入口，约 1440px
```

## 2. 规则

```text
长文使用 reading 宽度，避免行长过长。
卡片列表和面板页使用 content/dashboard 宽度，避免桌面过窄。
首页和地图允许更宽，但保留安全边距。
移动端优先单列，桌面端优先网格扩展空间。
任何页面不得出现非预期横向滚动。
```

## 3. 核心判断

```text
页面要有留白，
但不能让世界缩成一条窄栏。
```
