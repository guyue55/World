# 古月浮屿｜V1 页面投影密度策略

> 阶段：V1
> 性质：性能、页面体验和投影边界策略
> 目标：让页面密度可控，避免首页像数据库，状态页像日志堆。

## 1. 文件

```text
data/projection-density-strategy.json
```

## 2. 密度原则

```text
首页 curated
归档 scannable
地图 spatial-summary
节点详情 focused
路径 guided
状态/骨架 dashboard
```

## 3. 规则

```text
首页密度必须经过策展，不展示全部节点。
归档页密度以扫描效率为主。
详情页密度以阅读为主。
状态/骨架页允许信息多，但必须有摘要优先和栈边界。
超过密度阈值时优先分页、分组或折叠。
```

## 4. 核心判断

```text
页面密度要让访问者看见世界，
而不是看见数据库。
```
