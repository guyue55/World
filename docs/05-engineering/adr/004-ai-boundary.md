# ADR-004: AI 边界策略（AI 是灯塔不是太阳）

> 状态：已采纳
> 日期：2026-06

## 背景

AI 能增强内容发现和问答体验，但也可能引入成本、隐私和可靠性风险。古月浮屿需要明确 AI 的边界。

## 决策

1. **AI 只读公开上下文**：搜索和问答 API 只读取 visibility: public 的数据
2. **AI 不写入世界**：AI 不会修改 nodes.json、relations.json 等数据文件
3. **低光降级**：当 AI provider 不可用时，自动降级为静态索引 + 预设问答
4. **GET + force-static**：灯塔 API 使用 GET 方法 + force-static，避免写入信号
5. **成本控制**：每日请求上限、缓存策略、超限自动回退低光模式

## 理由

1. **隐私安全**：AI 永远不接触私密内容
2. **可靠性**：世界不依赖 AI，AI 不可用时世界仍然完整
3. **成本可控**：低光模式零成本，全光模式有预算上限

## 后果

- 正面：世界在无 AI 环境下完全可用
- 正面：AI 边界可被自动化测试验证
- 约束：灯塔 API 不能引入 POST/PUT 等写入方法
