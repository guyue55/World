# ADR-005: 内容门禁体系

> 状态：已采纳
> 日期：2026-06

## 背景

个人世界的内容容易随时间腐化：节点过短、关系断裂、标签失控、脚本膨胀。需要门禁体系来持续约束质量。

## 决策

建立三层门禁：

| 层级 | 命令 | 频率 | 检查内容 |
|---|---|---|---|
| 日常门禁 | check:daily | 每次提交 | 类型安全 + lint + 内容密度 + 关系数 + 路径数 + 事件数 + 脚本治理 |
| 边界门禁 | check:boundary | 涉及 API/权限时 | API 边界 + 权限边界 + 导入边界 |
| 发布候选 | release:local-rc | 每完成一个 Phase | 日常 + 边界 + smoke + 构建产物 |

## 理由

1. **门禁即文档**：检查脚本本身就是世界规则的可执行表达
2. **持续约束**：不依赖人工审查，每次提交自动验证
3. **可回溯**：门禁结果记录在审计报告中

## 后果

- 正面：内容质量持续达标，不会悄悄腐化
- 负面：门禁过多会降低维护效率，需要定期瘦身
- 约束：新增检查脚本需要注册到 legacy 注册表
