# ADR-002: 世界模型设计（Node -> Area -> Relation -> Path -> Timeline）

> 状态：已采纳
> 日期：2026-05

## 背景

传统个人知识库以文章列表或双向链接为核心，缺乏拓扑结构和引导路径。古月浮屿需要一种数据模型来表达世界而非文章集合。

## 决策

采用五层世界模型：

| 层级 | 含义 | 数据文件 |
|---|---|---|
| Node | 最小知识单元 | nodes.json |
| Area | 区域（节点的容器） | areas.json |
| Relation | 节点间关系星线 | relations.json |
| Path | 主题路径（节点序列） | paths.json |
| Event | 世界事件（时间线） | world-events.json |

## 理由

1. **拓扑优于列表**：区域 + 关系 + 路径构成可探索的星空，而非扁平文章流
2. **路径引导**：新访客可沿预设路径浏览，降低发现门槛
3. **时间维度**：世界事件让世界有历史感，可回望
4. **数据与渲染分离**：JSON 数据 + Markdown 正文，不耦合到渲染层

## 后果

- 正面：内容结构清晰，可被 AI 和外部工具理解
- 正面：新增节点只需编辑 JSON + 创建 Markdown，无需改代码
- 负面：五层数据需要交叉引用一致性，需要交叉引用验证脚本（T9.2）
