# ADR-003: 权限守门策略（后端控制权限，前端控制体现）

> 状态：已采纳
> 日期：2026-05

## 背景

个人数字世界需要同时服务公开访客和创作者。私密内容必须被服务端保护，不能依赖前端隐藏。

## 决策

1. **middleware 服务端拦截**：私密路由在 middleware.ts 中被拦截，返回 404
2. **API 自守门**：每个 API 路由在服务端自行检查权限（requireOwner / requirePermission），middleware 不覆盖 /api
3. **前端只控制体现**：前端根据权限控制显隐（如隐藏管理入口），但不作为安全边界
4. **公开构建只读 public 数据**：build:public-json 只提取 visibility: public 的数据

## 理由

1. **安全纵深**：即使前端被绕过，服务端仍然拒绝
2. **不硬编码**：前端权限判断来自后端返回的角色/权限，不在前端硬编码
3. **降门槛**：访客无需登录即可浏览公开世界

## 后果

- 正面：私密内容永远不会泄漏到公开构建
- 负面：API 路由需要各自实现权限检查，有重复代码风险
- 约束：API 边界注册表（25 条路由）需要持续维护
