# WorldOS 生命世界体验蓝图

> [!WARNING]
> 本文档保留为体验设计研究输入。当前可执行验收与否决项已收束到 `docs/00-overview/worldos-living-world-experience-acceptance-2026-07-11.md`；两者冲突时以后者为准。

> 状态：体验方向已收敛，尚未进入实现
> 日期：2026-07-11
> 目标：定义“世界正在运行”对用户到底意味着什么，防止再次把基础设施、背景动效或检查通过误称为最终体验。

## 1. 产品本质

WorldOS 是个人内容、关系、时间、路径和记忆的空间化操作系统，不是：

- 带星空皮肤的博客。
- 要求玩家学习复杂操作的网页游戏。
- 依赖实时 AI 才能成立的聊天产品。
- 以粒子、3D 和音乐掩盖内容不足的展示站。

访问者应在前 30 秒内形成四个判断：

1. 我进入了一个属于某个人的世界。
2. 我知道自己在哪、能去哪、为什么值得去。
3. 这里的时间、内容和足迹会留下变化。
4. 我可以安静阅读，也可以继续探索。

## 2. 世界法则

### 法则一：世界先于界面

内容、关系、时间和访问记忆先存在；页面只是某个场景对它们的投影。任何 UI 组件都不得成为第二事实源。

### 法则二：生命必须有来源

生命信号分三类：

| 类型 | 来源 | 例子 |
| --- | --- | --- |
| 真实信号 | 时间、内容、关系、路径、访问、AI 状态 | 夜晚、刚更新节点、未完成旅程 |
| 确定性变化 | 由日期 / 场景 / 会话种子生成 | 星点呼吸相位、尘埃轨迹、河面细纹 |
| 纯装饰 | 不携带语义 | 微弱噪声、材质闪烁 |

真实信号可以改变语义；确定性变化只能丰富节奏；纯装饰必须可关闭。不得用随机数决定推荐、内容状态、权限或路径进度。

### 法则三：场景拥有不同自然规律

共享的是世界信号，不共享表现模板。Atlas 的“活”是关系和远近，Timeline 的“活”是流速和回声，Archive 的“活”是光、尘与索引变化，Node 的“活”必须让位于阅读。

### 法则四：世界会呼吸，也会安静

持续运动必须低频、低对比、可暂停。进入阅读、系统 reduced-motion、用户静谧设置、页面隐藏或低性能模式后，环境可以停止，但空间语义不能消失。

### 法则五：移动必须留下上下文

场景迁移不是遮罩动画。来源对象、目标对象、迁移意图、路径进度和返回位置必须可延续。

### 法则六：AI 是灯塔，不是造物主

灯塔解释已有世界、照亮路线、承认未知。它不写事实、不改权限、不替用户做决定，也不伪装成实时 Provider。

## 3. 三种完整形态

| 形态 | 触发 | 必须保留 | 可以失去 |
| --- | --- | --- | --- |
| 静态世界 | JS 关闭、客户端失败 | 内容、关系、场景身份、全部出口、权限边界 | 连续环境、迁移编舞、声音 |
| 动态世界 | JS 可用、正常偏好 | 世界时钟、场景生命、迁移、记忆、可选声音 | 无 |
| 静谧世界 | reduced-motion / reduced-sensory / 用户选择 | 状态变化、抵达说明、操作与内容 | 大范围移动、持续粒子、声音 |

三种形态消费同一事实源。动态形态是渐进增强，不是另一个产品。

## 4. 世界时间与季节

### 4.1 时间状态

生命世界需要连续快照，而不是四个离散 class：

| 字段 | 含义 | 更新节奏 |
| --- | --- | --- |
| `now` | 当前真实时间 | 每次逻辑 tick 与恢复时 |
| `timeZone` | 世界配置时区，默认项目时区 | 配置变化时 |
| `dayProgress` | 当日 0-1 连续进度 | 每分钟 |
| `dayPeriod` | dawn / day / dusk / night | 跨边界时 |
| `season` | spring / summer / autumn / winter | 每日或配置变化时 |
| `seasonProgress` | 当前季节 0-1 进度 | 每日 |
| `worldDateKey` | 用于确定性环境种子 | 每日 |
| `sessionElapsed` | 当前会话经过时间 | 仅运行时，不持久化每帧 |

默认使用项目配置时区，不根据 LAN 访问者 IP 猜测。未来可在 `/status` 提供时间 / 季节预览，公开页面不能出现工程预览控件。

### 4.2 时间变化原则

- 日夜变化应在数分钟尺度缓慢插值，不突然切换整页滤镜。
- 四季改变材质、密度、色温、活性和声景参数，不只是 hue。
- 从后台恢复时直接跳到真实当前状态，不播放错过的昼夜过程。
- 内容阅读区的对比度永远优先于氛围。
- 首次访问不强制播放冗长“开机仪式”；用户可立即进入。

### 4.3 中文世界日历

第一版只要求四季和昼夜。架构允许未来以数据配置加入节气、纪念日和世界事件，但它们必须有真实日期、内容关系和作者审核，不在第一轮硬编码复杂历法。

## 5. 世界生命信号

| 信号 | 来源 | 能影响 | 不得影响 |
| --- | --- | --- | --- |
| 时间 | 世界时钟 | 光线、天空、材质、声景参数 | 内容事实和权限 |
| 季节 | 世界日历 | 粒子类型、颜色、河流 / 植被活性 | 页面可读性 |
| 内容新鲜度 | 事实更新时间 | 星光强弱、未读提示、归档位置 | 自动判定内容价值 |
| 关系密度 | Relation 投影 | Atlas 连线、Node 门、灯塔路线 | 私密关系泄漏 |
| 路径进度 | 本地访问记忆 | 路标、下一站、抵达印记 | 服务端事实 |
| 返回间隔 | 本地访问记忆 | 问候、继续入口、变化摘要 | 未经同意的长期画像 |
| 场景活动 | 交互状态 | 当前焦点、局部运动强度 | 其他场景后台运行 |
| AI 状态 | 服务端结果 | 灯塔亮度、措辞、来源提示 | 世界真相 |
| 偏好 | 系统 / 用户 | 动效、声音、质量 | 导航和内容可达性 |

## 6. 场景生命矩阵

### 6.1 Gateway：世界入口

**主体：** 月下浮屿、远处灯塔、通往各空间的真实方向。

**持续生命：**

- 星点以确定性相位微弱呼吸，不能是全屏高速粒子。
- 浮屿只有极慢视差或漂移，阅读和 reduced-motion 时停止。
- 灯塔按稳定周期扫过，但不会频繁照亮正文。
- 日夜改变天光和远景；季节改变雾、植被、漂浮物和声音参数。
- 新内容或未完成旅程可在对应方向出现一次低调脉冲。

**交互语法：** 观察方向、选择目的地、回到上次旅程、向灯塔问路。

**再访变化：** 显示“继续上次旅程”“世界自上次以来的变化”，不重复完整首访仪式。

**静态形态：** 七个方向和当前世界摘要仍是普通链接和文本。

### 6.2 Atlas：星图与关系空间

**主体：** 区域星域、节点星体、关系航线和可移动视野。

**持续生命：**

- 星体亮度由生命周期、新鲜度和访问状态共同投影。
- 关系线有极慢能量传递，但不让所有线同时闪烁。
- 远景星尘与前景节点分层移动；未聚焦区域降低活动度。
- 夜间更显星线，白天更显区域轮廓；四季改变星域材质而非布局。

**交互语法：** 平移、缩放、区域聚焦、节点预览、关系理由、进入地点。

**输入替代：** 拖动之外必须有区域列表、方向控制、搜索和单击聚焦。

**规模规则：** 默认只显示有意义的一级关系和当前区域；缩放 / 聚焦后再展开，禁止一次绘制全部图谱。

### 6.3 Timeline：时间河

**主体：** 一条确实流动的河，事件是停靠点，阶段是河段。

**持续生命：**

- 河面纹理沿路径稳定流动，速度低且与真实时间 / 当前焦点相关。
- 事件锚点产生局部涟漪，不是整条河不断闪光。
- 上游 / 下游远近和时间方向保持一致。
- 季节影响河色、雾、漂浮物；内容密度影响河段宽度或岸边活动，但不改事件日期。

**交互语法：** 顺流浏览、选择年份 / 阶段、停靠事件、进入地点、回看相关档案。

**输入替代：** 滚动、点击时间锚、事件列表和键盘导航均能完成同一任务。

**静态形态：** 按时间排序的事件和节点关系完整可读。

### 6.4 Archive：档案馆

**主体：** 馆厅、分区、书架 / 卷宗、索引台。

**持续生命：**

- 活动度低：光束缓慢移动、尘埃稀疏、纸张 / 柜体有微小材质变化。
- 筛选时馆内分区和索引位置发生有因果的重排。
- 新增 / 更新内容进入“新入馆”区域，访问后回到常规位置。
- 夜间更聚焦灯光，白天更开阔；季节影响纸张、窗光和环境声参数。

**交互语法：** 检索、分区、筛选、抽出卷宗、返回原架位。

**红线：** 不能让持续动画妨碍浏览、输入和长列表扫描。

### 6.5 Paths：旅程系统

**主体：** 从起点到终点的道路、路标、脚印、岔路和抵达印记。

**持续生命：**

- 未完成道路有微弱前向流动；已完成段落沉静并留下印记。
- 下一站根据真实进度亮起，不随机切换。
- 季节改变道路边缘、天色和声音，不改变路径顺序。

**交互语法：** 开始、继续、查看前后站、进入节点、完成、重置本地进度。

**再访变化：** 直接回到未完成站点，并说明上次停留位置。

### 6.6 Node：内容地点

**主体：** 与内容类型相符的房间、正文、地点护照、关系门和出口。

**持续生命：**

- 阅读时环境降到最低，只保留窗光、远景或一次性细节。
- 新 / 成长 / 沉淀状态通过房间材质、光线和护照共同表达。
- 关系门的方向和理由来自真实关系；访问过的门可留下本地足迹。
- 昼夜改变窗光，季节改变房间外部环境，不重绘正文主题。

**交互语法：** 阅读、查看护照、穿过关系门、返回来源、进入相关路径 / 时间 / 档案、问灯塔。

**红线：** Node 首要任务永远是舒适阅读，不追求持续高活动度。

### 6.7 Lighthouse：陪伴型观测站

**主体：** 灯塔、扫光、观测窗、当前位置、路线与对话。

**持续生命：**

- 光束稳定扫过可达方向；Provider thinking 只改变局部灯光节奏。
- 低光模式仍有真实的确定性路线和解释，不伪装在线模型。
- Gateway 与 Lighthouse 共享一个音乐动机的不同变奏，形成世界记忆。
- 夜间灯塔更强，白天更像观测台；季节改变海雾 / 天空和声景。

**交互语法：** 询问当前位置、寻找主题、解释关系、推荐下一站、返回来源。

**红线：** 不做全屏聊天应用，不输出无来源长文，不让 AI 改世界。

## 7. 场景活动度

| 等级 | 场景 / 状态 | 允许的持续运动 |
| --- | --- | --- |
| 高 | Gateway、Atlas 聚焦、场景迁移 | 多层但低速，可由用户暂停 |
| 中 | Timeline、Paths、Lighthouse | 一个主运动 + 少量环境运动 |
| 低 | Archive、Node 阅读 | 稀疏光线 / 材质 / 远景，正文附近不动 |
| 静 | reduced-motion、页面隐藏、错误、打印 | 无位移运动，只保留状态和对比 |

持续生命不等于每个场景同样热闹。

## 8. 迁移语法

### 8.1 四段状态

1. **离开**：来源对象响应，保留来源几何和意图。
2. **途中**：只表达方向和介质，不播放无关炫技。
3. **抵达**：目标对象先被识别，随后场景环境接管。
4. **沉淀**：焦点、滚动、路径和返回上下文稳定后才完成。

### 8.2 路线族

| 路线 | 迁移隐喻 | 上下文 |
| --- | --- | --- |
| Gateway -> Atlas | 远景星域拉近 | 目标区域 |
| Atlas -> Node | 从星体下降到地点 | 节点与来源星域 |
| Timeline -> Node | 从河面停靠上岸 | 事件与时间位置 |
| Archive -> Node | 抽出卷宗进入房间 | 筛选与架位 |
| Paths -> Node | 沿道路抵达下一站 | 路径进度 |
| 任意场景 -> Lighthouse | 光束收束到观测站 | 来源场景 / 节点 / 问题 |
| Node -> 关系 Node | 穿过关系门 | 关系理由与返回门 |
| 任意 -> Gateway | 视野拉远，回到世界入口 | 上次位置和继续入口 |

不支持或被中断时，退化为立即导航 + 抵达状态，不得阻塞。

## 9. 首访、直达与回访

### 首次进入

- 3-6 秒内完成可跳过的抵达，不做长片头。
- 首先让用户看见世界主体，再给一句中文方向提示。
- 声音保持关闭，不弹出强制授权。
- 用户选择一个方向后，仪式立即让位于探索。

### 深链直达

- 从外部直接进入 Node、Timeline 或 Path 时，不强迫返回 Gateway。
- 先显示当前地点与可读内容，再补一句“你从世界外直接抵达”。
- 提供回星图、回来源和问灯塔的出口。

### 同日再访

- 回到未完成旅程或最近地点。
- 只提示真实新增 / 更新内容，不制造假事件。

### 长期离开后返回

- 重新计算真实昼夜与季节。
- 用可展开摘要说明期间新增内容、路径变化和上次未完成事项。
- 不补播离线期间的每个环境帧。

## 10. 内容生命

节点生命来自事实，不来自视觉标签：

| 状态 | 事实依据 | 世界表达 |
| --- | --- | --- |
| seed | 新建、关系少、正文短 | 微弱新光、等待连接 |
| growing | 正文 / 关系 / 路径持续补充 | 更清晰的门和航线 |
| active | 近期更新、被路径 / 事件使用 | 稳定明亮、多个入口 |
| settled | 长期稳定、有完整关系 | 沉静材质、档案感 |
| archived | 作者明确归档 | 进入档案馆，仍可追溯 |

自动指标只能提出建议，生命周期最终由事实规则或作者确认。AI 不得自行改变状态。

新增或更新一个公开节点后，应自动影响：

- Atlas 星体与关系。
- Timeline 事件或时间痕迹。
- Archive 分区和检索。
- Paths 的站点 / 相关路线。
- Node 房间与关系门。
- Lighthouse 的公开上下文和推荐。

## 11. 声音与音乐

### 11.1 声音层次

| 层 | 职责 | 并发上限 |
| --- | --- | ---: |
| 世界底色 | 极轻、场景相关的环境层 | 1 |
| 场景动机 | Gateway / Lighthouse 等短旋律记忆 | 1 个短 cue |
| 交互反馈 | 抵达、停靠、开门、完成 | 同时 1 个，避免连发 |

### 11.2 规则

- 每次浏览会话都必须由用户手势重新启用声音。
- 场景切换用 crossfade，不硬切多个 loop。
- hidden、静音、静谧模式和组件销毁时立即停止 / suspend。
- Archive 和 Node 的声音密度最低。
- 日夜与季节优先调制音量、滤波、音色和事件密度，不为每种状态加载整首歌曲。
- 所有文件资产有来源、许可、hash、大小和用途。
- 程序化音色也必须进行真实听感审查，不能用离线频率表代替试听。

## 12. 灯塔能力阶梯

| 模式 | 能力 | 用户提示 |
| --- | --- | --- |
| low-light | 确定性搜索、关系解释、预设导览、下一站 | “低光导览，未连接实时模型” |
| local-provider | 本地模型基于公开上下文回答 | 显示本地模型和可能较慢 |
| cloud-provider | 服务端云模型回答 | 显示 Provider 状态、来源与耗时摘要 |
| unavailable | 搜索 / 服务异常 | 保留静态路线和返回出口 |

所有模式共享：中文优先、来源 ID、下一站、置信 / 不确定说明、拒答和权限过滤。

## 13. 必验用户旅程

1. 首访 Gateway -> Atlas -> Node -> 关系 Node -> Gateway。
2. 直达 Node -> Timeline -> Archive -> 返回原 Node。
3. 开始 Path -> 完成一站 -> 退出 -> 再访继续。
4. Atlas 搜索 -> 区域聚焦 -> 节点预览 -> 问灯塔。
5. Timeline 顺流 / 锚点 / 键盘三种方式定位同一事件。
6. Archive 搜索、筛选、抽卷、返回原架位。
7. 声音开启 -> 三场景迁移 -> 静音 -> hidden -> 恢复。
8. reduced-motion + reduced-sensory 完成所有主要旅程。
9. JS 关闭完成阅读、关系跳转、路径和灯塔低光入口。
10. Provider 不可用时灯塔明确回退，无假回答。

## 14. “不死板又可扩展”的真实含义

不死板：

- 同一内容因时间、季节、访问和场景而产生不同但可解释的投影。
- 装饰运动使用确定性变化，避免每次完全相同，也避免无意义随机。
- 每个场景有可组合的环境 recipe，而不是硬编码单一动画。

可扩展：

- 新内容进入事实源后自动投影。
- 新场景通过统一 Scene Module 契约接入信号、迁移、静态替代和质量矩阵。
- 新效果先成为某场景私有能力；至少三处真实重复后才提取公共 primitive。
- 规模越界按测量升级渲染和搜索，不预装“无限引擎”。

不承诺：

- 无限制节点、无限粒子或无限场景。
- AI 自动创造和维护全部世界。
- 一次 Goal 完成多年演化和真实用户回访价值。

## 15. 真实完成画面

达到“生命世界候选”时，用户应能在不看工程说明的情况下观察到：

- 停留数分钟，天光、星点、河流、灯塔和环境声仍在自然推进。
- 进入不同场景，持续生命的方式完全不同。
- 切换场景时知道从哪里离开、如何穿过、在哪里抵达。
- 返回时世界记得上次旅程，但可以一键清除。
- 新内容会在星图、时间河、档案馆、路径和灯塔中自然出现。
- 声音关闭、减少动效或 JS 关闭后，仍能完成同样的阅读和导航。
- AI 缺席时世界不死；AI 存在时只让方向更清晰。

若仍可概括为“静态大图 + 热点 + 一点循环动效”，则本目标未达到。
