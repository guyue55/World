# 古月浮屿｜视觉设计系统

> 性质：Visual Design System / UI Style Guide  
> 目标：让古月浮屿拥有长期一致、克制、温柔、有世界感的视觉体系。

---

## 1. 视觉总方向

主美学：

```text
纸上宇宙 / 书桌星河 / 自然轻科技
```

核心画面：

```text
一张安静的书桌，连接着一片无垠星河。
```

视觉气质：

- 安静
- 温柔
- 克制
- 自然
- 有星尘感
- 有纸张和档案感
- 有轻科技感
- 不赛博朋克
- 不 Web3 模板
- 不游戏启动页
- 不过度童话

## 4. 字体系统

### 4.1 中文字体方向

推荐气质：

- 清晰
- 温润
- 不过度书法化
- 长文可读

可选：

- 系统中文字体
- 思源黑体
- 霞鹜文楷用于少量标题或引文
- 不建议大面积使用装饰字体

### 4.2 字体层级

| 层级 | 用途 |
|---|---|
| Display | 首页主宣言 |
| H1 | 页面标题 |
| H2 | 区域标题 |
| H3 | 卡片标题 |
| Body | 正文 |
| Meta | 节点护照、时间、状态 |
| Caption | 注释、现实解释 |

### 4.3 排版原则

- 正文优先可读
- 技术文章行高充足
- 代码块清晰
- 首页标题可诗意
- 后台和档案视图清楚优先

## 5. 布局系统

### 5.1 页面布局

#### 首页

- Hero 大留白
- 中景区域卡片
- 精选路径
- 最近发生
- About
- Manifesto

#### 地图页

- 左侧或上方区域说明
- 中间地图
- 右侧节点 / 路径详情
- 移动端改为纵向卡片

#### 节点页

- 顶部位置
- 标题区
- 正文区
- 节点护照
- 相关节点

#### 档案馆

- 搜索栏
- 筛选器
- 列表 / 卡片
- 分页或虚拟滚动

### 5.2 空间感

使用：

- 分层背景
- 微弱星尘
- 纸张卡片
- 柔和阴影
- 轻玻璃感
- 远景淡化

不要：

- 过度毛玻璃
- 过度阴影
- 大量边框
- 卡片堆满屏幕

## 6. 组件视觉

### 6.1 NodeCard

应显示：

- 标题
- 世界标题
- 摘要
- 类型
- 区域
- 生命周期
- 标签
- 时间
- 可见性

视觉：

- 纸页感
- 柔和圆角
- 微阴影
- hover 时轻微浮起
- 不过度装饰

### 6.2 AreaCard

显示：

- 世界名
- 现实名
- 说明
- 状态
- 节点数量
- 进入按钮

### 6.3 PathCard

显示：

- 路径标题
- 适合谁
- 预计阅读时间
- 节点数量
- 简介

### 6.4 WorldEventCard

显示：

- 日期
- 事件标题
- 事件类型
- 关联区域
- 关联节点

### 6.5 NodePassport

视觉应像轻量档案卡：

- 类型
- 区域
- 生命周期
- 权限
- 来源
- AI 参与
- 下一步动作

## 8. 动效系统

### 8.1 动效原则

- 轻
- 慢
- 有语义
- 可关闭
- 不影响阅读
- 移动端降级

### 8.2 动效语义

| 动效 | 语义 |
|---|---|
| 点亮 | 发布、精选、完成 |
| 沉入 | 归档、记忆、封存 |
| 漂浮 | 灵感、未定型 |
| 展开 | 阅读、打开 |
| 收拢 | 整理、归位 |
| 暗淡 | 过期、低光、沉睡 |
| 连接 | 建立关系 |
| 波纹 | 时间、记忆、湖面 |

### 8.3 动效预算

每页只允许有限动效：

- 一个主入场动效
- 一个背景微动效
- 若干交互反馈
- 阅读区无强动效

## 9. 明暗模式

### 9.1 浅色模式

主调：

- 纸张
- 柔光
- 星尘
- 米白

适合：

- 阅读
- 白天探索
- 档案馆
- 创世台

### 9.2 深色模式

主调：

- 夜蓝
- 墨绿
- 星光
- 灯塔

适合：

- 夜间探索
- AI 灯塔
- 时间河
- 技术星域

### 9.3 不使用纯黑

深色背景建议：

```text
#111513
#121A18
#141821
```

## 10. 移动端视觉

### 10.1 移动端定位

```text
掌中浮屿
```

### 10.2 规则

- 不压缩桌面地图
- 改为纵向区域卡片
- 底部导航清晰
- 阅读优先
- 动效减少
- 背景轻量
- 卡片间距足够

---

## 11. 禁区

不要：

- 满屏粒子
- 强制开场动画
- 重型 3D
- 霓虹赛博朋克
- Web3 风格
- 商务作品集模板
- 过度卡片堆叠
- 首页信息过密
- 阅读页背景太花
- 技术页过度诗意
- 后台危险操作诗意化

---

## 12. 视觉最终原则

```text
首页要像抵达世界。
地图要像展开星图。
节点要像打开纸页。
时间河要像看见生长。
档案馆要像可靠系统。
创世台要像清醒工作台。
```

```text
高级感来自克制，
不是来自堆满效果。
```
