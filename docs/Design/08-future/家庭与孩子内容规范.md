# 古月浮屿｜家庭与孩子内容规范

> 阶段：V4  
> 性质：Family & Child Memory Policy  
> 目标：确保家庭、爱人、孩子和共同记忆被温柔保存，同时严格保护隐私、边界和未来权利。

---

## 1. 设计初衷

古月浮屿不只是技术与项目世界，也会承载生活、家庭、爱与孩子成长。

但这些内容与公开技术文章不同。它们包含：

- 他人的隐私
- 共同记忆
- 孩子的未来权利
- 家庭边界
- 真实地点
- 情感细节
- 亲密关系

因此必须有专门规范。

核心原则：

```text
共同记忆不因被我保存，就自动只属于我。
```

```text
孩子的成长可以被我保存，
但不应被我默认公开。
```

---

## 2. 家庭内容总原则

### 2.1 默认私密

家庭内容默认：

- private
- family
- vault

不默认 public。

### 2.2 公开只展示温度

可以公开：

- 抽象故事
- 不敏感照片
- 脱敏旅行片段
- 家庭氛围
- 节日感受
- 模糊化的生活片刻

不应公开：

- 真实地址
- 学校
- 作息
- 路线
- 门牌
- 证件
- 大量正脸
- 医疗信息
- 亲密争吵
- 他人聊天原文

### 2.3 家庭成员不是素材

家庭成员不是内容资产。  
他们是共同记忆的参与者。

## 3. 家庭灯火

### 3.1 定义

家庭灯火是用于记录家庭记忆的空间。

隐喻：

```text
每个家庭成员是一盏灯。
团聚时灯火靠近。
照片是光留下的痕迹。
```

### 3.2 内容范围

可记录：

- 节日
- 团圆饭
- 生日
- 家常菜
- 家人故事
- 宠物
- 老物件
- 家庭旅行
- 家庭合影
- 家庭语音
- 写给家人的信

### 3.3 权限

默认：

- family
- private

公开时仅使用 semiPublic 摘要。

### 3.4 节点模板

```md
# 家庭记忆标题

> 区域：家庭灯火  
> 权限：family / private  
> 时间：YYYY-MM-DD  
> 人物：可选，公开版需脱敏  

## 一句话

## 发生了什么

## 想保存的原因

## 是否可公开

## 公开摘要

## 相关记忆
```

## 4. 爱的小屋

### 4.1 定义

爱的小屋是亲密关系和共同记忆空间。

它不是公开秀恩爱页面，而是共同记忆的温柔容器。

### 4.2 双层结构

```text
爱的小屋
├── 外庭：公开 / 半公开摘要
└── 内室：partner / private
```

### 4.3 可记录内容

- 初遇
- 纪念日
- 第一次旅行
- 共同饭菜
- 礼物
- 电影票
- 信件
- 共同计划
- 生活片段
- 争吵与和解
- 未来愿望

### 4.4 不默认公开

以下不公开：

- 聊天原文
- 争吵细节
- 私密照片
- 对方隐私
- 真实住处
- 未经同意的共同记忆

### 4.5 公开版

公开版可以是：

```text
这是一段被温柔保存的共同旅程。
有些光可以被看见，
有些记忆只留在屋内。
```

### 4.6 共同记忆权属

字段建议：

```ts
type MemoryOwnership = {
  owner: 'self' | 'sharedWithPartner'
  requiresConsentBeforePublic: boolean
  publicVersionAllowed: boolean
}
```

## 5. 孩子年轮

### 5.1 定义

孩子年轮用于记录孩子成长，但默认不是公开内容。

核心法则：

```text
有些记忆现在由我保存，
未来应由孩子决定如何被看见。
```

### 5.2 内容范围

可记录：

- 出生
- 第一次笑
- 翻身
- 走路
- 说话
- 趣言
- 画作
- 身高体重
- 生日
- 声音
- 写给未来的信
- 家庭故事
- 成长阶段

### 5.3 默认权限

孩子内容默认：

- vault
- private

不进入：

- public build
- public search
- public AI
- public timeline
- public recommendations

### 5.4 不公开内容

绝不默认公开：

- 真实姓名
- 学校
- 班级
- 医疗信息
- 证件
- 具体住址
- 日常路线
- 作息
- 大量正脸照
- 可定位背景

### 5.5 未来权利

孩子相关内容应支持未来：

- 导出给孩子
- 由孩子选择保留 / 隐藏 / 删除
- 由孩子决定公开范围
- 查看父母写给自己的信

字段建议：

```ts
type ChildMemoryPolicy = {
  defaultVisibility: 'vault' | 'private'
  futureOwner: 'child'
  futureReviewAge?: 18
  exportableForChild: boolean
  publicAllowedByDefault: false
  aiIndexableByDefault: false
}
```

### 6.1 定义

住宅记忆记录家的空间，但必须保护安全。

可记录：

- 客厅片刻
- 厨房饭菜
- 书房深夜
- 阳台植物
- 搬家
- 家中物件

### 6.2 不公开

不公开：

- 门牌
- 小区
- 户型
- 室内结构
- 实时在家状态
- 安防细节
- 可定位窗景

### 6.3 公开方式

可以公开：

- 抽象光影
- 桌面局部
- 一杯茶
- 一本书
- 一束光

## 7. 旅行与家庭出行

### 7.1 旅行内容

可记录：

- 城市
- 风景
- 食物
- 路线
- 纪念品
- 故事
- 遗憾
- 下次想去

### 7.2 实时安全

不实时公开：

- 当前住处
- 当前房间
- 家中无人
- 精确路线
- 票据证件

### 7.3 家庭旅行

默认 private / family。  
公开版应脱敏同行人和具体住宿。

- 雨声

### 8.2 权限

---

## 9. 家庭与孩子 AI 规则

AI 默认不读取：

- 孩子原始内容
- 家庭私密照片
- 爱人信件
- 住宅内容
- 家庭聊天记录

AI 可做但需授权：

- 私密整理
- 时间线草案
- 导出纪念册草案
- 写给未来的信草案

AI 不可做：

- 自动公开
- 自动评价关系
- 自动总结孩子人格
- 自动生成公开孩子内容
- 自动把争吵解释为结论

---

## 10. 公开前检查

家庭 / 孩子 / 爱人内容公开前必须检查：

- 是否经共同记忆边界确认
- 是否包含真实姓名
- 是否包含学校
- 是否包含地址
- 是否包含作息
- 是否包含证件
- 是否包含正脸照片
- 是否含 EXIF
- 是否含他人聊天
- 是否由 AI 生成且未审核

---

## 11. 家庭与孩子内容最终原则

```text
记录不是占有。
保存不是公开。
爱不是展示。
孩子不是内容素材。
```

```text
对外可以是一束暖光，
对内才是完整的家。
```
