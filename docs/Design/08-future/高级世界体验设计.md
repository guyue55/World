# 古月浮屿｜高级世界体验设计

> 阶段：V6 候选  
> 性质：Advanced Experience Design  
> 目标：在核心系统稳定后，探索 2.5D / 3D / Canvas / 声音层 / 多尺度缩放等高级体验。

---

## 1. 设计前提

高级世界体验只能在以下前提满足后再做：

- V1 公开世界已稳定
- V2 创世台可用
- V3 AI 灯塔边界清晰
- V4 私密内容安全
- V5 导出与回望机制稳定

否则高级体验容易变成炫技，拖垮核心建设。

原则：

```text
高级体验只能锦上添花，
不能成为世界成立的前提。
```

---

## 2. 可选方向

### 2.1 Canvas 星图

用于：

- 节点关系
- 主题星座
- 技术星域
- 世界地图深潜视图

优点：

- 性能可控
- 视觉表现强
- 不必重型 3D

风险：

- 连线过多会混乱
- 移动端交互复杂

### 2.2 2.5D 浮岛地图

用于：

- Atlas 世界地图
- 区域呈现
- 主世界入口

表现：

- 纸上浮岛
- 星尘连接
- 河流与湖面
- 工坊、档案馆、灯塔等地标

优点：

- 世界感强
- 比 3D 轻

### 2.3 轻量 Three.js 观星台

仅用于可选深潜：

- 技术星域
- AI 关系图
- 世界星图
- 主题宇宙

不用于：

- 文章阅读
- 创世台
- 移动端核心流程

### 2.4 声音层

用于：

- 进入世界时的低音环境
- 记忆湖的水声
- 时间河的流动声
- 灯塔低光声
- 声音瓶播放

规则：

- 默认关闭或低音量
- 不自动强播放
- 可关闭
- 不影响阅读

### 2.5 多尺度缩放

从：

```text
书桌
  ↓
房间
  ↓
浮岛
  ↓
星域
  ↓
宇宙
```

用途：

- 表达世界层次
- 展示主干与枝叶
- 深潜探索

风险：

- 容易复杂
- 需要强引导

### 2.6 世界海报生成

用于：

- 年度世界册封面
- 主题展览海报
- 项目卡片
- 旅行纪念图

### 2.7 纪念册设计器

用于：

- 年度册
- 孩子成长册
- 旅行册
- 家庭纪念册

---

## 3. 高级体验不应做什么

不要：

- 让所有页面都 3D
- 让阅读页有强动效
- 让后台变成游戏界面
- 让移动端加载巨大场景
- 强制开场动画
- 自动播放声音
- 用视觉遮盖内容空洞
- 把世界地图做成无法理解的复杂图

---

## 4. 高级体验分层

### 4.1 默认体验

- 轻量 2D
- SVG / CSS
- 卡片和纸页
- 少量动效

### 4.2 增强体验

- Canvas 星图
- 动态地图
- 轻声音
- 关系动画

### 4.3 深潜体验

- 2.5D / 3D
- 多尺度缩放
- 观星台
- 展览模式

用户应能选择是否进入增强体验。

---

## 5. 性能预算

高级体验必须遵守：

- 不阻塞首屏
- 懒加载
- 移动端降级
- 可关闭
- 不影响 SEO
- 不影响阅读
- 不影响无障碍

---

## 6. 与 V1 的关系

V1 不做高级体验。

V1 只保留：

- 轻星尘
- 纸张质感
- 柔光
- 简单地图
- 节点卡片
- 时间河

---

## 7. 高级体验最终原则

```text
世界感来自秩序、路径、时间和记忆，
不只是来自 3D 和动效。
```

```text
如果一个效果不能帮助理解、回望、安放或创造，
它就不应进入核心体验。
```
