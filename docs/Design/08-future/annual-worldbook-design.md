# 古月浮屿｜年度世界册设计

> 阶段：V5  
> 性质：Yearbook Product Design  
> 目标：把一年中的技术、项目、灵感、生活、记忆、失败、规则变化整理为可阅读、可导出、可保存的年度册。

---

## 1. 产品定位

年度世界册是古月浮屿的年度回望系统。

它不是简单统计：

- 今年写了多少文章
- 有多少访问量
- 发了多少内容

而是回答：

- 这一年世界如何生长？
- 哪些节点从种子变成结果？
- 哪些项目完成或归档？
- 哪些灵感值得继续？
- 哪些生活片刻值得保存？
- 哪些失败留下了经验？
- 世界宪章有哪些修订？
- 下一年想点亮什么？

---

## 2. 年度册结构

### 2.1 封面

内容：

- 年份
- 世界名称
- 年度主题
- 年度一句话
- 年度代表图
- 世界状态

示例：

```text
古月浮屿 2026 年度世界册

年度主题：
从博客到个人数字世界

年度一句话：
一张书桌连接星河。
```

### 2.2 创世与世界变化

包含：

- 世界版本变化
- 新增区域
- 世界宪章修订
- 世界规则变化
- AI 灯塔状态
- 重大快照

### 2.3 技术星域

包含：

- 年度技术主题
- 代表技术文章
- 踩坑记录
- 工具链变化
- AI Agent 探索
- CLI / MCP / ADK 记录
- 技术复盘

### 2.4 产品工坊

包含：

- 项目进展
- 新工具
- Demo
- 失败项目
- 复盘
- 代表作品
- 下一步

### 2.5 灵感云层

包含：

- 年度灵感种子
- 成长为文章 / 项目的灵感
- 仍在漂浮的种子
- 值得明年继续的主题

### 2.6 记忆湖

包含：

- 生活片刻
- 旅行光影
- 家庭摘要
- 私密内容只进入私密年度册
- 公开版只展示脱敏摘要

### 2.7 时间河

包含：

- 月度时间线
- 关键世界事件
- 一年前的自己
- 旧内容回流

### 2.8 失败博物馆

包含：

- 失败项目
- 未完成计划
- 错误判断
- 踩坑
- 留下的经验
- 是否复活

### 2.9 年度关键词

例如：

- AI Agent
- CLI
- 个人数字世界
- 无 AI 自然演化
- 记忆湖
- 隐私守门
- 世界宪章

### 2.10 下一年计划

包括：

- 想继续生长的节点
- 想归档的项目
- 想点亮的新区域
- 想暂停的内容
- 想守护的记忆
- 想完成的导出

## 3. 年度候选机制

节点可标记：

```ts
featured?: {
  yearbookCandidate?: boolean
  representative?: boolean
  timelineKey?: boolean
}
```

自动候选来源：

- featured=true
- lifeStage=fruit
- timelineKey=true
- 关联节点多
- 年内更新多
- 标记为“值得回望”
- 项目完成
- 失败复盘
- 重要生活片刻

---

## 4. 年度册类型

### 4.1 公开年度册

只包含：

- public 节点
- semiPublic 摘要
- 公开项目
- 公开技术文章
- 世界变化
- 脱敏生活片段

### 4.2 私密年度册

包含：

- private 生活
- 家庭
- 爱的小屋
- 私密反思
- 情绪记录
- 时间胶囊摘要

仅自己可见。

### 4.3 家庭年度册

包含：

- family 内容
- 家庭灯火
- 节日
- 旅行
- 家庭照片

### 4.4 孩子成长年册

包含：

- child vault 内容
- 年度成长记录
- 声音
- 画作
- 趣言
- 写给未来的信

默认不公开。

### 4.5 技术年度册

包含：

- 技术文章
- 项目
- 工具
- 踩坑
- 技术路线

## 5. 生成流程

```text
收集候选节点
  ↓
按区域分组
  ↓
按月份生成时间线
  ↓
选择代表节点
  ↓
生成章节
  ↓
隐私过滤
  ↓
AI 生成草案摘要
  ↓
人工审核
  ↓
导出 Markdown / PDF / HTML
```

---

## 6. AI 参与规则

AI 可以：

- 生成年度摘要草案
- 提取关键词
- 按月份整理事件
- 生成章节导语
- 提供复盘问题
- 推荐代表节点

AI 不可：

- 读取 vault，除非明确授权且不公开
- 自动公开年度册
- 把私密内容写入公开年度册
- 替你定义某段记忆意义
- 自动总结孩子人格

---

## 7. 年度册数据模型

```ts
type Yearbook = {
  id: string
  year: number
  title: string
  theme?: string
  visibility: 'public' | 'private' | 'family' | 'vault'
  sections: YearbookSection[]
  generatedBy?: 'manual' | 'ai-assisted'
  reviewed: boolean
  createdAt: string
  exportedAt?: string
}

type YearbookSection = {
  id: string
  title: string
  description?: string
  nodeIds: string[]
  eventIds?: string[]
}
```

---

## 8. 导出格式

支持：

- Markdown
- PDF
- 静态 HTML
- JSON
- ZIP
- 打印友好版本

导出前检查：

- visibility
- private 泄露
- AI 未审核
- 图片 EXIF
- 附件范围

## 9. 年度仪式

### 9.1 年末仪式

文案：

```text
时间河即将进入新的年份。
有一些星光值得装订成册。
```

### 9.2 生成后

文案：

```text
这一年的世界已经被装订。
你可以回望，也可以继续生长。
```

---

## 10. 年度册最终原则

```text
年度册不是 KPI 报告，
而是一年世界生长的截面。
```

```text
最新不等于重要。
访问量不等于价值。
真正值得装订的，是那些仍会在未来发光的节点。
```
