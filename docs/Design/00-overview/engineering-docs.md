# 古月浮屿｜补充工程文档集

> 说明：前面已经生成了世界宪章、世界手册、世界规则、AI 守则、产品总览、PRD、信息架构、交互设计、视觉系统、内容规范、技术架构和路线图。  
> 本组文档只补真正必要的工程落地缺口，避免继续制造大量冗余文档。

## 为什么还需要这组文档

已有文档解决了：

- 世界是什么
- 产品为什么存在
- V1 做什么
- 视觉与交互如何设计
- 内容如何组织
- 技术栈如何选择

但还缺少几个偏工程实施和质量保障的关键文档：

1. **系统设计文档**  
   把“世界协议、数据、页面、构建、搜索、权限、AI”连成完整系统。

2. **数据模型与内容协议文档**  
   明确每个 JSON / Markdown / Node / Area / Relation / Path / Event 的字段和校验规则。

3. **接口与集成设计文档**  
   为未来接入 AI、搜索、导入器、GitHub、图片、创世台、导出器预留边界。

4. **安全隐私与发布守门文档**  
   这是本项目的高优先级，尤其涉及家庭、孩子、住宅、私密内容、公开构建隔离。

5. **测试验收与质量保障文档**  
   确保 V1 真正可用，不只是看起来漂亮。

6. **开发任务拆解文档**  
   把 V1 拆成可执行任务，方便进入开发。

## 暂不建议再单独生成的文档

为了避免冗余，以下暂不单独生成：

- 用户增长文档：当前不是流量产品
- 商业化文档：当前不是商业 SaaS
- 运营文档：当前核心是个人世界与内容沉淀
- 社区规范：V1 不做社区
- 多角色权限后台说明：V1 仅模型预留
- 完整 AI RAG 技术方案：V3 再细化
- 复杂 3D 场景设计：V1 不做
- 多租户系统设计：不符合当前定位
