# 古月浮屿｜术语词典

> 性质：世界语言词典 / 命名规范 / 现实解释表
> 作用：避免世界概念越来越多后产生混乱。所有诗意名称都必须有现实解释。

---

## 1. 命名总则

### 1.1 双命名系统

每个重要概念都应有：

- 世界名：用于体验和氛围
- 现实名：用于理解和管理

例如：

```text
记忆湖
生活照片、日常片刻、旅行和家庭记忆
```

### 1.2 命名纪律

- 一级区域：稳定、短、好记
- 二级区域：可诗意，但要有现实解释
- 节点标题：现实标题优先
- 世界标题：可作为副标题
- URL：现实清晰，不要过度诗意
- 危险操作：必须清晰说明现实后果

---

## 2. 核心世界词

### 古月浮屿

- 类型：World
- 现实名：个人数字世界
- 说明：用于安放技术、项目、灵感、生活、记忆与未来的长期系统。
- 核心句：一张书桌连接星河。

### 创世原点

- 类型：Area
- 现实名：世界入口与核心说明
- 作用：解释世界是什么、为什么存在、如何进入。
- 默认权限：public

### 时间河

- 类型：Area
- 现实名：时间线与世界生长记录
- 作用：记录节点、区域、法则与世界自身如何在时间中生长。
- 包含：世界年表、节点成长史、今天历史上、时间胶囊、时光回声。

### 技术星域

- 类型：Area
- 现实名：技术文章与工程实践
- 作用：承载 AI Agent、CLI、MCP、ADK、系统排查、工程复盘。
- 默认权限：public draft

### 产品工坊

- 类型：Area
- 现实名：项目、工具与产品实验
- 作用：承载项目、工具、原型、Demo、README、复盘、失败项目。
- 隐喻：装置、草图、零件、试运行、点亮。

### 灵感云层

- 类型：Area
- 现实名：碎片想法与未来可能性
- 作用：承载一句话想法、草稿、设计概念、未命名种子。
- 默认权限：private

### 记忆湖

- 类型：Area
- 现实名：生活照片与日常记忆
- 作用：保存生活片刻、旅行、家庭、日常照片等内容。
- 默认权限：private

### 档案馆

- 类型：Area
- 现实名：文章、文档与归档内容
- 作用：现实视图、列表、筛选、搜索、旧项目、文档。
- 默认权限：public / private 按节点决定

### AI 灯塔

- 类型：System / Area
- 现实名：搜索、问答、导览与整理助手
- 无 AI 模式：预设导览 + 本地搜索 + 精选路径
- AI 增强：语义问答、摘要、路径生成、隐私检查
- 法则：AI 是灯塔，不是太阳。

## 3. 生活与记忆词

### 爱的小屋

- 类型：Private Area
- 现实名：亲密关系与共同记忆空间
- 结构：外庭 / 内室
- 默认权限：partner / private
- 说明：记录纪念日、共同旅行、信件、礼物、共同生活。

### 家庭灯火

- 类型：Private / Family Area
- 现实名：家庭记忆空间
- 默认权限：family / private
- 说明：每个家庭成员是一盏灯，团聚时灯火靠近。

### 年轮树

- 类型：Vault Area
- 现实名：孩子成长记录
- 默认权限：vault / private
- 说明：记录孩子成长，但未来应由孩子决定如何被看见。

### 旅行远方

- 类型：Area
- 现实名：旅行记录与路线记忆
- 默认权限：private / semiPublic
- 包含：路线、照片、票据、纪念品、城市印象。

### 声音瓶

- 类型：Object / Area
- 现实名：声音记忆
- 默认权限：private
- 内容：孩子声音、旅行环境声、家人笑声、雨声、海浪、语音日记。

### 安静角落

- 类型：Private Area
- 现实名：沉默内容与不想解释的记忆
- 默认权限：silent / private
- 规则：不推荐、不 AI、不主动回流。

---

## 4. 时间与回望词

### 时光回声

- 类型：Area / Mode
- 现实名：基于历史内容的自我回望空间
- 所属：时间河
- 无 AI 模式：按年份、主题、项目聚合旧内容
- AI 增强：基于历史内容生成回声式对话
- 必须提示：AI 回声不是过去真实意识。

### 过去的我

- 类型：Mode
- 现实名：历史自我内容聚合与对话
- 所属：时光回声 / 时间河
- 默认权限：private

### 历史剧场

- 类型：Experimental Area
- 现实名：历史资料与沉浸式时间线
- 规则：必须区分史实、解释、想象、AI 生成。

### 时间胶囊

- 类型：Node / Container
- 现实名：指定时间或条件打开的信件 / 内容
- 字段：收件人、开启时间、开启条件、是否允许 AI 总结、是否可导出。

### 世界年表

- 类型：Timeline View
- 现实名：世界自身的版本与成长记录
- 内容：创世想法、宪章修订、区域点亮、V1 发布等。

## 5. 创作与项目词

### 星尘港

- 类型：Inbox
- 现实名：收集箱 / 未安放之地
- 作用：接收所有新内容。
- 法则：先接住，再整理。

### 创世台

- 类型：Admin / Workbench
- 现实名：后台管理与世界维护工作台
- 功能：记录、整理、权限、规则、AI 草案、导出、快照。

### 失败博物馆

- 类型：Area
- 现实名：失败项目、废案、踩坑与复盘
- 法则：失败不是垃圾，而是世界地层。

### 实验荒原

- 类型：Experimental Area
- 现实名：新功能试验区
- 作用：新想法先试验，不污染主世界。

### 梦境图书馆

- 类型：Future / Experimental Area
- 现实名：未实现想法、幻想产品、未来设定
- 默认权限：private / semiPublic

## 6. 数据与协议词

### Node

- 世界名：节点
- 现实名：内容基本单位
- 说明：文章、项目、灵感、照片、文档、信件、地点、物件都可成为节点。

### Area

- 世界名：区域 / 地貌
- 现实名：内容归属空间
- 说明：技术星域、产品工坊、记忆湖等都是 Area。

### Entity

- 世界名：实体
- 现实名：世界中的视觉表现
- 说明：星、纸、门、灯、湖面光片、装置、卷宗等。

### Projection

- 世界名：投影
- 现实名：同一节点在不同区域或视图中的呈现
- 说明：一个 Node 可以投影到地图、时间河、档案馆、路径和首页。

### Relation

- 世界名：星线 / 关系
- 现实名：节点之间的连接
- 类型：topic、time、project、place、person、inspired、derived、implemented 等。

### Path

- 世界名：路径 / 星路
- 现实名：精选探索路线
- 作用：帮助访客理解世界。

### Rule

- 世界名：世界法则
- 现实名：自动演化规则
- 作用：无 AI 时仍能推动世界自然演化。

### Permission

- 世界名：门 / 权限
- 现实名：可见性和访问边界
- 类型：public、semiPublic、private、family、partner、vault、sealed、silent。

### WorldEvent

- 世界名：世界事件
- 现实名：世界变化日志
- 内容：节点创建、区域点亮、规则触发、AI 建议采纳等。

### Snapshot

- 世界名：快照
- 现实名：世界状态备份
- 作用：回滚、发布、归档。

## 8. 状态词典

### 节点状态

| 英文 | 中文 | 含义 |
|---|---|---|
| seed | 种子 | 只有想法或一句话 |
| sprout | 新芽 | 初步展开 |
| growing | 生长 | 内容正在完善 |
| bloom | 盛放 | 可公开或可阅读 |
| fruit | 结果 | 成为成果或代表内容 |
| archive | 归档 | 停止活跃但保留 |
| relic | 遗迹 | 旧内容但有历史价值 |
| dormant | 沉睡 | 暂时不处理 |
| silent | 沉默 | 不推荐、不 AI、不回流 |

### 区域状态

| 状态 | 含义 |
|---|---|
| draft | 草案 |
| experimental | 实验中 |
| active | 已点亮 |
| quiet | 低光 |
| archived | 归档 |
| sealed | 封存 |

## 10. 文案词典

### 保存成功

- 一颗新的种子已落入灵感云层。
- 这张纸页已经收进档案馆。
- 一片光影已沉入记忆湖。
- 这台装置已在工坊中点亮。
- 这封信已放入时间抽屉。

### 404

```text
你走到了一片尚未命名的雾区。
这里还没有稳定的路径。
```

### 无权限

```text
这扇门只为特定的人打开。
里面保存着更私密的记忆。
```

### AI 不可用

```text
灯塔暂时低光运行。
地图、时间河和档案馆仍然可用。
```

### 空状态

- 这片云层正在等待第一颗种子。
- 这座工坊还没有装置被点亮。
- 这片湖面仍然安静。
- 这里仍在雾中，未来会长出新的路径。

## 11. 口号层级

### 短口号

```text
一张书桌连接星河。
```

### 中口号

```text
这里安放技术、生活、灵感、爱与记忆，
让它们在时间中自然生长。
```

### 长宣言

```text
这不是一个传统博客，而是一个正在生长的个人数字世界。
我把技术、项目、灵感、生活、旅行、家庭与未来，一点点安放在这里。
有些内容已经成形，有些仍是种子，有些只在世界深处安静发光。
```

## 12. 词典最终原则

```text
诗意负责吸引。
解释负责理解。
协议负责稳定。
规则负责生长。
```

所有新词加入世界前，都应回答：

- 它的世界名是什么？
- 它的现实名是什么？
- 它属于哪类对象？
- 它归属哪个区域？
- 它默认权限是什么？
- 无 AI 时如何运行？
- AI 可增强什么？
