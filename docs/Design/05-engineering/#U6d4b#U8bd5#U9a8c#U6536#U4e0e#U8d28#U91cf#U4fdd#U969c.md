# 古月浮屿｜测试验收与质量保障

> 性质：QA / Acceptance Criteria / Quality Gate
> 目标：确保 V1 不只是“看起来好”，而是真正可用、清晰、安全、可维护。

---

## 1. 测试总原则

V1 必须通过五类测试：

1. 产品体验测试
2. 信息架构测试
3. 内容与数据测试
4. 安全隐私测试
5. 技术构建测试

---

## 2. 产品体验测试

### 2.1 首页测试

验收：

- 打开后不像普通博客
- 10 秒内知道这里是什么
- 主按钮不超过 5 个
- 有世界宣言
- 有主区域说明
- 有精选路径
- 有最近发生
- 移动端可读

失败信号：

- 像文章列表
- 首屏概念太多
- 入口过多
- 用户不知道点哪里

### 2.2 世界地图测试

验收：

- 主区域清晰
- 每个区域有现实解释
- 初始不展示过多节点
- 点击区域可进入或查看说明
- 可返回首页和档案馆
- 移动端可用

失败信号：

- 地图复杂得像蜘蛛网
- 只有诗意没有解释
- 无法返回
- 移动端点不动

### 2.3 节点页测试

验收：

- 标题清楚
- 正文可读
- 有节点护照
- 有返回地图
- 相关节点不干扰
- 代码块可读
- 移动端舒适

失败信号：

- 动效影响阅读
- 页面不知道属于哪里
- 相关内容过多
- 没有更新时间

### 2.4 时间河测试

验收：

- 展示世界事件
- 不只是文章发布时间
- 事件可关联节点
- 能体现世界生长

### 2.5 档案馆测试

验收：

- 可搜索
- 可筛选
- 可排序
- 能快速找到内容
- 现实模式清楚

### 2.6 AI 灯塔低光测试

验收：

- 无 AI 时不报错
- 有低光文案
- 有预设路径
- 有本地搜索
- 不显示失效输入框

---

## 3. 信息架构测试

### 3.1 反迷路测试

每个页面检查：

- 我在哪里
- 这里是什么
- 我能做什么
- 我能去哪
- 如何回地图
- 如何搜索
- 如何切换档案视图

### 3.2 路径测试

V1 至少有：

- 第一次来到古月浮屿
- 技术与 AI
- 这个世界如何被创造

每条路径检查：

- 节点存在
- 顺序合理
- 每一步有说明
- 不引用私密节点
- 完成后有下一步

---

## 4. 内容质量测试

### 4.1 public 节点

必须有：

- title
- summary
- areaId
- visibility
- lifeStage
- createdAt
- tags

### 4.2 技术文章

检查：

- 背景
- 环境
- 问题
- 步骤
- 结果
- 适用版本
- 最后更新

### 4.3 项目节点

检查：

- 项目是什么
- 解决什么问题
- 状态
- 功能
- 技术栈
- 下一步

### 4.4 灵感节点

允许短，但必须有：

- 一句话
- 来源
- 生命周期
- 权限

---

## 5. 数据一致性测试

自动检查：

- node id 唯一
- slug 唯一
- areaId 存在
- contentPath 存在
- relation from/to 存在
- path nodeSlugs 存在
- world-event nodeIds 存在
- date 格式正确
- visibility 合法
- lifeStage 合法
- type 合法

---

## 6. 安全隐私测试

必须检查：

- private 不进入 public build
- vault 不进入 public build
- public search index 不含 private
- public path 不引用 private
- AI 未审核内容不 public
- 图片 EXIF 已检查
- API Key / Token 未出现
- 地址、车牌、证件未出现
- 孩子内容未公开
- 住宅细节未公开

---

## 7. AI 测试

### 7.1 无 AI

- Ask 低光可用
- 本地搜索可用
- 预设路径可用
- 页面不报错

### 7.2 有 AI（未来）

检查：

- AI 只读授权内容
- AI 输出进入草案
- AI 不直接改 original
- AI 不改 visibility
- AI 内容有标识
- AI 回答说明依据

---

## 8. 可访问性测试

检查：

- 颜色对比
- 图片 alt
- 键盘导航
- 可关闭动效
- 文本可缩放
- 移动端触控区域足够
- 语义结构合理

---

## 9. 性能测试

目标：

- 首屏不被大图和粒子拖慢
- 移动端滚动流畅
- 搜索响应快
- 图片懒加载
- 动效不阻塞
- JS 包不过大

避免：

- V1 使用重型 Three.js
- 满屏粒子
- 自动播放音频
- 巨大未压缩图片

---

## 10. SEO 与公开发布测试

检查：

- title
- description
- og:image
- sitemap
- robots
- canonical
- RSS / JSON Feed 可选
- public 页面可被索引
- private 页面不在 sitemap

## 12. V1 最终验收 20 问

1. 打开首页是否不像普通博客？
2. 10 秒内是否知道这里是什么？
3. 是否知道从哪里开始？
4. 地图是否清楚？
5. 区域是否有现实解释？
6. 任意节点是否能回到地图？
7. 阅读页是否舒服？
8. 档案馆是否能找到内容？
9. 时间河是否体现世界生长？
10. AI 不可用时是否仍可探索？
11. 移动端是否可读？
12. 是否有空状态文案？
13. 是否有错误状态文案？
14. public 节点是否有 summary？
15. private 是否没进 public build？
16. 搜索索引是否只含 public？
17. 是否有三条精选路径？
18. README 是否解释清楚？
19. 是否有第一批真实内容？
20. 是否愿意自己继续回来记录？

---

## 13. 质量最终原则

```text
漂亮不是验收标准。
可理解、可使用、可维护、可守护，才是验收标准。
```
