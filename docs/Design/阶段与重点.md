## 阶段与重点

### 第一阶段：世界骨架 / 地基阶段

重点是：**让古月浮屿作为一个“个人数字世界”站起来**。

核心内容：

```text
世界观基础
信息架构
世界节点
路径系统
时间河
Atlas 空间入口
AI 灯塔边界
公开 / 私密边界
工程骨架
数据协议
基础页面
基础质量门禁
```

可以理解为：

```text
第一阶段 = 世界骨架、协议地基、工程地基
```

### 第二阶段：真实体验产品化阶段

重点是：**从协议和骨架，转向真实可访问、可浏览、可体验的产品页面**。

核心内容：

```text
首页产品化
导航与交互修复
内容与档案馆产品化
节点阅读体验
路径导览
Atlas 空间入口
时间河产品化
AI 灯塔产品化
状态骨架产品化
阅读舒适度
Lint / QA 收口
```

可以理解为：

```text
第二阶段 = 让世界从“设计稿/骨架”变成“可体验的网站”
```

### 第三阶段：世界能力增长阶段

重点是：**让古月浮屿从可浏览网站，发展成可持续生长的个人世界系统**。

第三阶段已经完成了三条核心线：

```text
第三阶段规划线：4 / 4
第三阶段实现线：4 / 4
第三阶段体验整合线：4 / 4
```

核心内容：

```text
主题展览
AI 建议工作台
导出中心
第三阶段入口聚合
体验整合
路由整合
QA 扩展
内容生长规划
AI 灯塔增强规划
运营导出规划
```

可以理解为：

```text
第三阶段 = 内容生长、AI 辅助、导出运营、体验整合
```

### 第三阶段之后 / 第四阶段之前：发布候选与验收收口阶段

这部分也已经完成：

```text
真实环境补证准备线：4 / 4
真实环境补证执行辅助线：4 / 4
真实执行落地线：4 / 4
发布干跑与交接线：4 / 4
发布候选冻结线：4 / 4
发布候选验收证据线：4 / 4
第四阶段前最终封版线：4 / 4
```

重点是：**不是继续加功能，而是把 V1 进入发布候选状态前后的证据、门禁、交接、风险、冻结、签收流程补齐**。

可以理解为：

```text
第三阶段后收口 = 发布候选准备、验收证据、交接门禁、第四阶段入口判断
```

## 后面还有几个阶段？

按照目前项目演进，建议后面至少还有 **3 个大阶段**。

## 第四阶段：真实内容生长与运营阶段

重点：**让世界真正长内容、长栏目、长展览，而不是继续堆结构。**

建议重点：

```text
真实文章内容体系
主题展览真实内容填充
首页内容真实化
节点内容扩写
时间河真实事件
路径导览真实阅读体验
AI 建议工作台真实辅助流程
导出中心真实使用
内容运营节奏
```

一句话：

```text
第四阶段 = 让古月浮屿真正“有内容、有生命、有运营节奏”
```

## 第五阶段：私密世界 / 个人档案阶段

重点：**建设公开世界背后的私密档案、家庭内容、时间胶囊、传承系统。**

建议重点：

```text
私密档案系统
家庭与孩子内容
时间胶囊
年度世界册
私密 / 半公开 / 家庭 / 伴侣 / vault 分层
导出与传承
长期记忆整理
敏感内容边界
```

一句话：

```text
第五阶段 = 从公开博客走向私人数字档案馆
```

## 第六阶段：AI 深度协作与世界智能阶段

重点：**让 AI 不只是建议，而是成为可控、可审计、可解释的世界维护助手。**

建议重点：

```text
AI 读世界协议
AI 内容建议
AI 路径推荐
AI 摘要与导览
AI 展览策划
AI 维护提醒
AI 风险提示
AI 不越权机制
人工确认工作流
```

注意边界仍然是：

```text
AI 是灯塔，不是太阳。
AI 可以照亮、建议、解释；
不能自动发布、删除、改可见性、绕过私密边界。
```

一句话：

```text
第六阶段 = AI 成为世界灯塔和维护助手
```
