# 古月浮屿｜PRD 产品需求文档

> 版本：v0.1  
> 范围：V1 核心版本 + V2/V3 延展规划  
> 目标：把个人数字世界从理念转化为可执行产品需求。

---

## 1. 产品概述

### 1.1 产品名称

古月浮屿

### 1.2 产品类型

个人数字世界 / 个人创作与记忆系统 / 世界化博客 / AI 辅助内容宇宙。

### 1.3 产品目标

V1 目标：

```text
建立一个不像普通博客的、可运行的个人数字世界核心版本。
```

要求：

- 有世界入口
- 有世界地图
- 有节点系统
- 有时间河
- 有档案馆
- 有精选路径
- 有 AI 灯塔低光模式
- 有世界宪章
- 有第一批真实内容
- 无 AI 时可运行
- 移动端可用

---

## 2. 用户角色

### 2.1 造物主

即古月本人。

目标：

- 快速记录
- 整理内容
- 管理世界
- 设置权限
- 维护路径
- 审核 AI 草案
- 回望过去
- 导出备份

### 2.2 旅人 / 访客

公开访问者。

目标：

- 理解这里是什么
- 认识古月
- 看技术文章
- 看项目和作品
- 沿路径探索
- 搜索公开内容
- 不迷路

### 2.3 深潜者

对世界设计、AI、个人数字世界感兴趣的人。

目标：

- 理解世界如何被创造
- 阅读世界宪章
- 查看世界数据模型
- 沿深潜路径探索

### 2.4 亲友 / 家庭成员

未来可能授权访问部分内容。

目标：

- 查看家庭 / 旅行 / 生活片段
- 不接触不该公开的内容
- 可进入特定私密区域

### 2.5 AI

作为系统辅助角色。

目标：

- 读取授权内容
- 生成建议
- 进入草案
- 不越权

| 模块 | 说明 | 优先级 |
|---|---|---|
| 首页 | 世界入口、宣言、主路径 | P0 |
| 世界地图 | 主区域展示与导航 | P0 |
| 节点详情 | 统一内容页 | P0 |
| 时间河 | 世界事件与节点生长 | P0 |
| 档案馆 | 搜索、筛选、现实视图 | P0 |
| 精选路径 | 第一次来、技术与 AI、世界创造路径 | P0 |
| 世界宪章 | 理念与法则 | P0 |
| 关于古月 | 个人介绍与世界说明 | P0 |
| AI 灯塔低光 | 无 AI 导览入口 | P1 |
| 基础搜索 | 本地搜索 / 标签筛选 | P1 |

### 3.2 V1 不实现

| 模块 | 推迟原因 |
|---|---|
| 完整 3D 宇宙 | 成本高，影响性能 |
| 完整 AI RAG | AI 不是生命层 |
| 多用户权限 | V1 先模型预留 |
| 爱的小屋正式版 | 私密系统需更严谨 |
| 孩子年轮正式版 | 权限和伦理要求高 |
| 历史剧场正式版 | 事实/想象边界复杂 |
| 声音系统 | 非核心 |
| 社交系统 | 与世界定位不符 |
| 大后台 | V1 静态优先 |

## 4. 用户故事

### 4.1 访客第一次访问

作为访客，我希望进入首页后能立刻知道这里不是普通博客，并能选择一条路径继续探索。

验收：

- 首页首屏有清晰宣言
- 入口按钮不超过 5 个
- 有“第一次来到古月浮屿”路径
- 可返回地图

### 4.2 访客查看技术内容

作为技术访客，我希望快速找到 AI、CLI、MCP、ADK 等相关内容。

验收：

- 有技术与 AI 路径
- 技术星域有现实解释
- 档案馆可按技术标签筛选
- 节点页代码和正文可读

### 4.3 访客理解世界设计

作为深潜者，我希望理解这个世界如何从博客演化为个人数字世界。

验收：

- 有“这个世界如何被创造”路径
- 有世界宪章
- 有世界地图说明
- 有无 AI 自然演化原则节点

### 4.4 造物主添加灵感

作为造物主，我希望能快速保存一句想法，而不用马上完整分类。

验收：

- 可进入收集箱 / 星尘港
- fragment 默认 seed
- 默认 private
- 后续可安放到区域

### 4.5 造物主整理技术记录

作为造物主，我希望把一次排查记录整理成技术节点，并关联项目和标签。

验收：

- Node 支持 article / project / fragment
- 节点有 tags、areaId、lifeStage
- 相关节点可显示

### 4.6 造物主公开内容

作为造物主，我希望公开前能确认摘要、权限、隐私风险。

验收：

- public 节点必须有 summary
- visibility 字段存在
- 私密内容不进入公开构建
- AI 生成内容需 reviewed

### 4.7 无 AI 情况

作为用户，我希望没有配置 AI 时，网站仍然完整可用。

验收：

- Ask 页面显示低光模式
- 地图、时间河、档案馆可用
- 搜索可用
- 精选路径可用

## 5. 功能需求

### 5.1 首页

#### 需求

首页必须完成：

- 传达世界气质
- 解释产品定位
- 展示主区域
- 展示精选路径
- 展示最近发生
- 引导地图 / 时间河 / 灯塔

#### 内容结构

1. Hero
2. What is this world
3. Areas
4. Paths
5. Recent World Events
6. About Guyue
7. Manifesto CTA

#### 验收

- 首屏不出现普通文章列表
- 主按钮不超过 5 个
- 移动端首屏可读
- 有明确进入路径

---

### 5.2 世界地图 Atlas

#### 需求

- 显示主区域
- 每个区域有世界名、现实名、解释
- 支持进入区域或查看相关节点
- 支持世界视图和目录视图
- 支持返回首页和档案馆

#### V1 区域

- 创世原点
- 时间河
- 技术星域
- 产品工坊
- 灵感云层
- 记忆湖
- 档案馆
- AI 灯塔

#### 验收

- 地图不展示过多内容
- 首次用户能理解区域含义
- 每个区域有现实解释
- 移动端可用

---

### 5.3 节点详情页

#### 需求

展示任意 Node。

包括：

- title
- worldTitle
- type
- area
- summary
- content
- tags
- visibility
- lifeStage
- source
- layer
- AI 参与状态
- createdAt / updatedAt
- relatedNodes
- paths

#### 验收

- 技术文章可读
- 返回路径清晰
- 节点护照存在
- 相关节点不超过 3 个强推荐
- 移动端排版舒适

---

### 5.4 时间河

#### 需求

展示 WorldEvent。

事件类型：

- node-created
- node-updated
- node-published
- node-archived
- area-created
- principle-created
- rule-triggered
- snapshot-created

#### 验收

- 时间河不是普通文章列表
- 能看到世界在生长
- 每个事件可关联节点或区域
- 支持按年份 / 类型筛选

---

### 5.5 档案馆

#### 需求

提供高效查找。

筛选：

- type
- area
- tag
- lifeStage
- visibility
- createdAt
- updatedAt

搜索：

- title
- summary
- tags
- content index

#### 验收

- 能快速找到内容
- 不依赖世界化地图
- 可作为现实模式兜底

---

### 5.6 精选路径

#### 需求

V1 至少三条：

1. 第一次来到古月浮屿
2. 技术与 AI 探索路径
3. 这个世界如何被创造

路径字段：

- title
- description
- audience
- estimatedTime
- nodeSlugs
- nextPaths

#### 验收

- 访客可顺着路径理解世界
- 每一步有解释
- 路径结束有下一步建议

---

### 5.7 AI 灯塔低光模式

#### 需求

无 AI 时提供导览。

内容：

- 预设问题
- 精选路径
- 本地搜索
- 随机漂流

#### 文案

```text
灯塔当前处于低光模式。
我不能实时对话，但已经为你准备了几条路径。
```

#### 验收

- 不配置 AI 时不报错
- 不出现失效聊天框
- 仍然可导航

## 6. 非功能需求

### 6.1 性能

- 首屏快速显示
- 动效不阻塞内容
- 图片懒加载
- 移动端流畅
- AI 请求不阻塞页面

### 6.2 可访问性

- 语义 HTML
- 文字对比足够
- 动效可关闭
- 图片 alt
- 键盘可导航
- 不依赖颜色表达唯一状态

### 6.3 隐私

- 私密数据不进入公开构建
- public build 只读取 public / semiPublic 摘要
- 图片公开前应清理 EXIF
- AI 不读取 vault

### 6.4 可迁移

- 内容 Markdown
- 元数据 JSON
- 媒体文件独立存储
- 路径、关系、规则可导出

## 9. V1 验收标准

1. 打开首页不像普通博客
2. 用户 10 秒内知道这里是什么
3. 有明确进入路径
4. 世界地图清楚
5. 任意节点可返回地图
6. 阅读页清晰专业
7. 没有 AI 也可探索
8. 档案馆能快速找内容
9. 内容有区域、时间、状态、权限
10. 移动端可用
11. public 与 private 数据模型分离
12. README 解释清楚项目理念
