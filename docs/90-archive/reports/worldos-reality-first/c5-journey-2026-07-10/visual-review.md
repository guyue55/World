# C5 路径与内容地点真实视觉审查

## 逐图结论

- Paths 首页由分岔广场、六条可选择星路和实体路标构成，隐藏文字后仍能看出多路分岔，不再是大标题与卡片目录。
- Path Detail 在同一首屏显示路线、站点、当前位置、可达与未达状态；进度后已走站出现勾选，完成后出现抵达层和下一条路线，而不是只显示 `7/7`。
- mobile 路线沿纵向道路排列，当前站详情、重置与底部导航均在安全区内；未发现横向溢出。
- Node 首屏由阅读房间、区域窗、关系门、正文入口和自然出口组成。地点护照压缩到房间右上物件，不再成为正文的大侧栏。
- Node 正文保持约 68-76ch，关系区逐门显示“为什么相关”；source context 写入关系链接，Path context 在房间内提供回路线与完成此站。
- 图片阻断时 Paths 和 Node 都切换为同事实源静态列表，未出现空白或空间控件叠加。

## 真实流程

- 完整走完 `first-visit` 七站：Path -> Node 1 -> ... -> Node 7 -> Path complete；七个 pathname 唯一，local progress 为 completed，重置后回到第 1 站。
- `path-journey.mp4` 逐帧包含稳定路线起点、第一站、完成动作、第二站、回路线并恢复第 2 站位置。
- `node-explore.mp4` 逐帧包含地点抵达、正文阅读、关系门区域、进入相关地点与浏览器返回来源。
- 12 个按公开索引等距抽样的 Node 均有标题、至少两个关系门、四个场景出口、正文入口；无公开工程文案、横向溢出和浏览器错误。
