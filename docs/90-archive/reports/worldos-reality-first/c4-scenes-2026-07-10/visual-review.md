# C4 时间河与档案馆真实视觉审查

## 审查结论

- Timeline 的主体是贯穿首屏的河道、七个时间锚和随河段变化的事件水纹；Archive 的主体是八组馆藏书架、卷宗标记和检索台。两者不再使用相同的双栏 hero、统计卡或 route guide。
- arrival 与 text-hidden 图均逐张并排查看。隐藏所有文字后，月光河谷、弯曲锚链与档案厅、书架矩阵仍可直接区分，因此 C4 的空间差异门禁通过。
- 390x844 下，时间锚沿纵向河道分布；档案馆筛选器为两列，底部导航未遮挡检索台。浏览器观察未发现横向溢出、公开工程文案、控制台错误或 inspector 与导航重叠。
- 图片被阻断时，两页都切换为同一公开事实源生成的可读列表，依赖背景坐标的控件不再残留叠加。

## 流程审查

- `timeline-review.mp4` 逐帧包含：进入旧河段、显示当前事件、打开事件详情、关闭详情、回到最新河段。
- `archive-search.mp4` 逐帧包含：输入查询、馆内卷宗与分区计数变化、打开卷宗详情、关闭详情、清除查询并恢复馆藏。
- keyboard smoke 使激活时间锚发生变化；刷新可恢复 6 月 30 日河段。
- Archive 输入前 Fuse 状态为 idle；首次输入后才新增一个脚本资源。筛选把可见卷宗从默认集合缩小到 1-5 个，空结果、清除、进入 Node、浏览器返回并恢复上下文均通过。

## 两轮修复

1. 首轮修复 mobile 时间锚重叠、档案馆筛选器截断、详情与 mobile 截图串状态。
2. 次轮修复图片降级层叠加，并给时间河录屏增加详情出现与返回最新河段的硬断言。

最终证据在 fresh production build 后重新生成；`browser-observations.json` 的 16 个模式与 9 条 flow check 均无失败。

## 未冒充完成的全局残余

- 当前 production build 的 shared First Load JS 为 158 kB，仍高于冻结的 130 kB。C4 没有新增全站重型运行时，Fuse 保持按输入动态加载；该全局残余继续由 C6-C8 关闭，不能据此宣称终局性能通过。
- 本记录只确认 C4。Paths、Node、迁移叙事、感官层、Lighthouse、静态等价与最终独立审查仍按 C5-C9 继续。
