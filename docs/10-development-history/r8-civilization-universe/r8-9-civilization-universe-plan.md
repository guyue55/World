# R8.9｜真实宇宙文明与节点完成线

## 定位

R8.9 继续从 R8.8 的“场景宇宙”向“文明宇宙”推进：让页面不仅有场景和动作，而且有文明层级、对象工作台、节点生命、七个不迷路锚点、现实/世界解释与静态安全 API。

## 目标与准则

- 前台浪漫，后台清醒，档案可靠。
- 入口清澈，深处浩瀚。
- 世界感必须服务理解、探索、复访、维护与回望。
- 任何动态体验不得依赖真实数据库或真实 AI 才能成立。
- 编码保持模块化、组件化、高内聚低耦合。

## 任务规划

### 阶段 1：文明层与对象层

- 建立 `data/r8-civilization-universe/civilization-universe.json`。
- 定义自然层、生活层、创作层、工程层、知识层、精神层。
- 定义纸页、灯、罗盘、时间河、卷宗、雾门等对象及动作。

### 阶段 2：节点生命与页面锚点

- 建立节点护照、成长轨迹、关系说明、可信层、下一步动作。
- 为核心路径建立“我在哪里 / 这里是什么 / 能做什么 / 能去哪 / 谁能看到”。

### 阶段 3：全站动态接入

- 新增文明运行浮层、对象工作台、世界路标。
- 核心页面接入 `CivilizationUniverseSection` 与 `NodeLifeConstellation`。
- 保持 reduced-motion、移动端信息降级和静态安全。

### 阶段 4：质量门禁与全量打包

- 新增 R8.9 聚合检查与边界检查。
- 复跑 check / lint / typecheck / routes / audit / artifact verification。
- 使用 ASCII 英文文件名全量打包，避免中文文件名乱码。

## 验收标准

- 核心页面不再只是背景动效，而具备文明解释、对象动作与节点生命。
- 访问者知道自己在哪里、这里是什么、能做什么、能去哪。
- 节点不是普通文章，而能展示护照、成长、关系与下一步动作。
- 动态层不产生生产写入，不读取私密 vault，不依赖真实 AI。
