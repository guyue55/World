# V9｜服务化平台版：全任务与扩展任务规划

## 总目标

V9 的目标是把古月浮屿从静态优先的个人数字世界推进到服务化平台骨架：身份、RBAC、API、审计、存储端口、集成适配器、平台控制台和 V10 长期智能世界交接。

## 阶段规划

| 阶段 | 批次 | 重点 |
|---:|---:|---|
| 阶段 1 | 01-04 | 服务平台边界、身份与租户地基 |
| 阶段 2 | 05-08 | API、存储、审计与适配器骨架 |
| 阶段 3 | 09-12 | 平台控制台、队列、治理与协作 |
| 阶段 4 | 13-16 | 服务化硬化、安全边界与 V10 交接 |

## 扩展任务

- 多租户边界
- 身份提供者端口
- RBAC 权限矩阵
- API 网关契约
- 存储端口与迁移运行手册
- 审计台账
- 平台控制台
- 操作队列
- 集成适配器
- 密钥边界
- 服务质量预算
- 默认私密原则
- AI 服务代理策略
- V10 长期智能世界交接

## 结构约束

- `data/v9-service-platform/` 只放平台数据与证据。
- `src/features/v9-service-platform/` 只提供领域数据、类型与守门逻辑。
- `src/components/v9-platform/` 只提供 V9 页面组件。
- `/v9-platform` 是页面入口，`/api/v9/service-health` 是安全 API 样板。
- 不声明真实服务上线，`serviceLive` 与 `productionLive` 必须保持 `false`。
