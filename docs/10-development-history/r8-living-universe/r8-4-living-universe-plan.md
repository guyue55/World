# R8.4｜真实世界 / 宇宙动态化完成线

## 定位

R8.4 继续基于 R8.3，不再只做“全站 route-aware 动效”，而是把古月浮屿推进为更接近真实世界 / 宇宙的动态体验：每个页面都有三景深、区域身份证、文明痕迹、世界动作、旅程连续和降级边界。

## 设计依据

- 访问者不是打开网站，而是进入世界。
- 先创造世界，再让内容降临其中。
- 世界需要近景、中景、远景，而不是单层平面 UI。
- 每个关键行为都应该像世界中的动作。
- 诗意命名负责氛围，现实说明负责理解。
- 前台浪漫，后台清醒，档案可靠。

## 阶段规划

### 阶段 1：宇宙三景深与文明痕迹

- 近景：书桌、灯、纸页、罗盘、卷宗、节点护照。
- 中景：地图、路径、节点、档案、灯塔、工作台。
- 远景：雾区、未来房间、沉睡岛、未命名星域。
- 文明痕迹：世界宣言、区域身份证、证据链、审计、生命周期。

### 阶段 2：区域身份证与不迷路系统

每个页面回答：

1. 我在哪里。
2. 这里是什么。
3. 我能做什么。
4. 我能去哪。
5. 谁能看到。

### 阶段 3：世界动作与连续旅程

新增全局动作：观测、展开、漂流、回声、点亮、归档、归途。所有动作都是本地前端反馈，不依赖数据库或真实 AI。

### 阶段 4：质量边界与封包

- 支持 reduced-motion。
- 移动端保留可读性。
- 不引入重型 3D、真实 AI、数据库依赖。
- 使用 ASCII 英文文件名打包，避免中文文件名乱码。

## 完成标准

- 首页像创世原点。
- Atlas 像可观测星图。
- 时间河像带回声和成长轨迹的河。
- 档案馆像现实模式下的文明书库。
- 节点像有护照和生命阶段的存在物。
- 灯塔像有边界的世界角色。
- 创世台像主人工作层。
- 所有页面都可回入口、地图、时间河或灯塔。
