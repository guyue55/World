# 古月浮屿｜V1 关键文案冻结

> 阶段：V1  
> 性质：关键界面文案冻结文件  
> 目标：统一第一阶段的世界气质、入口表达、空状态、错误状态和关键操作反馈。

---

## 1. 文案总原则

### 1.1 文案气质

文案应当：

- 温柔
- 克制
- 清晰
- 有世界感
- 有现实解释
- 不玄虚
- 不过度营销
- 不像普通博客模板

### 1.2 双层语言

世界语言负责氛围，现实解释负责理解。

例如：

```text
记忆湖
生活照片、日常片刻、旅行与家庭记忆。
```

### 1.3 危险操作去诗化

危险操作必须清楚说明现实后果。

例如：

```text
永久删除后无法恢复。
```

不要只写：

```text
让它化作星尘。
```

## 3. 世界说明文案

```text
古月浮屿不是传统博客，也不是单纯作品集。

这里的内容不是简单列表，
而是世界中的节点。

一篇技术文章可以成为技术星域里的一颗星，
一个项目可以在产品工坊中被打磨，
一段灵感可以先作为种子漂浮在云层，
一张生活照片可以沉入记忆湖，
一次世界变化会被时间河记录。

AI 可以作为灯塔辅助搜索、整理和导览，
但这个世界必须在没有 AI 时依然能够自然运行。
```

## 4. 主区域说明文案

### 4.1 创世原点

```text
一切从这里开始。
这里解释古月浮屿是什么，以及如何进入这个世界。
```

### 4.2 技术星域

```text
工程实践、AI Agent、CLI 工具、系统排查与技术复盘在这里汇成星图。
```

### 4.3 产品工坊

```text
想法在这里被拆解、打磨、装配，最终变成工具、产品或实验。
```

### 4.4 灵感云层

```text
尚未成形的想法先在这里漂浮，等待发芽、归位或沉睡。
```

### 4.5 记忆湖

```text
生活中的光影沉入这里，随时间泛起涟漪。
公开层只显示被允许看见的片刻，更深处默认被保护。
```

### 4.6 时间河

```text
这里记录节点、区域、法则与世界自身如何在时间中生长。
```

### 4.7 档案馆

```text
当世界太浪漫时，档案馆提供一条清晰的现实道路。
```

### 4.8 AI 灯塔

```text
灯塔照亮路径，但不替造物主选择道路。
无 AI 时，它以低光模式继续指引。
```

```text

地图、路径和档案馆仍然可用。
```

## 7. 错误状态文案

### 7.1 404

```text
你走到了一片尚未命名的雾区。
这里还没有稳定的路径。
```

按钮：

- 回到世界地图
- 回到创世原点
- 打开档案馆
- 点亮灯塔

### 7.2 无权限

```text
这扇门只为特定的人打开。
里面保存着更私密的记忆。
```

按钮：

- 返回外庭
- 查看公开摘要
- 回到地图

### 7.3 内容未公开

```text
这颗星还没有被点亮。
它可能仍在生长，或只属于世界深处。
```

### 7.4 数据加载失败

```text
星图暂时没有展开。
请稍后重试，或先前往档案馆。
```

## 8. 操作反馈文案

### 8.1 保存

```text
一颗新的种子已被安放。
```

或按类型：

- 一颗新的种子已落入灵感云层。
- 这张纸页已经收进档案馆。
- 一片光影已沉入记忆湖。
- 这台装置已在工坊中点亮。
- 这封信已放入时间抽屉。

### 8.2 发布

```text
这颗星已经被点亮，旅人可以看见它了。
```

必须附现实说明：

```text
发布后，所有访客都可以访问该内容。
```

### 8.3 归档

```text
它已沉入档案馆。
不再活跃，但仍可被回望。
```

### 8.4 封存

```text
它已被封存。
不会主动展示，也不会出现在公开路径中。
```

### 8.5 删除

```text
永久删除后无法恢复。
也可以选择封存或归档。
```

### 8.6 建立关系

```text
两颗星之间出现了一条新的星线。
```

### 8.7 复活

```text
一颗沉睡的星重新亮起。
```

## 9. 节点护照文案

```text
节点护照
```

字段说明：

- 类型：它是什么
- 区域：它属于哪里
- 生命阶段：它现在生长到哪里
- 来源：它从哪里来
- 权限：谁能看见它
- AI 参与：是否由 AI 辅助生成或整理
- 相关路径：它属于哪条探索路线
- 下一步：它还能如何继续生长
