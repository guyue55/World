# V8｜第八轮真实生产发布与运维闭环：所有任务与扩展任务规划

## 总目标

V8 不是继续堆概念，而是把 V7 的公开发布运营结构推进到真实生产运维闭环。它必须把部署流水线、生产观测、事故响应、域名 SEO、性能证据、人工签收、回滚演练和 V9 服务化交接全部结构化落库。

## 阶段 1 / 4：真实生产部署流水线与环境准备

- 批次 01：生产部署契约
- 批次 02：环境变量与密钥边界
- 批次 03：预览环境 smoke test 计划
- 批次 04：阶段一收口与 lint

扩展任务：区分 preview / production / rollback / signoff 证据，不允许把本地结构通过伪装成生产上线。

## 阶段 2 / 4：生产观测、告警与事故响应

- 批次 05：生产观测矩阵
- 批次 06：告警分级模型
- 批次 07：事故响应 playbook
- 批次 08：阶段二收口与 lint

扩展任务：建立 P0-P3 分级、隐私泄露 P0、AI 越权 P0、生产不可访问 P0/P1。

## 阶段 3 / 4：域名、SEO、性能与外部访问证据

- 批次 09：域名与 HTTPS 检查清单
- 批次 10：SEO 索引提交策略
- 批次 11：Web Vitals 外部证据
- 批次 12：阶段三收口与 lint

扩展任务：保留 canonical、robots、sitemap、OG、Search Console、移动端 smoke、外部性能证据。

## 阶段 4 / 4：人工签收、回滚演练与 V9 交接

- 批次 13：人工发布签收
- 批次 14：回滚演练
- 批次 15：生产证据台账
- 批次 16：V9 服务化平台交接

扩展任务：把身份系统、RBAC、真实审计、后端 API、私密档案服务、多端协作交给 V9，不污染 V8 边界。

## 工程原则

- 模块化：feature 层聚合数据与守门逻辑。
- 页面化：`/v8-production` 只做生产运维入口。
- 组件化：Hero、Dashboard、StageBoard、OperationsMatrix、HandoffPanel 独立。
- 高内聚低耦合：V8 不直接修改 V7 业务模块，仅继承其发布运营边界。
- 证据优先：没有真实生产地址、人工签收和外部访问证据，`productionLive` 必须保持 false。
