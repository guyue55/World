## 关键问题

### P0 / P1：体验页未完全接入全局导航

当前新增页面存在：

```text
/world-map
/constellation
/time-river
/lighthouse
/memory-graph
/theme-system
```

但全局导航数据仍主要指向旧入口：

```text
/atlas
/timeline
/ask
/archive
/paths
/status
```

位置：

```text
data/domains/experience/navigation-state-contract.json
```

影响：

```text
用户仍会通过主导航进入旧页面，而不是新的宇宙化体验页。
“重做导航为宇宙导航 / 世界地图”没有完全兑现到全局导航层。
```

建议：

```text
把导航契约升级为 experience-aware：
地图 -> /world-map
时间河 -> /time-river
灯塔 -> /lighthouse
新增：星图 /constellation、记忆图谱 /memory-graph、主题 /theme-system
保留旧页面为兼容入口，但不要作为体验主入口。
```

### P1：体验检查脚本覆盖偏弱

文件：

```text
scripts/check-experience-realization.ts
```

当前主要检查：

```text
文件是否存在
8 个批次 JSON 是否存在
```

不足：

```text
没有检查导航是否接入新路由
没有检查首页是否引用 UniverseHero
没有检查 /world-map 是否包含全部 experienceNodes
没有检查主题系统是否有 4 个主题
没有检查 V6 world network 页面是否保留 private 脱敏边界
没有检查 route artifact smoke
```

建议补强为：

```text
读取 src/lib/navigation 或 navigation-state-contract.json
校验新体验路由进入导航或至少进入首页/世界地图
校验 data/experience/experience-realization-stage.json 的 routes 与实际 src/app 路由一致
校验新页面不能使用 Date.now / Math.random / typeof window
校验构建后 .next/server/app 下产物存在
```

### P1：patch 交付不够干净

应用 patch 时观察到：

```text
Hydration 修复 patch 包含 tsconfig.tsbuildinfo 相关 hunk，导致 patch 末尾 malformed / skip。
体验兑现 patch 与 hydration patch 在 scripts/check-v6-24.ts 上有重复修改，叠加应用时产生 .rej。
```

影响：

```text
源码本体可运行，但交付 patch 不够干净。
后续别人按顺序套 patch，容易产生 .rej 或误以为失败。
```

建议：

```text
重新生成干净 patch：
排除 tsconfig.tsbuildinfo
排除 .next / reports / node_modules / .rej
以 v6-封版 + hydration fix 后的干净基线生成体验兑现 patch
或者直接重新打一个新的“体验兑现后完整代码包”
```

### P1：体验仍偏静态展示，数据驱动不足

当前新增页面能访问，但多处为静态数组：

```text
src/app/constellation/page.tsx
src/app/time-river/page.tsx
src/components/experience/LighthouseQueue.tsx
src/components/experience/MemoryGraphView.tsx
```

影响：

```text
星图、时间河、AI 灯塔、记忆图谱还不是从真实内容/版本/AI审计数据生成。
视觉上比普通博客强，但产品逻辑仍偏“静态说明页”。
```

建议：

```text
constellation 从 node/content 数据生成
time-river 从版本阶段数据生成
lighthouse 从 AI 建议审计队列数据生成
memory-graph 从公开 memory graph 数据生成，private 只暴露脱敏节点
```

### P2：主题系统只是主题方案展示，尚未形成真正切换机制

当前：

```text
/theme-system 展示 nature / cosmos / library / atelier 四种主题说明。
```

不足：

```text
没有全局主题状态
没有主题切换控件
没有主题 token / layout variant 落地
没有把不同主题应用到首页、星图、时间河等页面
```

建议：

```text
建立 src/features/theme-system/
建立 ThemeProvider 或服务端 cookie/theme param 策略
CSS variables + layout variants
移动端主题降级策略
```

### P2：全局导航仍是普通 pill nav，不是宇宙导航

组件：

```text
src/components/navigation/RouteAwareNav.tsx
src/components/navigation/MobileRouteNav.tsx
```

当前仍是常规顶部导航和底部导航。

建议：

```text
新增 CosmicRouteNav / WorldMapOverlay
桌面端提供星图式主入口
移动端提供“世界罗盘”式入口
保留原导航为降级模式
```
