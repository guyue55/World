# R8.1 Dynamic World Experience Plan

## Positioning

R8.1 is a frontend dynamic experience repair line. It does not wait for database, login, real AI, vector search, or production deployment. Its purpose is to turn the current static world pages into an interactive, breathable, explorable frontend world.

## Goals

1. Add WorldRuntimeProvider for local runtime signals.
2. Add global motion layer with reduced-motion support.
3. Rebuild the home hero as an arrival ritual instead of a static introduction.
4. Add interactive Atlas focus and route preview.
5. Add interactive Time River filtering and event expansion.
6. Add node opening ritual preview.
7. Add Lighthouse interactive frontend console.
8. Keep homepage progressive and avoid governance-panel overload.
9. Keep all new docs and packaged filenames ASCII-safe.

## Non-goals

1. No database migration.
2. No auth provider integration.
3. No real AI model integration.
4. No productionLive claim.
5. No Chinese artifact filenames in the final package.
