# R2｜世界入口与沉浸体验重构：全任务与扩展任务规划

## 总目标

R2 的目标不是继续堆新概念，而是让访问者第一眼明确感到：这里不是普通博客，而是进入古月浮屿。

R2 必须服务最终目标：对外可进入、可探索、可停留；对内仍保持模块化、页面化、组件化、高内聚低耦合，方便未来扩展为中大型项目。

## 阶段 1：世界入口定型与渐进揭示

批次 01-04：体验原则、首页三景深、渐进揭示规则、阶段检查。

## 阶段 2：Atlas 世界地图与罗盘导航

批次 05-08：地图层级、七锚点罗盘、区域说明卡、阶段检查。

## 阶段 3：内容节点仪式与区域沉浸

批次 09-12：区域抵达仪式、节点展开仪式、世界/现实/静读模式、阶段检查。

## 阶段 4：移动端、可访问性与 R3 交接

批次 13-16：移动端沉浸、减动效与可访问性、R3 真实内容交接、封版打包准备。

## 扩展任务

- 世界入口首屏
- 近景 / 中景 / 远景三景深
- 渐进揭示
- Atlas 旅行者地图
- 七锚点罗盘
- 区域身份证
- 区域抵达仪式
- 节点展开仪式
- 世界模式 / 现实模式桥接
- 移动端沉浸适配
- 减动效与可访问性
- R3 真实内容交接

## 质量要求

每完成一个阶段后必须执行对应 `check:r2-world-experience:stage-0X` 和 `lint`。最终执行 JSON、仓库、R2 聚合、R2 边界、lint、typecheck、build、audit，并全量打包。
