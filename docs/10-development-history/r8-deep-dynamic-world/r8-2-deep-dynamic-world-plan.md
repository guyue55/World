# R8.2｜真实宇宙动态化深化线

## 目标

R8.1 已让首页和若干入口具备动态体验，但整体仍有大量页面像静态信息面板。R8.2 的目标是把动态体验从“局部组件”扩展为“全站世界运行层”：访问者无论从首页、地图、时间河、档案馆、路径、灯塔还是节点详情进入，都能感到自己处在同一个会呼吸、可反馈、可继续探索的古月浮屿。

## 阶段

1. 全局宇宙运行层：宇宙场、路线转场、动态罗盘、旅程记忆。
2. 核心页面动态化：Atlas、Timeline、Archive、Paths、Ask。
3. 节点与阅读宇宙化：阅读进度、关系轨道、节点护照、继续旅程。
4. 质量边界与封装：检查脚本、无障碍、性能边界、ASCII 打包。

## 不做

- 不接数据库。
- 不接真实登录。
- 不接真实 AI。
- 不把动画做成阻塞体验。
- 不牺牲可读性和无障碍。

## 完成标准

- 前端不再只是静态页面。
- 全站有统一世界运行时。
- 关键页面都有可操作状态和反馈。
- reduced motion 下仍可完整使用。
- 打包文件名继续使用 ASCII，避免中文乱码。
