# R8.8｜真实宇宙场景与节点完成线

## 定位

R8.8 继续从 R8.7 的“可操作世界动作系统”向下推进，把动态化落到具体场景、节点生命、区域故事、现实解释、主线旅程和模式切换上。

## 目标

- 页面不是背景动效，而是可抵达的场景。
- 节点不是文章，而是有生命阶段、关系、护照、来源和下一步动作的存在物。
- 世界模式负责沉浸，现实模式负责效率，静读模式负责阅读。
- 每个核心页面都有“我在哪里、这里是什么、能做什么、能去哪”的答案。
- 动态层必须静态安全，不依赖真实数据库和真实 AI。

## 任务规划

### 阶段 1：场景宇宙数据协议

1. 建立 `data/r8-scene-universe/scene-universe.json`。
2. 为核心路由声明世界名、现实名、地点、故事、景深、生命阶段和下一步路径。
3. 建立近景 / 中景 / 远景对象白名单。
4. 声明 R8.8 完成标准。

### 阶段 2：全站场景运行层

1. 新增 `SceneUniverseEngine`。
2. 新增 `SceneDepthField`，提供三景深宇宙场。
3. 将 R8.8 接入 `WorldShell`。
4. 通过 localStorage 记录轻量复访旅程，并加入容错。

### 阶段 3：核心页面逐个动态化

1. 新增 `SceneUniverseSection`。
2. 接入首页、Atlas、Timeline、Archive、Paths、Ask、节点页、R4、R7 等核心页面。
3. 支持世界 / 现实 / 静读模式切换。
4. 为每页提供动作仪式、场景物件和继续路径。

### 阶段 4：质量门禁与交接

1. 新增 R8.8 聚合检查脚本。
2. 新增 R8.8 边界检查脚本。
3. 更新 `package.json` 脚本。
4. 保持静态安全、隐私边界、reduced-motion 和移动端降级。

## 编码准则

- 数据在 `data/`，组件在 `src/components/`，API 在 `src/app/api/`，检查脚本在 `scripts/`，文档在 `docs/`。
- 组件只负责呈现与轻量本地交互，不产生生产写入。
- localStorage 必须有 try/catch 容错。
- 动态效果必须尊重 `reducedMotion`。
- 不引入新依赖，不污染已有数据协议。
