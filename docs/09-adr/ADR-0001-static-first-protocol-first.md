# ADR-0001｜采用静态优先与协议先行

## 状态

Accepted

## 背景

古月浮屿需要长期存在、可迁移、可导出、可重建。V1 阶段如果过早引入数据库、复杂后台或强依赖在线服务，会提高维护成本，并让世界骨架受实现细节牵制。

## 决策

V1 采用：

```text
静态优先
协议先行
Markdown + JSON
Git 可追踪
本地检查脚本守门
```

## 后果

好处：

```text
部署简单
迁移容易
内容可读
AI 可插拔
无后台也可运行
```

代价：

```text
V1 不提供完整后台编辑器
V1 不提供复杂权限在线管理
V1 不提供完整 RAG
```

## 结论

```text
先让世界站起来，
再让它变得更神秘。
```
